## 주요 신규 기능 & 실전 활용

### v2.1.143 — 플러그인 의존성 관리, 백그라운드 세션 유연성 강화 (5/15)

v2.1.143은 플러그인 생태계의 안정성과 백그라운드 작업 유연성을 높이는 업데이트입니다.

**플러그인 의존성 관리**: `claude plugin disable`이 다른 활성 플러그인이 의존하는 플러그인을 비활성화하려 할 때 거부합니다. 의존성 체인을 한 번에 해제할 수 있는 복사·붙여넣기 가능한 명령어 힌트도 함께 표시합니다. 반대로 `claude plugin enable`은 필요한 의존성을 자동으로 함께 활성화합니다.

**마켓플레이스 토큰 비용 표시**: `/plugin` 마켓플레이스 탐색 화면에 각 플러그인의 예상 컨텍스트 비용(토큰 추정치)이 표시됩니다. 플러그인 설치 전에 컨텍스트 윈도우에 미치는 영향을 사전에 파악할 수 있습니다.

**`worktree.bgIsolation: "none"` 설정**: 백그라운드 세션이 worktree를 생성하지 않고 작업 디렉토리를 직접 편집하도록 설정할 수 있습니다. 서브모듈이 많거나 worktree가 실용적이지 않은 저장소에서 유용합니다.

```json
{
  "worktree": {
    "bgIsolation": "none"
  }
}
```

**PowerShell `-ExecutionPolicy Bypass`**: PowerShell 도구가 기본적으로 `-ExecutionPolicy Bypass`를 전달합니다. 기존 실행 정책을 유지하려면 `CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1` 환경변수로 비활성화할 수 있습니다.

**백그라운드 세션 모델/에포트 유지**: 백그라운드 세션이 idle 상태에서 깨어난 후에도 이전에 설정한 모델과 effort 레벨을 유지합니다.

## 개발자 워크플로우 팁

### CLAUDE.md는 "권고", Hook은 "강제" — 규칙 분리 전략

Claude Code에서 가장 빈번한 실수 중 하나는 모든 규칙을 CLAUDE.md에 넣는 것입니다. 2026년 베스트 프랙티스 가이드들이 공통적으로 강조하는 핵심 구분이 있습니다.

**CLAUDE.md (advisory, ~80% 준수)**: 코딩 스타일, 아키텍처 가이드, 네이밍 컨벤션 등 "지키면 좋은" 규칙. Claude가 컨텍스트가 복잡해지면 잊을 수 있습니다.

**Hook (deterministic, 100% 실행)**: 린트, 포맷팅, 위험한 명령 차단 등 "반드시 지켜야 하는" 규칙. 종료 코드 0은 허용, 2는 차단이며, 예외 없이 매번 실행됩니다.

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{
        "type": "command",
        "args": ["prettier", "--write", "$CLAUDE_FILE_PATH"]
      }]
    }]
  }
}
```

**실무 기준**: "Claude가 이 규칙을 한 번이라도 무시하면 문제가 생기는가?" — Yes이면 Hook, No이면 CLAUDE.md입니다.

[SmartScope](https://smartscope.blog/en/generative-ai/claude/claude-code-best-practices-advanced-2026/) | [Claude Code Hooks Docs](https://code.claude.com/docs/en/hooks)

### 서브에이전트 컨텍스트 격리 — "탐색 오염"을 구조적으로 방지하기

대규모 코드베이스에서 Claude가 탐색과 구현을 같은 세션에서 수행하면, 실패한 접근 방식이 컨텍스트에 쌓여 후속 시도의 품질이 떨어집니다. `.claude/agents/` 디렉토리에 탐색 전용 서브에이전트를 정의하면, 탐색이 별도 컨텍스트 윈도우에서 이루어져 **구조적으로 오염을 방지**합니다.

핵심은 "병렬화"가 아니라 "컨텍스트 분리"입니다. 실패한 탐색 경로가 구현 세션에 영향을 주지 않으므로, 여러 접근 방식을 시도한 후에도 구현 세션은 깨끗한 상태를 유지합니다.

[Claude Blog: Large Codebases](https://claude.com/blog/how-claude-code-works-in-large-codebases-best-practices-and-where-to-start)
