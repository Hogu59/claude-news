## 개발자 워크플로우 팁

### 컨텍스트 60% 임계값 규칙 — /compact보다 효과적인 리셋 전략

여러 실무자들이 독립적으로 같은 규칙에 도달했습니다: **컨텍스트가 60%를 초과하면 세션을 리셋**하는 것입니다. `/compact`보다 더 효과적인 이유는, 어떤 정보가 살아남을지를 사용자가 직접 통제할 수 있기 때문입니다.

**방법**:
1. 현재 계획과 진행 상황을 마크다운 파일에 덤프
2. `/clear`로 세션 리셋
3. 새 세션에서 해당 파일을 읽게 하여 재시작

```bash
# 진행 상황 저장 후 리셋 예시
# Claude에게: "현재까지의 계획과 진행 상황을 progress.md에 정리해줘"
# 그 후 /clear
# 새 세션: "progress.md를 읽고 이어서 작업해줘"
```

`/compact`는 Claude가 자동으로 요약하므로 중요한 맥락이 사라질 수 있습니다. 수동 리셋은 정확히 필요한 정보만 다음 세션에 전달합니다.

[Builder.io](https://www.builder.io/blog/claude-code-tips-best-practices)

### lessons.md 패턴 — Claude가 스스로를 개선하게 하기

Claude가 실수를 할 때마다 교정 후 `lessons.md`에 규칙을 추가하게 합니다. 시간이 지나면 Claude가 문자 그대로 **프로젝트에 맞춰 자기 자신을 개선**합니다. CLAUDE.md와 분리하여 유지하면 핵심 설정이 방해받지 않습니다.

```markdown
# lessons.md (예시)
- 이 프로젝트에서 import 경로는 항상 @/ alias를 사용한다
- 테스트 파일명은 *.spec.ts가 아니라 *.test.ts를 사용한다
- API 응답은 항상 { data, error } 구조를 따른다
```

CLAUDE.md에 `@lessons.md`를 참조하면, 세션마다 자동으로 로드됩니다.

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices)

[Code with Claude London](https://claude.com/code-with-claude/london) | [London Extended](https://claude.com/code-with-claude/london-extended)

### /ultrareview — CI/스크립트에서 클라우드 코드 리뷰 (리서치 프리뷰)

`claude ultrareview` 서브커맨드를 통해 인터랙티브 세션 없이 CI나 스크립트에서 ultrareview를 실행할 수 있습니다. 여러 리뷰어 에이전트가 클라우드 샌드박스에서 병렬로 변경사항을 분석하고, 발견된 버그를 **독립적으로 재현·검증**한 후 결과를 반환합니다. 스타일 제안이 아닌 실제 버그에 집중합니다.

```bash
# CI에서 ultrareview 실행
claude ultrareview
```

Pro/Max 무료 3회 이후 회당 ~$5–$20. v2.1.86 이상 필요.

[Claude Code Docs](https://code.claude.com/docs/en/ultrareview)
