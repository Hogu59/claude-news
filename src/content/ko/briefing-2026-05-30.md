## 주요 신규 기능 & 실전 활용

### v2.1.157 — 마켓플레이스 없이 로컬 플러그인, `claude plugin init` 스캐폴딩 (5/29)

v2.1.157은 어제 출시된 Opus 4.8·Dynamic Workflows의 후속 안정화 릴리스로, **플러그인 개발 경험**을 크게 개선했습니다.

**`.claude/skills` 자동 로딩**: 이제 `.claude/skills` 디렉터리에 있는 플러그인이 **마켓플레이스 등록 없이 자동으로 로드**됩니다. 프로젝트 로컬 플러그인을 만들거나 팀 내부에서 공유할 때 별도 마켓플레이스 설정이 필요 없습니다.

**`claude plugin init` — 플러그인 스캐폴딩**: 새 플러그인의 기본 구조를 `.claude/skills`에 즉시 생성합니다.

```bash
# .claude/skills에 새 플러그인 스캐폴딩 (마켓플레이스 불필요)
claude plugin init my-plugin

# /plugin 인자 자동완성: 서브명령·설치된 플러그인명·알려진 마켓플레이스 플러그인
/plugin
```

**`--agent` 오버라이드**: `claude agents`에서 `settings.json`의 `agent` 필드가 dispatched 세션에 적용되며, `--agent <name>`로 세션별 오버라이드가 가능합니다.

```bash
# dispatched 세션에 특정 에이전트 지정
claude agents --agent reviewer
```

**`EnterWorktree` 세션 중 전환**: 이제 세션 도중에 Claude가 관리하는 worktree 간을 전환할 수 있습니다. 또한 에이전트 종료 시 worktree가 unlock 상태로 남아 `git worktree remove`/`prune`로 정리할 수 있습니다.

[GitHub v2.1.157](https://github.com/anthropics/claude-code/releases/tag/v2.1.157)

## 개발자 워크플로우 팁

### 로컬 플러그인 개발 루프 — `claude plugin init` + 자동 로딩

v2.1.157의 두 기능을 결합하면 마켓플레이스 왕복 없이 플러그인을 빠르게 만들고 테스트할 수 있습니다.

**실전 패턴**:
1. `claude plugin init my-plugin`으로 `.claude/skills`에 구조 생성
2. 스킬·hook·명령을 편집
3. `/reload-skills`(v2.1.152)로 세션 재시작 없이 즉시 반영
4. 동작 확인 후 마켓플레이스 또는 git 소스로 배포

`.claude/skills`가 자동 로드되므로, 프로젝트 리포지토리에 플러그인을 커밋해두면 팀원이 클론만 해도 동일한 도구 세트를 사용할 수 있습니다. `disallowed-tools` frontmatter(v2.1.152)와 결합하면 리서치 전용·구현 전용 스킬을 분리할 수 있습니다.

[GitHub v2.1.157](https://github.com/anthropics/claude-code/releases/tag/v2.1.157)

### Dynamic Workflows 오발동 제어 — "workflow" 단어 트리거 끄기

어제 도입된 Dynamic Workflows는 프롬프트에 "workflow"라는 단어가 들어가면 자동으로 트리거됩니다. 일반 문맥에서 의도치 않게 워크플로우가 시작되는 것을 막는 제어 옵션이 v2.1.157에 추가되었습니다.

```bash
# /config에서 "Workflow keyword trigger" 설정을 끄면
# 프롬프트의 'workflow' 단어가 더 이상 Dynamic Workflow를 트리거하지 않습니다
/config
```

키워드가 트리거된 직후라면 **backspace**(또는 `alt+w`)를 누르면 워크플로우 요청이 취소됩니다(글자가 지워지지 않습니다). "workflow"라는 단어를 자주 쓰는 문서 작업·리뷰 세션에서 유용합니다.

[GitHub v2.1.157](https://github.com/anthropics/claude-code/releases/tag/v2.1.157)
