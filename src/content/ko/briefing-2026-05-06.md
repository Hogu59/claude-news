## 개발자 워크플로우 팁

### Context7 MCP — 라이브러리 문서를 실시간으로 Claude에 주입

Claude Code가 프레임워크나 라이브러리 코드를 작성할 때, 학습 데이터 기반의 오래된 API를 사용하는 문제를 겪었다면 **Context7 MCP**가 해결책입니다. Upstash가 개발한 이 MCP 서버는 라이브러리 이름을 감지하면 **공식 소스에서 해당 버전의 최신 문서를 가져와** Claude의 프롬프트 컨텍스트에 자동 주입합니다.

두 개의 도구(`resolve-library-id`, `query-docs`)가 자동으로 동작하여, 별도 프롬프트 없이도 정확한 API 문서가 제공됩니다.

```bash
# 설치 (한 줄)
claude mcp add context7 -- npx -y @upstash/context7-mcp@latest
```

오픈소스이며 무료로 사용 가능합니다. Claude Desktop, Claude Code, Cursor 등 MCP 지원 클라이언트에서 동작합니다.

[GitHub](https://github.com/upstash/context7) | [Claude Plugin](https://claude.com/plugins/context7)

### 계획 모드의 "아직 구현하지 마" 가드 구문

Claude Code 파워 유저들 사이에서 검증된 계획 워크플로우가 있습니다. 핵심은 **"don't implement yet"(아직 구현하지 마)**이라는 가드 구문입니다. 이 문구가 없으면 Claude는 수정 과정을 건너뛰고 곧바로 코드를 작성하기 시작합니다.

1. "계획을 세워줘. **아직 구현하지 마.**" → 초안 생성
2. 에디터에서 계획을 열고 틀린 부분에 주석 추가
3. "모든 주석을 반영해줘. **아직 구현하지 마.**" → 수정된 계획
4. 모든 결정이 확정되면 구현 시작

계획을 수정하는 비용은 반쯤 완성된 기능을 되돌리는 비용보다 항상 적습니다.

[Smart WebTech](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/)

## 보안/제한 이슈

### Dario Amodei, "위험의 순간" 사이버 경고 — Mythos가 수만 개 취약점 발견 (5/5)

Anthropic CEO Dario Amodei가 **6~12개월의 패치 윈도우**가 있다고 경고했습니다. Mythos 모델이 발견한 소프트웨어 취약점 규모가 전례 없는 수준이라는 것입니다.

**핵심 수치**:
- **Firefox**: 이전 모델이 약 20개 취약점을 찾았던 곳에서 Mythos는 **약 300개**를 발견
- **전체 소프트웨어**: 총 발견 수가 **수만 개**에 달함
- Amodei는 이를 "올바르게 대응하면 더 나은 세상을 만들 수 있는 **위험의 순간(moment of danger)**"이라고 표현

Anthropic이 Mythos를 소수 파트너 기업에만 제한적으로 제공하는 이유는, 범죄자나 적대국이 이 능력을 악용할 수 있다는 우려 때문입니다. "랜섬웨어가 학교, 병원, 은행에 가하는 재정적 피해가 급증할 위험이 있다"고 경고했습니다.

[CNBC](https://www.cnbc.com/2026/05/05/anthropic-ceo-cyber-moment-of-danger-mythos-vulnerabilities.html) | [Decrypt](https://decrypt.co/366891/anthropic-warns-cyber-risk-window-ai-uncovers-flaws)
