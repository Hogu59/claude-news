## 주요 신규 기능 & 실전 활용

### v2.1.140 — subagent_type 유연 매칭, /goal 안정성 개선, 플러그인 진단 강화 (5/12)

v2.1.140은 v2.1.139의 대규모 기능 추가 이후 안정성과 사용 편의성을 높이는 업데이트입니다.

**subagent_type 유연 매칭**: Agent 도구에서 `subagent_type`을 지정할 때 대소문자와 구분자(하이픈, 언더스코어, 공백)를 무시하고 매칭합니다. `"Code Reviewer"`, `"code-reviewer"`, `"code_reviewer"` 모두 동일하게 `code-reviewer` 에이전트로 연결됩니다. CLAUDE.md나 프롬프트에서 에이전트 이름의 정확한 형식을 기억할 필요가 없어졌습니다.

**`/goal` 훅 비활성화 시 수정**: `disableAllHooks` 또는 `allowManagedHooksOnly`가 설정된 환경에서 `/goal`이 무한 대기하던 문제가 수정되었습니다. 이제 해결되지 않는 인디케이터 대신 명확한 메시지를 표시합니다.

**플러그인 폴더 경고**: `plugin.json`에서 명시적으로 키를 설정한 경우, 기본 컴포넌트 폴더(예: `commands/`)가 자동으로 무시되는 상황을 `/doctor`, `claude plugin list`, `/plugin`에서 경고합니다. 플러그인 개발 시 "왜 내 커맨드가 로드되지 않지?"라는 디버깅 시간을 줄여줍니다.

**백그라운드 서비스 안정성**: `claude --bg` 실행 시 백그라운드 서비스가 idle-exit 직전에 "connection dropped" 오류가 발생하던 문제와, 엔터프라이즈 엔드포인트 보안 소프트웨어로 인한 시작 실패가 수정되었습니다.

## 개발자 워크플로우 팁

### Plan 모드를 "리서치 단계"로 분리하여 활용하기

최근 Claude Code 베스트 프랙티스 가이드들이 공통적으로 강조하는 패턴은 **리서치/계획과 구현의 명확한 분리**입니다. Plan 모드에서 코드베이스를 탐색하고 접근 방식을 설계한 뒤, 구현 단계로 넘어가는 2단계 워크플로우입니다.

**왜 효과적인가**: Plan 모드 없이 바로 구현에 들어가면, Claude가 "잘못된 문제를 자신 있게 20분간 풀다가" 되돌리는 상황이 발생합니다. 계획을 먼저 세우면 잘못된 가정을 코드 작성 전에 잡을 수 있고, 계획을 수정하는 것이 반쯤 완성된 기능을 되돌리는 것보다 훨씬 저렴합니다.

**적용 기준**:
- 여러 파일을 수정해야 하는 변경 → Plan 모드 필수
- 익숙하지 않은 코드베이스 → Plan 모드 필수
- 아키텍처 결정이 필요한 작업 → Plan 모드 필수
- 단일 파일 버그 수정 → 바로 구현 가능

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices) | [Smart WebTech](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/)

### 병렬 에이전트 세션 + Agent View로 다중 작업 관리하기

v2.1.139의 `claude agents` 명령과 결합하면, 4~5개의 Claude 세션을 별도 git 브랜치에서 병렬 실행하는 팀 워크플로우가 가능합니다. 핵심은 **각 세션에 명확한 계획을 먼저 제공하는 것**입니다. 계획 없는 병렬 에이전트는 "병렬 혼란"을 만들 뿐입니다.

```bash
# Agent view로 전체 세션 상태 확인
claude agents

# 각 세션은 독립된 worktree에서 실행
# settings.json에서 worktree 기준점 설정
```

```json
{
  "worktree": {
    "baseRef": "head"
  }
}
```

## 알아두면 좋은 소소한 변경사항

- **심링크 설정 핫리로드 수정**: `~/.claude/settings.json`이 심볼릭 링크인 경우 변경 이벤트가 잘못 귀속되어 불필요한 `ConfigChange` 훅이 발동하던 문제 수정 (v2.1.140)
- **원격 managed settings 401 재시도**: 요청이 401을 받을 때 토큰을 강제 갱신한 뒤 1회 재시도 (v2.1.140)
- **`/loop` 불필요한 폴링 제거**: 완료 시 자동 알림이 오는 백그라운드 태스크에 대해 중복 wakeup 예약 방지 (v2.1.140)
- **Windows 이벤트루프 멈춤 수정**: `gh` 등 누락된 실행 파일을 동기적으로 반복 탐색(`where.exe`)하여 이벤트루프가 정지하던 문제 수정 (v2.1.140)
- **`Read` 도구 offset 파싱 완화**: 공백이 포함되거나 `+` 접두사가 붙은 offset 문자열도 정상 처리 (v2.1.140)
- **HTTP/SSE MCP 응답 크기 제한**: 비프로토콜 데이터를 스트리밍하는 MCP 서버로 인한 메모리 무한 증가 방지 — SSE 프레임당 16MB 상한 (v2.1.140)
- **에이전트 색상 팔레트 업데이트**: 에이전트 식별을 위한 시각적 색상 개선 (v2.1.140)
