### Anthropic 공식 블로그: "Claude Code가 대규모 코드베이스에서 작동하는 방법" (5/16)

Anthropic이 대규모 코드베이스에서 Claude Code를 효과적으로 사용하는 방법을 공식 블로그로 발표했습니다. 핵심 메시지는 **"모델 자체보다 Harness(인프라 계층)가 성능을 결정한다"**는 것입니다.

**Harness 5개 계층**:
1. **CLAUDE.md**: 계층적 컨텍스트 — 루트에 전체 가이드, 서브디렉토리에 로컬 규칙
2. **Hook**: 일관된 동작을 강제하는 스크립트
3. **Skill**: 필요할 때만 로드되는 전문 지식 (세션 비대화 방지)
4. **Plugin**: 조직 전체에 배포 가능한 패키지 설정
5. **MCP 서버**: 내부 도구·API 연결

**실전 권장사항**: 서브디렉토리에서 시작하여 탐색 범위를 좁히고, 3~6개월마다 설정을 검토하며, **DRI(Directly Responsible Individual) 또는 "에이전트 매니저"**를 지정하여 Claude Code 생태계를 관리할 것을 권고합니다.

## 개발자 워크플로우 팁

### Harness 아키텍처 — "모델보다 인프라가 중요하다"

Anthropic 공식 블로그가 강조하는 핵심은, Claude Code의 성능이 모델 선택이 아닌 **주변 인프라 계층(Harness)**에 의해 결정된다는 것입니다. 실무에서 바로 적용할 수 있는 패턴입니다.

**계층적 CLAUDE.md 구성**:
```
project-root/
├── CLAUDE.md              # 전체 아키텍처, 테스트 전략, 코딩 스타일
├── src/
│   └── CLAUDE.md          # src 전용 규칙
├── packages/auth/
│   └── CLAUDE.md          # auth 모듈 전용 규칙
```

루트 파일에는 전체 그림을, 서브디렉토리에는 로컬 규칙을 배치합니다. Claude는 작업 디렉토리의 CLAUDE.md를 자동으로 로드하므로, 불필요한 컨텍스트 소비를 줄일 수 있습니다.

**DRI(Agent Manager) 도입**: 성공적인 대규모 배포 사례에서 공통적으로 나타나는 패턴은, Claude Code 생태계(CLAUDE.md, Hook, Plugin, MCP)를 관리하는 전담 담당자를 지정하는 것입니다. 설정이 방치되면 모델이 개선되어도 성능이 정체됩니다.

[Claude Blog](https://claude.com/blog/how-claude-code-works-in-large-codebases-best-practices-and-where-to-start)

### 테스트는 외부 검증 수단 — 컨텍스트가 커질수록 테스트가 중요

테스트 없이 Claude가 자신의 판단으로 작업을 검증하면, 컨텍스트가 채워질수록 판단 품질이 저하됩니다. 테스트는 **세션 전체에서 정확성을 유지하는 외부 기준점** 역할을 합니다. red-to-green 사이클마다 Claude에게 모호하지 않은 피드백을 제공합니다.

Anthropic 내부 테스트에서 가이드 없이 시도할 때 성공률은 약 **33%**였으나, 계획 수립 + 테스트 기반 검증을 적용하면 크게 향상됩니다.

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices)

## 생태계 & 플러그인

### Microsoft, Claude Code 라이선스 대규모 취소 — Copilot CLI로 전환 (5/14~)

Microsoft가 내부 개발자의 Claude Code 라이선스를 **대규모로 취소**하고, GitHub Copilot CLI로 전환을 추진하고 있습니다.

**배경**: 2025년 12월부터 개발자·PM·디자이너에게 Claude Code 접근권을 제공했으나, 6개월간 내부 인기가 급증하면서 Copilot CLI 사용이 잠식당했습니다. Experiences + Devices 부문(Windows, Microsoft 365, Outlook, Teams, Surface)은 **6월 30일까지** Claude Code 사용을 중단해야 합니다.

**동기**: Microsoft 회계연도가 6월 30일에 종료되어 운영 비용 절감 목적이 크며, 자사 제품인 Copilot CLI의 내부 채택을 강제하려는 전략적 판단입니다.

**개발자 반응**: 내부 개발자들 사이에서 불만이 있다는 보도가 있습니다.

[Windows Central](https://www.windowscentral.com/microsoft/microsoft-cancels-claude-code-licenses-shifting-developers-to-github-copilot-cli-a-move-likely-driven-by-financial-motives) | [Yahoo Tech](https://tech.yahoo.com/ai/copilot/articles/microsoft-ditching-claude-code-copilot-133318848.html)
