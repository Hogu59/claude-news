## 주요 신규 기능 & 실전 활용

### v2.1.128 — ZIP 플러그인, MCP 도구 카운트, EnterWorktree 수정 (5/4)

Code with Claude SF 하루 전에 릴리스된 v2.1.128은 기능 추가와 20개 이상의 버그 수정을 포함합니다.

**`--plugin-dir`에 ZIP 아카이브 지원**: 플러그인을 디렉토리 대신 `.zip` 파일로 전달할 수 있게 되었습니다. 팀 간 플러그인 배포가 간편해졌습니다.

**`/mcp` 도구 카운트 표시**: `/mcp` 명령이 연결된 서버별 도구 수를 표시하고, 0개인 서버를 경고로 표시합니다. MCP 서버 연결 문제를 빠르게 진단할 수 있습니다.

**`EnterWorktree` 로컬 HEAD 기반 브랜치 생성**: 이전에는 워크트리 생성 시 원격 브랜치 기준으로 브랜치가 만들어져 **아직 push하지 않은 커밋이 누락**되는 문제가 있었습니다. 이제 로컬 HEAD에서 브랜치를 생성하여 미push 커밋이 보존됩니다.

**`/model` 피커 정리**: 중복된 Opus 4.7 항목이 하나로 합쳐지고, "Opus 4.7" 대신 간결하게 "Opus"로 표시됩니다.

```bash
# ZIP 플러그인 사용
claude --plugin-dir ./my-plugin.zip

# MCP 서버 상태 확인
# 세션 내에서 /mcp 입력 → 서버별 도구 수 및 0-tool 경고 확인
```

**기타 주요 변경**:
- `--channels`가 콘솔(API 키) 인증에서도 동작
- 서브프로세스(Bash, Hook, MCP, LSP)가 `OTEL_*` 환경변수를 상속하지 않음
- Auto 모드 분류기 오류 시 retry, `/compact`, `--debug` 힌트 표시
- MCP 재연결 시 도구 목록을 서버 프리픽스별로 요약 (대화 범람 방지)
- vim 모드: Normal 모드에서 `Space`가 커서 오른쪽 이동 (표준 vi 동작)

[GitHub v2.1.128](https://github.com/anthropics/claude-code/releases/tag/v2.1.128)

## 개발자 워크플로우 팁

### Claude Code Skills — 반복 워크플로우를 한 번 정의하고 재사용

2026년 Claude Code 워크플로우에서 가장 과소평가된 기능 중 하나가 **Skill 시스템**입니다. Skill은 Claude가 자동으로 트리거하는 저장된 재사용 워크플로우로, 한 번 올바른 작업 방식을 가르치면 이후 일관되게 적용됩니다.

활용 예시:
- **코드 리뷰 Skill**: "PR을 리뷰할 때 보안, 성능, 테스트 커버리지 순서로 확인하고 심각도를 매겨라"
- **커밋 Skill**: "커밋 전에 lint, type check, 관련 테스트를 실행하고, conventional commit 형식을 사용하라"
- **배포 Skill**: "스테이징 환경에서 스모크 테스트를 실행한 뒤에만 프로덕션 배포를 진행하라"

```bash
# Skill 목록 확인
/skill

# 현재 대화에서 Skill로 저장
/learner
```

단순 CLAUDE.md 지시사항과 달리, Skill은 **컨텍스트가 맞을 때 자동으로 활성화**되므로 매번 지시를 반복할 필요가 없습니다.

[Claude Code Docs](https://code.claude.com/docs/en/plugins) | [Builder.io](https://www.builder.io/blog/claude-code-tips-best-practices)

### `/mcp`로 MCP 서버 건강 상태 관리하기

v2.1.128에서 개선된 `/mcp` 명령을 활용하면 MCP 서버 연결 문제를 빠르게 파악할 수 있습니다. 특히 여러 MCP 서버를 사용하는 환경에서 **도구가 0개인 서버**는 설정 오류나 인증 만료를 의미할 수 있으므로 즉시 확인이 필요합니다.

또한 `workspace`가 예약된 MCP 서버 이름이 되었으므로, 기존에 `workspace`라는 이름으로 MCP 서버를 설정한 경우 이름을 변경해야 합니다.

```json
// settings.json — workspace 이름 충돌 시 변경 필요
{
  "mcpServers": {
    "my-workspace": {  // "workspace" 대신 다른 이름 사용
      "command": "node",
      "args": ["./mcp-server.js"]
    }
  }
}
```

## 알아두면 좋은 소소한 변경사항

- **`workspace` MCP 서버 이름 예약**: 기존에 `workspace`라는 이름으로 MCP 서버를 설정한 경우, 건너뛰기 경고가 표시됨. 이름 변경 필요
- **서브프로세스 OTEL_* 환경변수 상속 차단**: Bash, Hook, MCP, LSP 서브프로세스가 OpenTelemetry 환경변수를 상속하지 않아, 서드파티 도구의 의도치 않은 텔레메트리 전송 방지
- **Kitty 터미널 `/exit` 시 데스크톱 알림 수정**: `4;0;` 문자열이 매 종료 시 알림으로 표시되던 문제 해결
- **대용량 stdin 크래시 루프 수정**: `claude -p`로 10MB 이상 입력을 파이프할 때 발생하던 크래시 루프 수정
- **병렬 셸 호출 안정화**: 읽기 전용 명령 실패 시 형제 호출이 취소되던 문제 수정
- **1M 컨텍스트 모델 autocompact 차단 수정**: 1M 컨텍스트 모델에서 더 작은 autocompact 윈도우 설정 시 세션이 잘못 차단되던 문제 해결
