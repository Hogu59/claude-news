## 주요 신규 기능 & 실전 활용

### Claude Opus 4.8 출시 — Dynamic Workflows로 수백 개 서브에이전트 동시 실행 (5/28)

Anthropic이 최신 플래그십 모델 **Claude Opus 4.8**(모델 ID: `claude-opus-4-8`)을 출시했습니다. Claude Code v2.1.154에 즉시 통합되었으며, 코딩·에이전틱 작업·전문 업무 전반에서 프론티어 성능을 제공합니다.

**Dynamic Workflows (리서치 프리뷰)**: Claude Code의 가장 큰 변화입니다. Claude가 오케스트레이션 스크립트를 작성하여 **수십~수백 개의 병렬 서브에이전트**를 한 세션에서 실행합니다. 최대 **1,000개 에이전트**까지 생성 가능하며, 중단된 워크플로우는 처음부터 다시 시작하지 않고 **재개(resume)** 됩니다.

```bash
# Dynamic Workflow 시작 예시
claude "이 코드베이스의 모든 REST 엔드포인트를 OpenAPI 3.1 스펙으로 마이그레이션해줘"

# 실행 중인 워크플로우 상태 확인
/workflows
```

**Effort 제어 확장**: Opus 4.8은 기본적으로 **high effort**로 동작합니다. 특히 어려운 작업에는 `/effort xhigh`를 사용할 수 있습니다.

**Fast Mode 비용 절감**: Opus 4.8 Fast Mode가 이전 대비 **2.5배 빠른 속도**를 **표준 요금의 2배**에 제공합니다. 이전 Opus 모델의 Fast Mode보다 상당히 저렴해졌습니다.

**벤치마크**: Super-Agent 벤치마크에서 **유일하게 모든 케이스를 엔드투엔드로 완료**한 모델이며, 이전 Opus 모델과 GPT-5.5를 동일 비용 수준에서 능가합니다.

**가격**: 입력 $5/M 토큰, 출력 $25/M 토큰 (Opus 4.7과 동일). 프롬프트 캐싱으로 최대 90%, 배치 처리로 50% 절감 가능합니다.

**사용 가능 범위**: Enterprise, Team, Max 플랜에서 Dynamic Workflows 사용 가능. 토큰 소비가 일반 세션보다 크게 증가하므로 비용에 주의가 필요합니다.

### v2.1.154 — Opus 4.8 통합, 백그라운드 셸, 플러그인 제어 강화 (5/28)

**Lean 시스템 프롬프트 기본 적용**: 대부분의 모델에서 Lean 시스템 프롬프트가 기본값이 되어 토큰 사용량이 줄어듭니다.

**불필요한 질문 감소**: 충분한 컨텍스트가 있을 때 불필요한 객관식 질문을 줄이도록 개선되었습니다.

## 개발자 워크플로우 팁

### Dynamic Workflows 실전 활용 — 코드베이스 규모 마이그레이션

Dynamic Workflows는 기존에 수일~수주가 걸리던 대규모 코드베이스 마이그레이션을 단일 세션에서 처리할 수 있게 합니다.

**적합한 작업 유형**:
1. **수십만 줄 규모의 API 마이그레이션**: REST → GraphQL, 레거시 ORM → 최신 ORM 등
2. **코드베이스 전체 린팅/포매팅 규칙 적용**: 수백 개 파일에 일관된 스타일 적용
3. **테스트 커버리지 대규모 확충**: 기존 테스트 스위트를 검증 기준으로 활용

**비용 주의사항**: 1,000개 서브에이전트를 생성할 수 있지만, 각 에이전트가 토큰을 소비합니다. Pro 플랜의 경우 Programmatic Usage Credits(6/15 시행)과 겹치면 예산이 빠르게 소진될 수 있습니다.

**실전 패턴**:
```bash
# 1. 먼저 소규모로 테스트
claude "src/api/ 폴더의 엔드포인트 3개만 먼저 마이그레이션해줘"

# 2. 결과 확인 후 전체 실행
claude "나머지 전체 엔드포인트도 같은 패턴으로 마이그레이션해줘"

# 3. 워크플로우 상태 모니터링
/workflows
```

## 보안/제한 이슈

### v2.1.153 보안 수정 — OAuth 크리덴셜 유출·MCP 정책 우회 방지 (5/27)

v2.1.153에서 3건의 보안 수정이 포함되었습니다.

**OAuth 크리덴셜 유출 방지**: 사용자의 Anthropic OAuth 크리덴셜이 **커스텀 API 게이트웨이로 전송**되는 문제가 수정되었습니다. 커스텀 API 엔드포인트를 사용하는 환경에서 인증 토큰이 의도치 않게 서드파티에 노출될 수 있었습니다.

**`--strict-mcp-config` 우회 수정**: 서브에이전트 frontmatter에서 MCP 서버가 `--strict-mcp-config`와 엔터프라이즈 관리 정책을 **무시**하던 문제가 수정되었습니다. 엔터프라이즈 환경에서 보안 정책을 우회할 수 있는 심각한 문제였습니다.

### v2.1.154 보안 수정 — 데이터 유출 탐지 강화·위험 경로 차단 (5/28)

**벌크 저장소 전송 탐지 강화**: 대량의 리포지토리 데이터를 외부로 전송하려는 시도에 대한 데이터 유출 탐지가 강화되었습니다.

**`rm -rf $HOME` 차단 보완**: 후행 슬래시가 포함된 경우(`rm -rf $HOME/`) 위험 경로 차단이 작동하지 않던 문제가 수정되었습니다.

**미승인 MCP 서버 자동 승인 방지**: 파이프된 출력에서 승인되지 않은 MCP 서버가 자동 승인되던 문제가 수정되었습니다.

## 알아두면 좋은 소소한 변경사항

- **`/model` 동작 변경**: `/model`로 변경한 모델이 이제 **새 세션의 기본값으로 저장**됩니다. 현재 세션에서만 변경하려면 `s` 키를 누르면 됩니다. 기존 `modelPicker:setAsDefault` 키바인딩은 `modelPicker:thisSessionOnly`로 이름이 변경되었습니다 (v2.1.153)
- **`COLUMNS`/`LINES` 환경변수 전달**: 상태 바 명령에 터미널 크기 정보가 전달되어 너비에 맞는 출력이 가능합니다 (v2.1.153)
- **`skipLfs` 옵션 추가**: GitHub/git 플러그인 마켓플레이스 소스에서 Git LFS 다운로드를 건너뛸 수 있습니다 (v2.1.153)
- **세션 메모리 최적화**: 저장된 세션이 많은 머신에서 세션 재개 시 과도한 메모리 사용 문제가 수정되었습니다 (v2.1.153)
- **스트리밍 도구 실행 기본 활성화**: 모든 플랫폼에서 스트리밍 도구 실행이 항상 활성화됩니다 (v2.1.154)
- **`CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE` 폐기 예정**: 6월 1일에 제거됩니다 (v2.1.154)
