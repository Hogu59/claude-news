## 주요 신규 기능 & 실전 활용

### 침묵하던 장기 실행에 심장박동 — heartbeat와 OTel 메시지 상관관계 (v2.1.214)

v2.1.214가 무인·장기 세션의 **관측성 공백 두 곳**을 메웠습니다. 그동안 오래 도는 도구 호출은 완료까지 아무 신호가 없어 *멈춘 건지 도는 건지* 구분할 수 없었고, OTel 로그는 이벤트를 메시지 단위로 묶어 볼 수단이 없었습니다.

- **진행 heartbeat**: **오래 도는 도구 호출이 주기적으로 진행 신호를 냅니다** — 침묵하던 장기 빌드·테스트·MCP 호출이 살아 있는지 화면에서 확인됩니다. 6/15 무한 행 사고 유형(끊긴 소켓에 침묵 대기)을 겪어 본 무인 파이프라인이라면, 워치독의 판정 재료가 하나 늘어난 셈입니다.
- **OTel 메시지 수준 상관관계**: 로그 이벤트에 **`message.uuid`·`client_request_id`·`tool_source` 속성**이 붙어, *어떤 메시지의 어떤 도구가 낸 이벤트인지*를 백엔드에서 이어 붙일 수 있습니다. 콘텐츠 속성의 60KB 절단 한도도 `CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH`로 조정 가능해졌습니다.
- **덤 두 개**: 메모리 파일 frontmatter에 **ISO `modified` 타임스탬프**가 붙고(자동 메모리의 최신성 판정에 유용), `subagentStatusLine` 페이로드에 **reasoning effort**가 실려 커스텀 에이전트 행이 모델·강도를 렌더링할 수 있습니다.

```bash
# 관측성 파이프라인의 OTel 콘텐츠 절단 한도를 조정 (기본 60KB)
export CLAUDE_CODE_OTEL_CONTENT_MAX_LENGTH=120000
```

핵심은 7/16의 `--forward-subagent-text`(서브에이전트 속내를 스트림에), 7/17의 트랜스크립트 reasoning effort 기록과 같은 흐름입니다 — **무인 세션의 '무슨 일이 있었는지'가 릴리스마다 한 겹씩 더 기록**되고 있습니다. 관측성 스택을 붙인 팀은 이번 속성 3종으로 대시보드의 조인 키를 다시 잡을 값어치가 있습니다. [GitHub v2.1.214](https://github.com/anthropics/claude-code/releases/tag/v2.1.214)

### EndConversation — 에이전트가 대화를 '끝낼' 공식 경로가 생겼다 (v2.1.214)

v2.1.214가 **EndConversation 도구**를 추가했습니다 — **심하게 어뷰징하는 사용자나 탈옥(jailbreak) 시도에 대해 Claude가 세션을 스스로 종료**할 수 있습니다.

- **무엇이 새로운가**: 지금까지 적대적 입력에 대한 Claude의 선택지는 *거부하고 계속 응대*뿐이었습니다. 이제 세션 종료라는 **명시적 탈출구**가 도구로 생겼습니다 — claude.ai 웹에서 먼저 도입됐던 대화 종료 능력이 CLI 하네스로 내려온 것입니다.
- **누구에게 의미 있나**: Claude Code를 **SDK·헤드리스로 외부 입력에 노출**하는 구성 — 고객 대면 봇, 웹훅 트리거 파이프라인, 공개 이슈·PR에 반응하는 에이전트 — 에서, 적대적 세션이 무한정 토큰을 태우며 이어지는 대신 **종료라는 최종 상태**로 수렴할 수 있습니다.

핵심은 7/17의 세션 단위 상한 3종(검색·서브에이전트·MCP)과 같은 자리입니다 — 그때는 *폭주의 양*을 상한으로 막았다면, 이번엔 *적대적 상호작용의 지속* 자체를 끊는 장치입니다. 외부 입력에 노출된 에이전트의 종료 조건을 설계하던 팀이라면, 자체 휴리스틱 위에 이 도구의 동작을 겹쳐 확인해 두세요. [GitHub v2.1.214](https://github.com/anthropics/claude-code/releases/tag/v2.1.214)

## 개발자 워크플로우 팁

### `dir/**` 권한 규칙의 의미가 바뀌었다 — allow 규칙과 hook 조건을 재점검하라 (v2.1.214)

v2.1.214의 권한 수정 중 **기존 설정의 동작이 실제로 바뀌는** 항목이 있습니다. `Edit(src/**)` 같은 **single-segment `dir/**` 허용 규칙이 지금까지 트리 어디에 있든 `src/` 하위면 자동 승인**해 왔는데 — 이제 **`<cwd>/src`만** 매칭합니다.

- **무엇이 새고 있었나**: `Edit(src/**)`를 *프로젝트 루트의 src*를 의도하고 썼더라도, 실제로는 `vendor/some-pkg/src/` 같은 **엉뚱한 중첩 디렉토리까지 자동 승인**되고 있었습니다. 의도보다 넓게 열려 있던 규칙이 의도대로 좁혀진 것입니다.
- **역방향 점검도 필요**: 반대로 *중첩 디렉토리까지 허용되는 동작에 기대던* 모노레포 구성이라면, 업그레이드 후 승인 프롬프트가 갑자기 늘어날 수 있습니다 — 필요한 경로를 명시적 규칙으로 추가하세요.
- **hook도 같은 변경**: **hook의 `if:` 조건에 쓰인 single-segment `dir/**`도 `<cwd>/dir`만** 매칭하도록 바뀌었습니다 — 가드레일 hook이 커버하던 범위가 조용히 달라질 수 있으니 함께 점검 대상입니다.

```json
// .claude/settings.json — 의도한 경로를 명시적으로
{
  "permissions": {
    "allow": [
      "Edit(src/**)",           // 이제 <cwd>/src 만 매칭
      "Edit(packages/*/src/**)" // 모노레포 하위 src는 명시적으로
    ]
  }
}
```

핵심은 7/15의 *권한 규칙 시작 경고*와 같은 자리입니다 — **권한 규칙은 쓴 대로가 아니라 매칭되는 대로 동작**하고, 그 매칭 규칙이 이번에 바뀌었습니다. `settings.json`에 `dir/**` 패턴이 있는 팀은 오늘 업그레이드하며 한 번 훑어 두세요. [GitHub v2.1.214](https://github.com/anthropics/claude-code/releases/tag/v2.1.214)

### 무인 파이프라인이라면 auto-update를 고정하라 — 'Auto-Continue 미스피처'의 교훈 (7/17)

Perl 커뮤니티의 베테랑 **Olaf Alders**가 7월 초의 **Auto-Continue 미스피처 사건을 해부한 글**을 올렸습니다(7/17). 사건 자체는 이렇습니다 — **v2.1.198(7/1)이 `AskUserQuestion` 도구에 60초 무응답 시 '최선의 판단으로' 자동 진행하는 동작을 릴리스 노트·문서·공개 커밋 없이** 실었고, 사용자들이 이상 동작으로 발견해 **384개 추천을 받은 이슈**가 서자, **이틀 만에 v2.1.200에서 opt-in으로 회귀**했습니다.

- **글의 논점**: 기능의 좋고 나쁨이 아니라 **배포 방식**입니다 — 분석 계측까지 붙어 있었으니 *우연한 기본값이 아니라 측정 장비가 달린 의도된 기능*이었고, 그것이 문서 없이 자동 업데이트로 흘러들었다는 것. 폐쇄 소스여도 배포된 바이너리는 분석 가능하다는 점이 이번에도 투명성의 사후 검증 수단이 됐습니다.
- **실무 행동 항목**: **무인·프로덕션 환경은 auto-update를 끄고 버전을 고정**하세요 — `DISABLE_AUTOUPDATER=1`을 설정하고, 업그레이드는 릴리스 노트를 읽고 명시적으로 올리는 절차로 바꾸는 것입니다. 안전 가정이 자기도 모르게 바뀌는 것을 막는 가장 싼 방법입니다.

```bash
# 무인 파이프라인 — 버전을 명시적으로 관리
export DISABLE_AUTOUPDATER=1
claude --version        # 현재 버전 기록
claude update           # 릴리스 노트 확인 후 명시적으로
```

핵심은 아래 보안/제한의 트래커 해명과 **정확히 같은 주제**입니다 — *조용히 배포된 동작 변경*이 이번 주 Claude Code 신뢰 논쟁의 공통 축이고, 사용자 쪽 방어선은 결국 **업데이트 시점을 자기가 쥐는 것**입니다. [Olaf Alders](https://www.olafalders.com/2026/07/17/claude-code-anatomy-of-a-misfeature/)

### v2.1.214 권한 검사 대수술 — 새는 자동 승인 경로를 fail-closed로 (7/18)

v2.1.214는 최근 릴리스들의 하드닝 연쇄에서 **가장 넓은 폭의 권한 수정**을 담았습니다. 표적은 *권한 검사기가 명령을 실제 셸과 다르게 해석해 자동 승인이 새던* 경로들입니다.

- **우회 경로 차단**: **Windows PowerShell 5.1 세션의 권한 검사 우회**, bash가 검사기와 다르게 파싱하는 **파일 디스크립터 리다이렉트 형태의 fail-closed 처리**, zsh 변수 서브스크립트를 무해한 텍스트로 오인하던 문제가 잡혔습니다.
- **자동 승인 축소**: **10,000자를 넘는 명령은 이제 항상 프롬프트**하고(검사기가 오판하기 쉬운 길이), 안전하지 않은 옵션·명령 치환을 실을 수 있던 **`help`·`man` 명령의 자동 승인이 제거**됐으며, **데몬 리다이렉트 플래그를 단 `docker` 명령**(Podman shim 포함)과 매직 파일을 지정하는 **`file -m`·`-f`**도 권한을 요구합니다.
- **프롬프트 무결성**: 원격 세션의 권한 프롬프트가 **로컬 확인 다이얼로그보다 먼저 진행될 수 있던** 문제도 수정됐습니다.

### browser-rs-mcp — 에이전트 여럿이 크롬 하나를 나눠 쓰는 초경량 MCP 브라우저 컨트롤러

멀티에이전트 브라우저 자동화의 **RAM 문제**를 정면으로 겨눈 MCP 서버가 공개됐습니다. 문제의식은 간단합니다 — *에이전트당 크롬 하나*가 기본값이면 **에이전트당 500MB~1GB의 RAM**이 나가고, 에이전트 수에 비례해 선형으로 불어납니다.

- **구조**: **크롬 인스턴스 하나를 여러 에이전트가 동시에 제어**하되, **에이전트별 탭 격리**로 서로를 밟지 않게 합니다. Rust 구현으로 **서버 바이너리가 6MB**, raw CDP(Chrome DevTools Protocol) 기반 **62개 도구**를 노출합니다.
- **실전 기능**: 실제 Chrome + 영속 사용자 프로필을 쓰는 **스텔스 구성**(자동화 탐지 회피), 접근성 트리, 네트워크·쿠키·스토리지 제어, PDF·파일 업로드·iframe·WebAuthn 지원, **stdio·HTTP·SSE** 인터페이스를 제공합니다.
- **어디에 쓰나**: Claude Code에서 서브에이전트 여럿에 웹 조사·E2E 검증을 흩뿌리는 구성이라면, 브라우저 자원을 공유 풀로 바꾸는 선택지입니다 — 7/17의 세션 단위 상한이 *호출 수*의 가드레일이었다면, 이쪽은 *메모리 풋프린트*의 가드레일입니다.

```bash
# Claude Code에 등록 (stdio)
claude mcp add browser-rs -- browser-rs-mcp --stdio
```
