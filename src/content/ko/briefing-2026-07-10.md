## 주요 신규 기능 & 실전 활용

### `/doctor`가 이제 비대한 CLAUDE.md '다이어트'를 제안한다 (v2.1.206)

v2.1.206은 `/doctor`에 **체크인된 CLAUDE.md 파일이 지나치게 크면 줄이라고 제안하는 점검 항목**을 더했습니다. 어제(7/9) 브리핑에서 다룬 관행 — *CLAUDE.md가 길어질수록 지시가 무시될 확률이 오르니 짧게 유지하라* — 이 이제 **도구 차원의 점검**으로 들어온 셈입니다.

```bash
# /doctor 를 실행하면 이제 점검 항목 하나가 더 붙습니다
/doctor
# → 비대한 CLAUDE.md 발견 시 트리밍을 제안
#   (반드시 매번 지켜야 하는 규칙은 문장 대신 hook으로 옮기는 게 정석)
```

`/doctor`가 7/8의 **풀 셋업 체크업**으로 확장된 데 이어, 이번엔 **컨텍스트 예산을 갉아먹는 비대한 메모리 파일까지** 잡아줍니다. 함께 `/cd`에도 `/add-dir`처럼 **디렉토리 경로 자동 제안**이 붙어, 작업 디렉토리 전환이 매끄러워졌습니다. 핵심은 — *성능 요건이던 CLAUDE.md 간결함이 이제 진단 가능한 항목*이 됐다는 점입니다. 메모리 파일을 오래 손 안 댔다면 오늘 `/doctor`를 한 번 돌려 보세요. [GitHub v2.1.206](https://github.com/anthropics/claude-code/releases/tag/v2.1.206)

### PR 루프·게이트웨이·워크트리 — 세션 연속성과 안전을 다진 항목들 (v2.1.206)

같은 릴리스의 나머지 신규 항목은 *무인·병렬 운영의 연속성과 안전*에 몰려 있습니다.

- **`/commit-push-pr` 자동 push**: 이제 리포에 **설정된 push 원격(remote)으로의 `git push`를 자동 허용**합니다 — commit→push→PR 루프가 권한 확인에서 멈추지 않고 한 번에 닫힙니다. 백그라운드 에이전트가 draft PR을 여는 흐름(v2.1.198)과 맞물려, 무인 PR 파이프라인의 마지막 수동 개입이 하나 줄었습니다.
- **Gateway 공개 엔드포인트 로그인**: `/login`이 이제 **Anthropic이 운영하는 공개 게이트웨이 엔드포인트**를 지원해, 게이트웨이 경유 인증의 진입 경로가 넓어졌습니다.
- **`EnterWorktree` 진입 가드**: 프로젝트의 `.claude/worktrees/` **밖에 있는 git worktree로 진입하기 전 확인을 묻습니다** — v2.1.205의 Windows worktree 데이터 손실 수정에 이어, 워크트리 오작동을 막는 안전장치가 한 겹 더 깔렸습니다.
- **백그라운드 자동 업그레이드**: 백그라운드 에이전트가 Claude Code 업데이트 **직후 백그라운드에서 새 버전으로 자동 업그레이드**합니다 — 장시간 도는 무인 세션이 구버전에 묶이지 않습니다.

### 커스텀 게이트웨이·MCP·무인 세션의 조용한 실패 지점을 재점검하라 (v2.1.206)

v2.1.206은 **커스텀 게이트웨이·MCP·백그라운드 자동화**를 굴리는 팀에 직접 닿는 수정을 담았습니다. 무인·프록시 환경이라면 그대로 점검 목록입니다.

- **`CLAUDE_CODE_EXTRA_BODY` 유실 수정**: `claude agents`·`--bg` 백그라운드 워커가 이 값을 **조용히 무시**하던 문제 해소 — 커스텀 요청 바디를 주입하는 파이프라인이라면 원인 불명의 동작 차이가 여기서 왔을 수 있습니다.
- **OAuth MCP 재인증 수정**: OAuth MCP 서버가 **토큰 갱신에 한 번만 실패해도 수동 재인증을 요구**하던 문제 해소.
- **MCP `request_timeout_ms` 무시 수정**: `--mcp-config`·`.mcp.json`으로 설정한 서버가 **서버별 `request_timeout_ms`를 무시**하던 문제 해소.
- **만료 로그인 오류 개선**: 로그인이 만료되면 **모든 모델이 오해를 부르는 오류로 실패**하던 것을, 이제 `/login`을 실행하라고 안내합니다.

핵심은 **"커스텀 인증·게이트웨이 경로의 조용한 실패가 또 한 겹 걷혔다"**는 점입니다 — 7/8의 `ANTHROPIC_BASE_URL` 유실 수정과 정확히 같은 자리입니다. 원인 모를 401·요청 바디 누락·MCP 타임아웃을 겪었다면 v2.1.206으로 올린 뒤 재현되는지 확인하세요. [GitHub v2.1.206](https://github.com/anthropics/claude-code/releases/tag/v2.1.206)

## 보안/제한 이슈

### 중국 백도어 경고 후속 — 추적 코드는 이미 v2.1.198(7/1)에 제거, Anthropic의 반박 (7/9)

어제(7/9) 다룬 중국 NVDB의 Claude Code 백도어 경고에 **후속 사실 두 가지**가 붙었습니다.

- **추적 기능은 경고 전에 이미 제거됐습니다**: 여러 보도(TechNadu·The Register)에 따르면, 문제의 위치 추적(스테가노그래피) 코드는 **v2.1.198(7/1 릴리스)에서 이미 제거**됐고, 영향 버전은 **2.1.91~2.1.196**(4~6월)으로 전해집니다. 즉 NVDB 권고(7/8) 시점엔 **최신 버전엔 해당 코드가 없던 셈**입니다 — 어제 엔지니어(Thariq Shihipar)가 말한 *다음 릴리스에서 롤백* 예고는 사실상 그 이전에 완료돼 있었고, 그는 *"이후 더 강력한 완화책을 적용했고, 한동안 이걸 내리려 해 왔다"*고 덧붙였습니다.
- **Anthropic의 반박**: SCMP 보도에 따르면 Anthropic은 이 사안을 *distillation·무단 리셀러를 막기 위한 올해 초 실험*으로 규정하면서, **중국에서는 Claude·Claude Code 사용 자체가 허용되지 않으므로** 제거를 권고받은 사용자들은 애초 인가되지 않은 사용자라는 취지로 반박했습니다.

핵심은 **"기술적 사실(이미 제거)과 정책적 논쟁(동의 없는 텔레메트리 대 무단 사용)을 분리해 읽으라"**는 점입니다 — 최신 버전(v2.1.206 등)을 쓰는 사용자에겐 해당 코드가 없지만, *남용 방지를 위한 동의 없는 위치·신원 텔레메트리가 정당한가*라는 판단은 여전히 정부와 기업이 갈리는 지점입니다. 어제와 마찬가지로 어느 쪽 손도 들지 않고, 위 사실로 각자 저울질하는 것이 맞습니다. [TechNadu](https://www.technadu.com/china-flags-alleged-backdoor-in-anthropics-claude-code-citing-data-transmission-risks/630586/) · [SCMP](https://www.scmp.com/news/china/article/3359901/anthropic-hits-back-after-china-warns-claude-code-backdoor-risks)
