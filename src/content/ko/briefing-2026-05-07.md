## 주요 신규 기능 & 실전 활용

### Code with Claude SF 핵심 발표 — Managed Agents에 Dreaming·Outcomes·멀티에이전트 추가 (5/6)

어제 개최된 Code with Claude SF에서 **새 모델 발표는 없었지만**, Claude Managed Agents에 세 가지 핵심 기능이 추가되었습니다.

**1. Dreaming (Research Preview)**: 에이전트가 이전 세션을 스스로 복기하여 패턴을 추출하고 메모리를 자동 개선합니다. 밤 사이에 플레이북을 생성하는 방식으로, Harvey는 테스트에서 에이전트 완료율이 **약 6배 향상**되었다고 보고했습니다. 자동 업데이트와 수동 검토 중 선택 가능합니다.

**2. Outcomes (Public Beta)**: 성공 기준을 루브릭으로 정의하면, 별도의 grader가 에이전트 출력을 독립된 컨텍스트 윈도우에서 평가합니다. 에이전트의 추론에 영향받지 않으므로 객관적 평가가 가능하며, 표준 프롬프팅 대비 **최대 10점 성공률 향상**을 달성했습니다.

**3. Multi-agent Orchestration (Public Beta)**: 리드 에이전트가 복잡한 작업을 분할하고 전문 에이전트에 위임하여 **병렬 실행**합니다. 공유 파일시스템과 영구 이벤트 히스토리를 통해 중간 체크가 가능합니다. Netflix는 수백 개 소스의 빌드 로그를 분석하는 에이전트에 이 기능을 활용하고 있습니다.

[Anthropic 블로그](https://claude.com/blog/new-in-claude-managed-agents) | [Simon Willison 라이브블로그](https://simonwillison.net/2026/May/6/code-w-claude-2026/)

### SpaceX Colossus 제휴 — Claude Code 사용량 한도 2배로 확대 (5/6)

Anthropic이 SpaceX의 **Colossus 1 데이터센터 전체 용량**에 대한 접근 계약을 체결했습니다. 300MW 이상의 인프라와 **22만 개 이상의 NVIDIA GPU**(H100, H200, GB200)가 한 달 내 가용됩니다.

**즉각적인 영향**:

- Pro/Max 계정의 피크타임 스로틀링 제거
- Claude Opus API 레이트 리밋 대폭 증가

향후 Anthropic과 SpaceX는 **수 기가와트 규모의 궤도 AI 컴퓨팅** 개발도 계획하고 있습니다.

### /ultrareview — 클라우드 멀티에이전트 코드 리뷰 (5/6)

Claude Code에 `/ultrareview` 명령이 추가되었습니다. 실행하면 **원격 샌드박스에서 리뷰어 에이전트 팀**이 브랜치나 PR을 분석합니다. 5~10분 소요되며 백그라운드 태스크로 실행되므로 작업을 계속할 수 있습니다.

Anthropic 내부 전 팀이 이미 사용 중이며, GitHub PR에 직접 코멘트를 남기는 방식으로 통합됩니다.

```bash
# 현재 브랜치 코드 리뷰
/ultrareview

# PR 대상 리뷰
/ultrareview --pr 123
```

[Claude 블로그](https://claude.com/blog/code-review) | [Claude Code Docs](https://code.claude.com/docs/en/ultrareview)

## 개발자 워크플로우 팁

### v2.1.129 — `--plugin-url`로 원격 플러그인 설치

v2.1.128의 ZIP 플러그인 지원에 이어, v2.1.129에서 **URL에서 직접 플러그인 ZIP 아카이브를 다운로드**하는 `--plugin-url` 플래그가 추가되었습니다. 팀 간 플러그인 공유가 한 단계 더 간편해졌습니다.

```bash
# 원격 URL에서 플러그인 로드
claude --plugin-url https://example.com/my-plugin.zip
```

또한 `CLAUDE_CODE_PACKAGE_MANAGER_AUTO_UPDATE` 환경변수로 **백그라운드 자동 업데이트**를 설정할 수 있게 되었습니다.

### v2.1.132 — 메모리 누수 수정과 fullscreen 비활성화

v2.1.132에서 MCP 서버가 비표준 출력을 보낼 때 발생하던 **10GB 이상의 메모리 증가 문제**가 수정되었습니다. 비 MCP 프로토콜 출력이 무한 버퍼링되던 것이 원인이었습니다.

새로운 환경변수 `CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1`을 설정하면 fullscreen 렌더러를 비활성화할 수 있어, 터미널 호환성 문제가 있는 환경에서 유용합니다.

```bash
# fullscreen 모드 비활성화
export CLAUDE_CODE_DISABLE_ALTERNATE_SCREEN=1

# 세션 ID를 서브프로세스에 전달 (자동화/로깅용)
# CLAUDE_CODE_SESSION_ID가 Bash 도구 서브프로세스에 자동 설정됨
```

## 생태계 & 플러그인

### Cloudflare + Stripe — 에이전트가 자율적으로 계정 생성·도메인 구매·배포 (5/7)

Cloudflare와 Stripe가 공동 설계한 새로운 프로토콜을 통해, AI 코딩 에이전트가 **자율적으로 Cloudflare 계정을 생성하고, 유료 구독을 시작하고, 도메인을 등록하고, 프로덕션에 배포**할 수 있게 되었습니다.

핵심 구조는 Discovery → Authorization → Payment의 3단계로, Stripe가 ID 제공자 역할을 하고 Cloudflare가 계정을 자동 프로비저닝합니다. 결제 정보는 에이전트에 노출되지 않으며, **월 $100 USD 기본 한도**가 설정됩니다.

```bash
# Stripe CLI 기반 프로젝트 초기화
stripe projects init
# 에이전트에게 "새 도메인에 빌드하고 배포해줘" 요청
```

[GeekNews](https://news.hada.io/topic?id=29249) | [Cloudflare 블로그](https://blog.cloudflare.com/agents-stripe-projects/)

## 알아두면 좋은 소소한 변경사항

- **`skillOverrides` 설정 지원**: v2.1.129에서 스킬 동작을 `off`, `user-invocable-only`, `name-only`로 제어 가능
- **Ctrl+R 히스토리 피커 개선**: 기본으로 모든 프롬프트를 표시하고, Ctrl+S로 좁히기 가능
- **게이트웨이 모델 디스커버리 opt-in 전환**: `CLAUDE_CODE_ENABLE_GATEWAY_MODEL_DISCOVERY=1`을 설정해야 활성화됨 (이전에는 기본 활성)
- **`/context`의 ASCII 시각화 그리드 제거**: 약 1.6k 토큰을 낭비하던 디버그 출력 수정
- **프롬프트 캐시 TTL 수정**: 1시간 TTL이 5분으로 다운그레이드되던 버그 수정 — 캐시 히트율에 직접 영향
- **외부 SIGINT 시 graceful shutdown**: 이전에는 터미널 모드가 복원되지 않던 문제 수정
