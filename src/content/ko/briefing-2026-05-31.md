## 주요 신규 기능 & 실전 활용

### v2.1.158 — Auto mode가 Bedrock·Vertex·Foundry로 확장 (5/30)

v2.1.158은 어제 출시된 v2.1.157(`.claude/skills` 자동 로딩)에 이은 작고 집중된 릴리스로, **Auto mode의 사용 범위를 클라우드 모델 엔드포인트로 확장**했습니다.

이제 **Amazon Bedrock, Google Vertex AI, Microsoft Foundry**에서 **Opus 4.7과 Opus 4.8** 모델로 Auto mode를 사용할 수 있습니다. 기존에는 Anthropic First-party API에 한정되던 무인 실행(권한 자동 승인) 모드가, 자체 클라우드 계정으로 Claude를 호스팅하는 엔터프라이즈 환경으로 열린 것입니다.

활성화는 환경변수 한 줄로 opt-in 합니다.

```bash
# Bedrock / Vertex / Foundry에서 Auto mode 활성화
export CLAUDE_CODE_ENABLE_AUTO_MODE=1
claude
```

규제·보안 요건 때문에 자체 클라우드(Bedrock/Vertex/Foundry)로만 Claude를 운영하던 팀도, 이제 동일한 환경에서 야간 배치·CI 무인 실행 워크플로우를 구성할 수 있습니다.

[GitHub v2.1.158](https://github.com/anthropics/claude-code/releases/tag/v2.1.158)

## 개발자 워크플로우 팁

### 클라우드 엔드포인트에서 무인 실행 — Auto mode + 안전장치 결합

v2.1.158로 Bedrock/Vertex/Foundry 사용 팀도 Auto mode 무인 실행이 가능해졌지만, 권한이 자동 승인되는 만큼 **가드레일을 함께 거는 것**이 핵심입니다.

**실전 패턴**:
1. `CLAUDE_CODE_ENABLE_AUTO_MODE=1`로 Auto mode를 켜되, `--allowedTools`로 허용 도구를 최소한으로 제한합니다.
2. 샌드박스(`/status`에서 확인 가능)와 결합하여 파일·네트워크 접근 범위를 좁힙니다.
3. 대규모 단일 작업 대신 작은 단위로 분할하여, 무인 실행 중 에러 루프로 토큰을 소진하는 것을 방지합니다.

자체 클라우드 계정으로 모델을 호스팅하는 환경은 First-party API와 레이트 리밋·과금 체계가 다르므로, 무인 실행 전 해당 클라우드 콘솔의 쿼터를 먼저 확인하는 것이 안전합니다.

[GitHub v2.1.158](https://github.com/anthropics/claude-code/releases/tag/v2.1.158)

### 컨텍스트 보존 — CLAUDE.md compaction 지시 + 서브에이전트 분리

장시간 세션에서 가장 흔한 품질 저하 원인은 **compaction(자동 요약) 과정에서 중요한 컨텍스트가 사라지는 것**입니다. 이를 막는 두 가지 실용 기법이 있습니다.

**1. CLAUDE.md에 compaction 보존 규칙 명시**: compaction 시 보존할 항목을 명령으로 적어두면, 요약 후에도 핵심 정보가 살아남습니다.

```markdown
<!-- CLAUDE.md -->
When compacting, always preserve the full list of modified files
and any test commands used in this session.
```

**2. 탐색은 서브에이전트로 위임**: 코드베이스를 탐색할 때 Claude는 많은 파일을 읽어 메인 컨텍스트를 소비합니다. 서브에이전트는 **별도 컨텍스트 윈도우**에서 실행되어 요약만 반환하므로, 메인 세션의 컨텍스트를 아껴 더 오래 일관성을 유지할 수 있습니다.

[Best practices for Claude Code](https://code.claude.com/docs/en/best-practices)

## 생태계 & 플러그인

### 공식 Resend 플러그인 출시 — MCP 서버 + 5개 스킬 번들 (5/26)

이메일 API 서비스 Resend가 **공식 Claude Code 플러그인**을 출시했습니다. MCP 서버와 5개 스킬을 하나의 설치로 묶어, 이메일 발송·템플릿 관리·인바운드 처리를 Claude Code 안에서 바로 다룰 수 있습니다.

**번들 구성**: `resend`(SDK/API), `react-email`(템플릿 작성), `email-best-practices`(전달성), `agent-email-inbox`(인바운드 처리), `resend-cli`(셸/CI 워크플로우). 작업 내용에 따라 관련 스킬이 자동 활성화됩니다.

```bash
# 공식 마켓플레이스에서 설치
claude plugin install resend@claude-plugins-official
```

설치 후 `~/.claude/config.json`의 `env`에 `RESEND_API_KEY`를 추가합니다. Notion·Stripe·Cloudflare·Slack·Vercel에 이어 **퍼스트파티(공식) MCP 플러그인**이 늘어나는 흐름의 한 사례입니다.

[Resend 체인지로그](https://resend.com/changelog/resend-claude-code-plugin)
