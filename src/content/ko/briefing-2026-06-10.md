### Claude Fable 5 — Anthropic 첫 일반 공개 'Mythos급' 모델 (6/9, v2.1.170로 접근)

6월 9일 Anthropic이 **Claude Fable 5**를 공개했습니다. Fable 5는 안전장치를 적용해 **일반 사용이 가능하도록 만든 첫 Mythos-class 모델**로, "우리가 일반에 내놓은 어떤 모델보다 뛰어나며 거의 모든 테스트 벤치마크에서 SOTA"라고 Anthropic은 밝혔습니다. 특히 **여러 날에 걸친 복잡한 작업을 끝까지 끌고 가는 자율성**에 강점이 있습니다.

개발자 관점에서 인상적인 증거들:

- **소프트웨어 엔지니어링**: Stripe가 초기 테스트에서 Fable 5에 **5,000만 줄 규모의 Ruby 코드베이스 마이그레이션**을 맡겨 **하루 만에** 끝냈습니다 — 수작업이면 팀이 약 **두 달** 걸릴 작업입니다. 가장 어려운 코딩 벤치마크에서 Opus 4.8을 **2배 이상** 앞선다고 합니다.
- **비전**: 스크린샷만으로 웹앱 소스를 재구성하고, 보조 도구 없이 **Pokémon FireRed를 클리어**하는 등 SOTA를 기록했습니다.
- **장기 컨텍스트 + 메모리**: 파일 기반 메모리를 줄 때 영속 메모리가 Fable 5의 성능을 **Opus 4.8보다 3배 더** 끌어올렸습니다 — 메모리·서브에이전트를 엮는 에이전틱 워크플로우에서 체감차가 크다는 신호입니다.

(제한판 **Mythos 5**는 신약 설계·분자생물학·유전체학에서 강한 성과를 보고했지만, 이는 일반 공개 대상이 아닌 별도 영역입니다 — 자세한 내용은 아래 보안/제한 이슈를 보세요.)

가격은 **입력 100만 토큰당 $10, 출력 100만 토큰당 $50**입니다(Claude Mythos Preview의 절반 이하, TechCrunch 기준 Opus 4.8의 약 2배). 접근 방법:


구독에서는 **6/9~6/22 Pro·Max·Team·시트형 Enterprise에 무료로 포함**되고, **6/23부터는 usage credit이 필요**합니다(표준 구독으로의 전면 복원은 용량 확보 시). API는 즉시 `claude-fable-5`로 표준 요금에 쓸 수 있습니다. (정확한 모델 ID·셀렉션 방법은 공식 문서를 따르세요.)

## 개발자 워크플로우 팁

### 세션이 끝난 뒤 정리·산출물 수집 — self-hosted runner `post-session` 훅 (v2.1.169)

v2.1.169에 self-hosted runner용 **`post-session` 라이프사이클 훅**이 추가됐습니다. 이름 그대로 **세션이 종료된 뒤, 워크스페이스가 삭제되기 직전에** 실행됩니다. CI에서 에이전트 세션을 워크스페이스째 띄웠다가 버리는 구조라면, 워크스페이스가 사라지기 전에 **로그·아티팩트·커버리지 리포트를 외부로 빼내거나, 임시 자원을 정리하는** 마지막 훅으로 쓰기 좋습니다.

`Stop` 훅이 "턴을 끝내기 전" 게이트라면, `post-session`은 "세션 전체가 끝난 뒤"의 정리 지점입니다 — 둘은 발동 시점이 다릅니다. (정확한 페이로드·설정 위치는 공식 self-hosted runner 문서를 따르세요.)

[GitHub v2.1.169](https://github.com/anthropics/claude-code/releases/tag/v2.1.169)

### 서브에이전트는 부모의 스킬을 물려받지 않는다 — 명시적으로 preload할 것

흔한 함정 하나: **서브에이전트는 부모 세션의 스킬을 자동으로 물려받지 않습니다.** 서브에이전트가 특정 스킬의 지식·절차에 의존해야 한다면, 그 스킬을 **명시적으로 preload**해야 합니다 — 스킬은 "쓸 수 있게 노출"되는 게 아니라 **시작 시 본문 전체가 주입**되어야 동작합니다. "메인에서 되던 게 서브에이전트에선 안 된다"는 대부분 여기서 비롯됩니다.

규칙으로 외우면 됩니다 — 같은 지시를 **인라인으로** 쓰고 싶으면 Skill, **컨텍스트 격리·병렬**이 필요하면 Subagent, 다만 Subagent에 스킬이 필요하면 **반드시 명시 전달**.

[Claude Code Docs — Best practices](https://code.claude.com/docs/en/best-practices)

### "끝났다" 선언 전에, 새 컨텍스트의 서브에이전트로 diff를 리뷰시키기

에이전트가 오래 무인으로 일할수록, **작업을 끝으로 세기 전 독립적인 점검**이 중요해집니다. 효과적인 패턴은 **새(fresh) 컨텍스트의 서브에이전트에게 diff만 보여주고** "요구사항 대비 빠진 게 뭔지" 보고시키는 것입니다. 메인 세션은 자기가 짠 코드에 이미 "익숙해져" 결함을 놓치기 쉬운데, 변경분과 기준만 받은 리뷰어는 **선입견 없이** 갭을 잡아냅니다.

앞서 다룬 `Stop` 훅 검증 게이트가 "테스트가 통과할 때까지"의 자동 가드라면, 이건 "사람이 보기 전 한 번 더 거르는" **독립 리뷰어** 단계입니다 — 둘을 같이 쓰면 무인 작업의 완성도가 올라갑니다.

[Claude Code Docs — Best practices](https://code.claude.com/docs/en/best-practices)

## 보안/제한 이슈

### Fable 5는 '안전장치를 내장한 채' 출시됐다 — 분류기 우회·Mythos 5·Project Glasswing (6/9)

Fable 5의 출시에서 개발자가 눈여겨볼 지점은 능력만이 아니라 **그 능력에 두른 제약**입니다. Fable 5에는 **3개의 분류기(classifier)** 가 들어가, 고위험 요청을 자동으로 **Claude Opus 4.8의 응답으로 우회**시킵니다.

- **발동 빈도**: 전체 세션의 **5% 미만**에서만 발동하고, **95% 이상**은 Fable 5 자체 응답으로 끝납니다.
- **대상 영역**: ① **사이버보안**(공격·익스플로잇 — 단일턴 유해 요청 준수 0건, 30개 공개 jailbreak에도), ② **생물·화학**(이중용도, 특히 AAV 유전자치료 설계), ③ **distillation**(권위주의 국가 경쟁사의 모델 추출 방지).
- **견고성**: 외부 버그바운티 **1,000시간 이상** 테스트에서 universal jailbreak이 나오지 않았습니다(UK AISI가 초기 창에서 일부 진전).
- **데이터 보존**: Mythos급 모델 트래픽에 **30일 보존** 정책이 새로 적용되며, 데이터는 안전 목적에만 쓰입니다.

안전장치를 **일부 해제한 동일 모델**이 **Mythos 5**인데, 이는 검증된 **사이버디펜더·핵심 인프라 제공자에게만** 제공됩니다 — 美 정부와 협력하는 **Project Glasswing**(4월의 Mythos Preview 업그레이드)을 통해서입니다. "가장 강력한 모델일수록 누가·무엇을 쓸 수 있는지를 더 좁게 통제한다"는 방향이 분명해진 출시입니다.

[Anthropic — Claude Fable 5 and Claude Mythos 5](https://www.anthropic.com/news/claude-fable-5-mythos-5)
