## 주요 신규 기능 & 실전 활용

### Claude Code Artifacts — 세션 작업을 '살아 있는 공유 웹페이지'로 만든다 (6/18, 베타)

Anthropic이 **Claude Code Artifacts**를 출시했습니다(6/18). 핵심은 Claude Code의 작업 결과를 **스스로 갱신되는 공유 가능한 시각 페이지(artifact)**로 굳히는 것입니다 — **PR 워크스루, 시스템 설명서, 대시보드, 릴리스 체크리스트, 인시던트 페이지, 라이선스 감사, 데이터 흐름도, 보안 점검 결과** 같은 산출물을 코딩 세션 안에서 바로 만들 수 있습니다.

- **세션 전체 컨텍스트로 조립**: artifact는 **코드베이스·연결된 connector·대화 자체**를 모두 동원해 만들어집니다. 예컨대 인시던트 페이지 하나가 **실패한 테스트와 그 뒤의 함수(코드에서)**, **모니터링 도구의 에러 스파이크(connector에서)**, **방금 돌린 세션의 근본원인 추론**을 한 페이지에 모읍니다.
- **실시간 갱신**: Claude Code가 artifact를 고치면 **열려 있는 페이지가 그 자리에서 새로고침**되고, 팀원은 **발행되는 즉시** 그 변경을 봅니다.
- **버전 관리**: 발행할 때마다 **같은 링크에 새 버전**이 쌓이고, **버전 히스토리로 언제든 복원**할 수 있으며, **갤러리**에서 만든 artifact를 모아 관리합니다.

```bash
# Claude Code 세션 안에서 바로 요청한다
"이 PR을 설명하는 artifact를 만들어줘 — diff, 판단 근거, 내가 테스트한 것"
"서드파티 의존성과 각 라이선스를 전부 나열한 artifact로 만들어줘"
"Terraform에서 클라우드 리소스를 뽑아 서비스별로 묶은 artifact를 만들어줘"
```

실무 의미가 분명합니다 — **6/20에 다룬 서드파티 `htmlbook`**(AI가 만든 HTML·markdown을 파일 대신 링크 하나로 공유)이 메우던 자리를, 이번엔 **Anthropic이 1군 기능으로 Claude Code에 내재화**한 셈입니다. 다만 차이가 있습니다: artifact는 **기본 비공개**이고 **org의 인증된 멤버만 열람**하며 **공개로 전환할 수 없습니다** — htmlbook 같은 공개 링크 공유와는 성격이 다른, **사내용 공유**입니다. 그리고 현재는 **Claude Team·Enterprise org 대상 베타**로, **Claude Code CLI와 데스크톱 앱**에서 만들고 어떤 브라우저로든 봅니다 — 개인 Pro/Max 사용자는 아직 쓸 수 없다는 점을 전제로 하세요. [Claude 블로그 — Artifacts in Claude Code](https://claude.com/blog/artifacts-in-claude-code) · [VentureBeat](https://venturebeat.com/data/anthropics-claude-code-artifacts-update-brings-live-shared-dashboards-and-interactive-workspaces-to-enterprises)

## 개발자 워크플로우 팁

### Artifacts는 '증거 조립' 레이어로 쓰되, '결론'은 사람이 직접 써라

위 Artifacts를 가장 잘 쓰는 방법은 **역할을 정확히 나누는 것**입니다. 6/21에 다룬 surfingcomplexity의 원칙 — *"글을 쓰는 행위 자체가 검증 단계"라서 LLM에 보고서를 통째로 맡기면 검증이 통째로 빠진다"* — 가 여기에 그대로 맞물립니다. Artifacts는 그 원칙을 깨는 도구가 아니라, **오히려 정확히 그 분업을 도구로 구현**합니다.

artifact가 잘하는 일은 **기계적이고 검증 가능한 '수집·조립'**입니다 — 실패한 테스트, 그 함수, 모니터링의 에러 스파이크, 관련 diff를 **한 페이지에 자동으로 끌어모으는** 것이죠. 반대로 **"무엇이 왜 일어났는가"의 종합과 결론은 여전히 사람의 몫**입니다. 그러니 인시던트·PR 워크스루 artifact를 만들 때, **에이전트에는 증거를 모아 페이지에 배치하게 하고, 결론 문단은 직접 쓰거나 최소한 직접 검수**하세요. 이렇게 하면 "그럴듯하지만 틀린 서사"를 피하면서도, 증거가 **버전 히스토리와 함께 한곳에 살아 있는** 이점을 얻습니다 — artifact의 자동 갱신은 *증거*에는 축복이지만, *결론*에는 사람의 잠금이 필요합니다. [Claude 블로그 — Artifacts in Claude Code](https://claude.com/blog/artifacts-in-claude-code)

### CLAUDE.md는 '권고'일 뿐 — 반드시 지켜야 할 규칙은 hook으로 강제하라

CLAUDE.md에 규칙을 아무리 많이 적어도 그대로 지켜지지 않는 이유가 있습니다. CLAUDE.md 지시는 **모델이 따르기로 *선택*해야 하는 soft 규칙**이라, 현장 보고로는 **좋은 날에도 준수율이 대략 60~70%** 수준입니다 — 게다가 **지시가 길어질수록 준수율이 균일하게 떨어집니다**(중요한 규칙이 잡음에 묻힙니다). `git push origin main` 금지, 프로덕션 데이터 삭제 금지처럼 **단 한 번의 예외도 안 되는** 규칙에는 이 신뢰도가 부족합니다.

해법은 **결정론적 hook**입니다 — 프롬프트가 아니라 **코드로 강제**하기 때문입니다. 반드시 매번 일어나야(또는 막혀야) 하는 동작은 CLAUDE.md의 문장이 아니라 **PreToolUse hook**으로 두세요(PreToolUse hook은 도구 입력을 **stdin에 JSON**으로 받고, **exit 2로 차단**합니다).

```json
// .claude/settings.json — 'main에 직접 push'를 런타임에서 차단
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Bash",
        "hooks": [
          {
            "type": "command",
            "command": "jq -e '.tool_input.command | test(\"git push.*(main|master)\")' >/dev/null && { echo 'blocked: push to main' >&2; exit 2; } || exit 0"
          }
        ]
      }
    ]
  }
}
```

핵심은 **계층을 나누는 것**입니다 — *권고*(스타일·컨벤션·선호)는 CLAUDE.md에 짧게 두고, *불변식*(보안·데이터 무결성·배포 가드)은 hook으로 못 박으세요. 이는 6/15에 다룬 "사후 hook은 차단 말고 비차단 피드백만"과 짝을 이룹니다 — *결과를 알려주는* 사후 hook은 비차단으로, *돌이키기 어려운 동작을 막는* 사전 hook은 차단형으로 쓰는 것입니다. 커뮤니티에서도 "CRITICAL로 표시한 CLAUDE.md 규칙은 프롬프트가 아니라 hook을 트리거해야 한다"는 요청이 올라와 있습니다. [Claude 블로그 — Steering Claude Code](https://claude.com/blog/steering-claude-code-skills-hooks-rules-subagents-and-more) · [GitHub 이슈 #32163](https://github.com/anthropics/claude-code/issues/32163)

## 보안/제한 이슈

### Fable 5, 6/23부터 구독 플랜에서 빠진다 — 이후엔 'usage credit'으로 (6/23 발효)

개발자가 **당장 챙겨야 할 시한**이 있습니다. **Fable 5**는 **6/9~6/22** 동안 **Pro·Max·Team·시트 기반 Enterprise 플랜에 무료로 포함**돼 있었는데, **6/23부터 그 포함이 종료**됩니다.

- **무료 기간에도 2배로 태웠다**: 포함 기간에도 Fable 5는 플랜 한도에서 **Opus의 약 2배**로 계산됐습니다 — 같은 작업이 Opus 4.8 대비 **한도를 약 2배 빨리** 소진했다는 뜻입니다.
- **6/23 이후**: Fable 5는 모델 피커에는 **그대로 남지만**, 사용하면 번들된 플랜 한도가 아니라 **usage credit에서 차감**됩니다.
- **API 직접 가격**: **입력 $10 / 출력 $50(100만 토큰당)** — **Opus 4.8의 2배**로, 현재 일반 출시된 프런티어 모델 중 **가장 비쌉니다.**
- **영구 가격대는 아니다**: Anthropic은 이것이 **가격 정책이 아니라 용량(capacity) 결정**이라고 못 박았습니다 — 용량이 확보되면 **Fable 5를 표준 플랜의 일부로 복원**하고, 변경은 미리 알리겠다는 입장입니다.

다만 **수출통제 맥락**을 함께 봐야 합니다 — 6/12~6/21에 다룬 대로, **외국 국적 개발자(한국·일본 등)는 수출통제로 Fable 5에 애초에 접근할 수 없습니다.** 따라서 이번 과금 변경은 **접근이 가능한(주로 미국) 구독자**에게 실질적입니다. 우리 쪽 실무 대응은 그대로입니다 — Fable 5를 자동화에 끼워 두지 말고 `fallbackModel`로 **Opus 4.8 같은 대체 경로**를 깔아 두세요. [Hacker News](https://news.ycombinator.com/item?id=48463982) · [Developers Digest](https://www.developersdigest.tech/blog/claude-fable-5-june-22-deadline)

### 이번 주 두 번째 안정성 잡음 — 6/20 'Opus 4.8 errors' 인시던트

6/18 전 플랫폼 장애(6/20 브리핑에서 다룸)에 이어, **6/20에 또 한 번** 짧은 인시던트가 있었습니다. status.claude.com 기록상 **6/20 17:23 UTC에 조사 시작 → 18:02 UTC 수정 적용·모니터링**으로, 제목은 **"Opus 4.8 errors"**였습니다(약 40분, 복구 완료). 6/21~6/22에도 Claude Code 쪽에서 산발적인 오류 신고가 있었습니다.

개별 인시던트는 짧고 복구됐지만, **일주일 새 둘 이상**이라는 점이 신호입니다 — 터미널의 Claude Code도 같은 백엔드에 묶여 있어, 짧은 장애에도 무인 작업이 통째로 깨질 수 있습니다. CI·cron으로 Claude Code를 돌린다면 **재시도·백오프와 실패 알림**을 파이프라인에 깔아 두는 것이 변함없는 실무 대응입니다. [Claude Status — Incident History](https://status.claude.com/history)

## 알아두면 좋은 소소한 변경사항

CLI 신규 릴리스가 없는 날이라, 오늘 다룬 Artifacts에서 **놓치기 쉬운 운영상 디테일**을 모았습니다.

- **Claude Code CLI는 v2.1.185(6/20) 이후 신규 릴리스 없음** — 다음 릴리스가 올라오면 정리하겠습니다 (6/22 기준)
- **Artifacts는 기본 비공개 + 공개 불가**: 모든 artifact는 작성자에게 **private**이 기본이고, **org의 인증된 멤버만** 볼 수 있으며 **공개(public)로 전환할 수 없습니다** — 사내 공유 전용입니다 (Artifacts)
- **Artifacts는 Team·Enterprise 베타**: **Claude Code CLI와 데스크톱 앱**에서 만들고, 페이지는 **어떤 브라우저로든** 열립니다 — 개인 Pro/Max는 아직 대상 아님 (Artifacts)
- **같은 링크 = 새 버전**: 발행할 때마다 같은 URL에 버전이 쌓이고 **버전 히스토리로 복원** 가능, **갤러리**에서 전체 관리 (Artifacts)
