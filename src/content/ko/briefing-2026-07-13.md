### Microsoft 365 커넥터에 쓰기 도구 — Claude가 이메일을 보내고 SharePoint 파일을 만든다 (7/7)

- **Outlook**: 이메일 **초안 작성과 발송**, **캘린더 이벤트 생성·관리**.
- **OneDrive·SharePoint**: **파일 생성과 업데이트**.
- **전제 조건**: **Microsoft Entra의 관리자 동의(admin consent)**가 필요합니다 — 개인이 켜는 게 아니라 조직 관리자가 허용해야 활성화됩니다.

```text
# 전제: Microsoft Entra에서 조직 관리자가 admin consent 승인
Settings → Connectors → Microsoft 365
→ 검색·요약(읽기)에 더해 이메일 발송·캘린더·파일 생성(쓰기)까지
```

핵심은 **"커넥터가 읽기 채널에서 출력 채널로 승격됐다"**는 점입니다 — 회의록을 요약해 달라는 요청이 *요약해서 팀에 메일로 보내 놓으라*로 한 단계 이어집니다. 다만 출력 채널이 생기는 순간 7/9 GitLost가 보여준 교훈이 그대로 적용됩니다: **에이전트가 읽을 수 있는 것과 밖으로 내보낼 수 있는 것의 경계**를 조직이 명시적으로 설계해야 합니다. Entra 관리자 동의가 필수인 것은 그 경계 설정을 관리자의 결정으로 강제한 셈입니다 — 도입 전 어떤 그룹에 쓰기까지 열지부터 정하세요. ## 개발자 워크플로우 팁

### Plan mode를 '단계 게이트'로 써라 — 1단계만 실행하고 멈추게 하라

Plan mode(Shift+Tab)로 계획을 세우게 하는 것까지는 널리 알려진 관행이지만, 이번 주 갱신된 실전 가이드들이 공통으로 짚는 다음 단계는 **플랜의 실행 자체를 게이트로 쪼개는 것**입니다.

- **플랜을 직접 편집하라**: 제안된 플랜을 **Ctrl+G로 에디터에 열어** 순서를 바꾸고 불필요한 단계를 지운 뒤 승인하세요 — 채팅으로 수정을 부탁하는 왕복보다 빠르고 정확합니다.
- **한 단계씩 끊어라**: 큰 기능일수록 전체 플랜을 무인으로 돌리지 말고 **"1단계만 구현하고 멈춰서 보여줘" → 검토 → "계속"**으로 진행하세요. 느려 보이지만, **잘못된 방향을 6단계째가 아니라 1단계에서** 잡기 때문에 실제로는 빠릅니다.

```text
Shift+Tab   # plan mode — 읽기 전용으로 계획만 수립
Ctrl+G      # 제안된 플랜을 에디터로 열어 직접 수정
"1단계만 구현하고 멈춰서 diff를 보여줘"   # 단계 게이트
"좋아, 계속"                               # 검토 후 다음 단계
```

핵심은 7/12의 *모델을 작업 유형별로 골라라*와 같은 자리의 **실행 단위 설계**입니다 — 모델 선택이 *누가 일할지*의 문제라면, 단계 게이트는 *한 번에 얼마나 맡길지*의 문제입니다. 사람의 검토가 들어갈 지점을 플랜 경계에 미리 심는 것이 무인 실행의 실패 비용을 가장 싸게 줄입니다. [Iwo Szapar — 8 Rules](https://www.iwoszapar.com/p/claude-code-best-practices) · [Builder.io — 50 Tips](https://www.builder.io/blog/claude-code-tips-best-practices)

### 시크릿 위생 — 에이전트의 자격증명 파일 접근을 deny 규칙으로 봉인하라

위 M365처럼 에이전트가 **쓰기 권한(이메일 발송·파일 생성)**까지 얻는 흐름에서, *읽히면 안 되는 파일*의 관리는 더 이상 매너가 아니라 방어선입니다.

- **채팅·CLAUDE.md에 키를 넣지 마세요**: API 키·토큰은 환경변수로 관리하고, `.env`는 `.gitignore`에 두세요 — 컨텍스트에 실린 시크릿은 트랜스크립트·로그·서브에이전트로 복제됩니다.
- **deny 규칙으로 기계적으로 차단하세요**: 습관에 기대지 말고 권한 설정으로 자격증명 파일 읽기를 막으세요.

```json
// .claude/settings.json — 자격증명 파일 접근 봉인
{
  "permissions": {
    "deny": [
      "Read(./.env)",
      "Read(./.env.*)",
      "Read(./secrets/**)"
    ]
  }
}
```

핵심은 7/10 브리핑의 *반드시 지켜야 하는 규칙은 문장이 아니라 hook으로*와 같은 원리입니다 — CLAUDE.md의 "시크릿을 읽지 마라"는 문장은 조언이지만, deny 규칙은 **강제**입니다. 서드파티 스킬·플러그인을 쓰는 저장소(7/8 SkillCloak)라면 이 봉인의 가치가 배가됩니다.
