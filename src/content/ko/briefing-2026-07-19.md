### Artifacts가 '열어 본 사람'의 MCP 커넥터를 호출한다 — 데모에서 사내 도구로 (Week 29)

Claude Code 문서의 Week 29(7/13~17) 디제스트가 조용히 큰 변화를 실었습니다 — **게시된 아티팩트가 페이지를 연 뷰어 각자의 MCP 커넥터를 통해 라이브 데이터를 당겨오고 액션을 실행**할 수 있습니다.

- **무엇이 달라지나**: 지금까지 아티팩트는 *만든 시점의 데이터가 박제된* 정적 산출물에 가까웠습니다. 이제 열어 본 사람의 권한으로 **그 사람의 커넥터(이슈 트래커·DB·사내 시스템)를 실시간 호출**하니, 같은 아티팩트가 보는 사람마다 자기 데이터로 동작하는 **작은 앱**이 됩니다.
- **배포 경로도 함께**: **공개 공유 링크**와 **Team·Enterprise의 편집자 역할**이 같이 붙었고, **Claude Tag 세션에서 만든 아티팩트**도 지원됩니다 — 만들기(세션)→다듬기(편집자)→배포(링크)의 흐름이 완성됐습니다.
- **어디에 쓰나**: Claude Code로 짠 대시보드·조회 도구를 사내에 나눠 줄 때, 지금까지는 호스팅·인증을 직접 해결해야 했습니다 — MCP 커넥터가 인증·데이터 접근을 대신하는 아티팩트는 그 배포 비용을 크게 줄이는 선택지입니다.

핵심은 **"MCP가 에이전트의 도구를 넘어 '배포된 산출물의 백엔드'로 확장됐다"**는 점입니다 — 7/16 워크플로 팁의 dynamic workflows가 *반복 오케스트레이션의 저장*이었다면, 이쪽은 *산출물 자체의 저장·배포*입니다. 사내 미니 도구를 자주 만드는 팀이라면 첫 실험 대상으로 삼을 만합니다. [Claude Code What's New](https://code.claude.com/docs/en/whats-new)


### 성공 주장 대신 증거를 요구하라 — 공식 best practices의 제1원칙

Claude Code 공식 best practices 문서가 **단 하나만 도입한다면 검증(verification)**이라고 못 박는 원칙을, 실무 절차로 정리합니다.

- **검증 수단을 먼저 줘라**: 에이전트에 *자기 출력을 스스로 확인할 방법*(테스트 스위트·타입 체크·실행 가능한 재현 스크립트)을 쥐여 주는 것이 가장 임팩트가 큰 단일 습관입니다 — 검증 수단이 없는 위임은 확신에 찬 오답을 낳습니다.
- **주장이 아니라 증거로 보고받아라**: 작업 완료를 *고쳤습니다*라는 문장이 아니라 **증거 — 실행한 명령과 그 출력, 테스트 결과, 스크린샷 — 로 보고**하게 하세요. 증거를 검토하는 것이 검증을 직접 재실행하는 것보다 빠릅니다.
- **CLAUDE.md에 명문화**: *완료 선언 전 반드시 테스트 출력을 첨부하라* 같은 규칙을 CLAUDE.md에 두면 매 세션 자동 적용됩니다 — 예외 없이 강제해야 하는 항목이라면 hook으로 올리는 것(7/13 팁)이 다음 단계입니다.

```markdown
# CLAUDE.md 예시
- 수정 완료를 선언하기 전에 반드시 관련 테스트를 실행하고 출력을 보여줄 것
- "동작합니다" 대신 실행한 명령·결과를 그대로 첨부할 것
```

핵심은 최근 릴리스 흐름과 정확히 맞물립니다 — v2.1.214의 heartbeat·OTel 상관관계(7/18)가 *런타임의 증거*를 기록한다면, 이 팁은 *대화 층위의 증거*를 강제합니다. 위임을 늘릴수록 증거 요구가 사람의 검토 비용을 지키는 마지막 지렛대입니다. ## 흥미로운 프로젝트 & 도구

- **Live Git — Claude Code가 원격 GPU 서버를 직접 모는 CLI**: 원격 GPU 개발의 고질병 — VS Code Remote는 IDE에 묶이고 로컬 AI 에이전트가 못 들어가며, rsync 루프는 대형 저장소에서 느린 — 을 겨눈 Go 기반 오픈소스 CLI입니다(`lg`). **FUSE 지연 로딩**으로 100GB급 원격 저장소도 메타데이터를 즉시 띄우고, **캐시된 SSH 터널**로 로컬 명령을 원격 GPU에서 실행하며(컬러 출력·진행 바·종료 코드 실시간 스트리밍), **분리형(detached) 잡**은 노트북이 잠들거나 네트워크가 끊겨도 살아남습니다. 눈에 띄는 건 에이전트 네이티브 설계입니다 — **AGENTS.md 가이드를 자동 생성해 Claude Code가 원격 파일 수정·GPU 명령 실행을 안전하게 스스로 반복**하게 합니다. ML 워크플로에서 *에이전트는 로컬, 컴퓨트는 원격*의 간극을 겪어 본 팀이라면 곧장 시험할 값어치가 있습니다. [GeekNews](https://news.hada.io/topic?id=31550) · [프로젝트](https://iamtaehyunpark.github.io/livegit/)
- **drawdb-collaborative — 셀프호스팅 실시간 협업 ERD**: 오픈소스 ERD 도구 drawDB를 포크해 **무료 셀프호스팅 실시간 협업**을 얹은 프로젝트입니다 — 같은 다이어그램에 접속한 사용자가 자동 연결되고, 마우스 커서가 서로 보이며, 제목·테이블·관계의 모든 수정이 실시간 동기화됩니다(한국어 로컬라이즈 포함). 기존 drawDB·ChartDB에 *무료로 함께 그리는* 선택지가 없다는 문제의식에서 출발했습니다 — 스키마 설계를 화면 공유로 받아 적던 팀에게 실용적인 대안입니다.
