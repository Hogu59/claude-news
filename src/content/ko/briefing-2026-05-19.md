### Claude Code Routines — 크론잡 없는 클라우드 자동화 (리서치 프리뷰)

Code with Claude SF에서 발표된 **Routines**가 리서치 프리뷰로 사용 가능합니다. 로컬 서버나 크론잡 없이 Claude Code 클라우드 인프라에서 자동화 워크플로우를 실행하는 기능입니다.

**트리거 유형 3가지**:
- **Scheduled**: 시간별, 야간, 주간 반복 실행
- **API**: HTTP POST 엔드포인트로 외부 트리거
- **Webhook**: GitHub 이벤트(PR 오픈, 머지 등)에 반응

**실전 활용**: 이슈 트리아지, 배포 검증, 알림 분석, 문서 갱신, 크로스 언어 SDK 동기화 등에 팀들이 이미 사용 중입니다.

```bash
# Routine 설정 예시 (claude.com 대시보드 또는 CLI)
claude routines create \
  --name "nightly-triage" \
  --schedule "0 2 * * *" \
  --repo "org/project" \
  --prompt "새 이슈를 분류하고 라벨을 붙여줘"
```

**한도**: Pro 플랜 일 5회, Max/Enterprise는 상향. Pro 이상 무료 제공.

## 개발자 워크플로우 팁

### AI와 복리처럼 성장하기 — Eugene Yan(Anthropic)의 5원칙

Anthropic 테크니컬 스태프 Eugene Yan이 AI 협업을 체계화하는 5가지 원칙을 공유했습니다. 핵심은 "한 번 설정하면 세션마다 복리처럼 효과가 쌓이는 구조"를 만드는 것입니다.

1. **컨텍스트 인프라**: `~/src`, `~/vault` 등 모델이 검색하기 쉬운 디렉토리 구조 + MCP 연결
2. **설정 인코딩**: CLAUDE.md를 "신입 온보딩 문서처럼" 작성 — 행동 규칙, 선호도, 금지사항 정의
3. **검증 자동화**: 린팅 훅 → LLM 리뷰 순으로 저비용 검증부터 사다리 구조로 배치
4. **위임 확대**: 병렬 세션으로 처리량 확대, 성공 기준과 메트릭을 미리 정의
5. **피드백 루프**: 과거 트랜스크립트를 마이닝하여 설정을 지속 개선

실무에서 바로 적용할 포인트는 3번(검증 사다리)입니다. 비용이 낮은 검증(린트, 타입 체크)을 먼저 실행하고, 통과한 경우에만 비용이 높은 검증(LLM 리뷰, 통합 테스트)을 수행합니다.

[Eugene Yan 블로그](https://eugeneyan.com) | [GeekNews](https://news.hada.io/topic?id=29606)

### 병목은 코딩이 아니라 문제 정의 — AI가 프로세스를 빠르게 만들지 않는 이유

AI 코드 생성 속도가 아무리 빨라도, 소프트웨어 개발이 느린 진짜 원인은 "모호한 요구사항을 명확한 문제로 전환하는 과정"에 있습니다. AI도 동일한 업스트림 문제에 직면합니다.

**실전 교훈**: Claude Code에 작업을 위임하기 전에, "병목 지점에 예측 가능하고 고품질의 입력을 공급"하는 것이 우선입니다. 스펙이 모호하면 Claude도 모호한 결과를 냅니다. 이것이 `plan` 모드를 먼저 사용하라는 Anthropic 권고와 일맥상통합니다.

## 보안/제한 이슈

### ClaudeBleed — Claude Chrome 확장 프로그램 하이재킹 취약점 (5/8)

보안 기업 **LayerX**가 Claude의 Chrome 확장 프로그램에서 **ClaudeBleed** 취약점을 공개했습니다. 다른 Chrome 확장이 Claude 에이전트를 탈취하여 Gmail, GitHub, Google Drive 데이터를 유출할 수 있는 문제입니다.

**공격 방식**: Claude in Chrome의 코드가 동일 origin에서 실행되는 모든 스크립트의 통신을 허용하지만, **누가 스크립트를 실행하는지 검증하지 않습니다**. 악성 확장이 Claude에 명령을 보내 이메일 전송, 데이터 삭제, 문서 공유를 수행할 수 있습니다.

**패치 상태**: Anthropic이 v1.0.70(5/6)을 배포했으나, LayerX는 "standard 모드에서만 원격 명령을 차단"하는 수정이라 공격자가 **privileged 모드로 전환하면 우회 가능**하다고 지적했습니다. 근본 원인이 해결되지 않은 상태입니다.

**권장 조치**:
- 설치된 Chrome 확장 목록을 정기적으로 점검
- 출처 불분명한 확장 프로그램 제거
- Claude Chrome 확장의 권한 범위를 최소화

[SecurityWeek](https://www.securityweek.com/vulnerability-in-claude-extension-for-chrome-exposes-ai-agent-to-takeover/) | [Hackread](https://hackread.com/claudebleed-vulnerability-hackers-claude-chrome-extension/) | [LayerX](https://layerxsecurity.com/blog/a-flaw-in-claudes-browser-extension-allows-any-extension-to-hijack-it/)
