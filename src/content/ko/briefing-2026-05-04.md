## 개발자 워크플로우 팁

### "CLAUDE.md는 제안이고, Hook이 법이다" — 준수율 70% vs 100%

Claude Code 커뮤니티에서 반복적으로 확인되는 중요한 교훈이 있습니다. **CLAUDE.md의 지시사항은 약 70%만 준수**됩니다. 스타일 선호도라면 괜찮지만, "main에 push 금지"나 "프로덕션 데이터 삭제 금지" 같은 규칙에서 70%는 프로덕션 사고를 기다리는 것과 같습니다.

**핵심 구분**:
- `CLAUDE.md` = 모델에 대한 **교육** (advisory, 확률적 준수)
- `settings.json` + Hook = 시스템 수준 **강제** (deterministic, 100% 준수)
- Hook은 LLM의 추론 체인 바깥에서 실행되므로, 모델이 무시할 수 없습니다

```json
// settings.json — main 브랜치 push 차단 (Hook으로 강제)
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -q 'git push.*main'; then echo 'BLOCKED: main push forbidden' >&2; exit 1; fi"
      }]
    }]
  }
}
```

정리하면: **스타일·관습은 CLAUDE.md에, 절대 규칙은 Hook에** 넣으십시오.

### 세션 종료 시 레트로 + lessons.md 패턴

세션이 끝날 때 Claude에게 "이번 세션에서 뭘 배웠어?"라고 물어보고, 그 결과를 `lessons.md`에 저장하는 패턴이 커뮤니티에서 확산되고 있습니다. Claude가 실수를 교정받을 때마다 해당 규칙을 `lessons.md`에 추가하면, 시간이 지남에 따라 **프로젝트 특화 학습이 누적**됩니다.

저장 위치 기준:
- 일반 프로젝트 컨셉 → `CLAUDE.md`
- 코딩 관습·좋은 관행 → 규칙 파일
- 기술 능력 → Skill 파일
- 프로젝트 세부사항 → Markdown 문서
- 아키텍처 결정 → ADR (Architecture Decision Records)

```bash
# 세션 종료 시 레트로
claude "이번 세션에서 배운 것을 정리해줘.
일반 규칙은 CLAUDE.md에, 코딩 관습은 lessons.md에 추가해줘."
```

### 러시아의 위키피디아 조직적 오염 — AI 훈련 데이터 오염 위협 (5/3)

Atlantic Council이 러시아의 조직적 위키피디아 조작 작전을 분석한 보고서를 발표했습니다. "Pravda Network"라는 80개 이상의 가짜 뉴스 포털이 친러시아 서사를 확산하고, 우크라이나 관련 48개 위키피디아 문서에서 조직적 편집 패턴이 탐지되었습니다.

**AI 개발자에게 중요한 이유**: 위키피디아는 LLM 훈련의 핵심 소스입니다. 가짜 친러시아 출처가 위키피디아에 삽입되면, 이를 학습한 **AI 챗봇 사용자가 반우크라이나·반서방 메시지에 노출**됩니다. 훈련 데이터 오염(data poisoning)이 단순 사이버 공격이 아닌 국가 수준의 인지전 도구로 활용되고 있다는 경고입니다.

Claude Code로 RAG나 지식 기반 시스템을 구축할 때, 위키피디아 데이터의 신뢰성 검증 레이어를 고려할 필요가 있습니다.

## 생태계 & 플러그인

### CTX — Claude Code 크로스세션 메모리 플러그인

Claude Code의 세션 간 컨텍스트 손실 문제를 해결하는 플러그인입니다. 프롬프트 제출 시 세 가지 컨텍스트를 자동 주입합니다:

- **G1**: Git 로그 기반 의사결정 타임라인 (과거 추론 이해)
- **G2**: BM25 시맨틱 검색으로 관련 파일 자동 주입
- **CM**: SQLite FTS5 기반 대화 이력 검색 (선택적 벡터 지원)

10,000+ 턴에서 **0.880 메모리 리콜 정확도**, 컨텍스트 주입 지연 **1ms 이내**를 달성합니다. 클라우드 의존성 없이 완전 로컬로 동작합니다.

```bash
# 설치
pip install ctx-retriever && ctx-install
# 또는 Claude Code에서
/plugin install ctx@jaytoone
```

### VibeFrame — 코딩 에이전트를 위한 스토리보드 기반 영상 CLI

Claude Code 같은 코딩 에이전트가 **마크다운 파일로 영상을 제작**할 수 있게 해주는 CLI 도구입니다. `STORYBOARD.md`와 `DESIGN.md` 파일이 에이전트가 읽고 수정하는 인터페이스 역할을 합니다.

- `vibe build`: 에셋 생성 및 씬 합성
- `vibe render`: MP4 출력
- `vibe inspect`: 렌더 결과 검토
- HTML/CSS/JS 기반 결정론적 합성, FFmpeg 편집

이미지·영상·내레이션·음악 생성을 위한 멀티 프로바이더를 지원하며, dry-run으로 사전 테스트가 가능합니다.
