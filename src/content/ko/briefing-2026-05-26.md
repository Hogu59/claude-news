## 주요 신규 기능 & 실전 활용

### Project Glasswing 첫 결과 — Claude Mythos Preview, 10,000+ 고위험 취약점 발견 (5/22)

Anthropic의 보안 AI 프로젝트 **Project Glasswing**이 첫 공식 결과를 발표했습니다. 약 50개 파트너 조직이 **Claude Mythos Preview**를 방어적 보안 워크플로우에 배포하여, 1개월간 인터넷에서 가장 널리 사용되는 소프트웨어에서 **10,000건 이상의 고위험/치명적 취약점**을 발견했습니다.

**주요 발견 사례**:
- **Cloudflare**: 2,000건의 버그 발견, 그중 400건이 고위험 이상
- **Mozilla Firefox 150**: 271건의 취약점 발견 및 수정 완료
- **wolfSSL 암호화 라이브러리**: 인증서 위조 결함(CVE-2026-5194) 발견 — 가짜 은행 사이트를 브라우저 경고 없이 생성할 수 있는 익스플로잇까지 구축
- **오픈소스 1,000+ 프로젝트**: 약 3,900건의 고위험 발견 확인 예정

**Anthropic의 경고**: "Mythos는 개발자가 패치할 수 있는 속도보다 빠르게 버그를 찾고 있다"며, 강력한 안전장치가 마련될 때까지 일반 공개를 보류한다고 밝혔습니다. 독립 분석가들은 제한적 기업 접근이 2026년 하반기, 일반 공개는 2027년 이후로 전망합니다.

**Claude Code 개발자에게의 시사점**: Mythos급 보안 스캐닝이 향후 Claude Code에 통합될 가능성이 높습니다. 현재는 **Claude Security(퍼블릭 베타)**를 통해 코드베이스 스캔, 취약점 분류, 수정 생성이 가능합니다.

[TechTimes](https://www.techtimes.com/articles/317076/20260524/anthropic-moves-closer-public-claude-mythos-release-10000-critical-bugs-found-first.htm) | [The Decoder](https://the-decoder.com/anthropic-warns-claude-mythos-preview-finds-bugs-faster-than-developers-can-patch-them/) | [BusinessToday](https://www.businesstoday.in/technology/artificial-intelligence/story/anthropic-project-glasswing-mythos-preview-flagged-over-10000-security-flaws-across-critical-systems-533098-2026-05-25)

## 개발자 워크플로우 팁

### AI로 더 나은 코드를 더 천천히 작성하기 — 멀티모델 PR 리뷰 전략

Nolan Lawson이 5월 25일에 공개한 글에서, AI를 빠른 코드 생성이 아닌 **철저한 코드 리뷰**에 활용하는 접근법을 제시합니다. 핵심은 **여러 모델을 조합하여 오탐(false positive)을 줄이는 것**입니다.

**실전 패턴**:
1. **Claude 서브에이전트**로 PR 변경 사항에 대한 상세 리뷰 수행
2. **Codex**로 독립적인 두 번째 리뷰 실행
3. **두 모델이 모두 지적한 항목만** 우선 처리 — 단일 모델 대비 오탐이 크게 감소

Claude Code에서 `/code-review high`와 외부 도구를 병행하면, 이전 브리핑에서 다룬 "PR을 읽지 않고 머지하는 시대"의 위험을 실질적으로 줄일 수 있습니다. 속도가 아닌 **코드베이스 건강성**에 초점을 맞추는 것이 핵심 메시지입니다.

[Nolan Lawson](https://nolanlawson.com/2026/05/25/using-ai-to-write-better-code-more-slowly/) | [GeekNews](https://news.hada.io/topic?id=29870)

### 병목은 "조직"에 있다 — AI 도구 도입의 진짜 성공 조건

O'Reilly Radar에서 발행한 분석글이 AI 코딩 도구의 도입 성패를 가르는 요인을 지적합니다. 코드 작성 속도가 빨라져도 **실제 가치 전달 속도는 조직의 소프트웨어 엔지니어링 성숙도에 의존**합니다.

**핵심 통찰**: CI/CD 파이프라인, 자동화 테스트, 가드레일, 엔지니어링 인에이블먼트 등 기초 인프라가 갖춰진 조직만이 AI 도구의 속도를 실제 가치로 전환할 수 있습니다. 기초 투자 없이 Claude Code를 도입하면 **혼란만 증폭**될 수 있습니다.

**Claude Code 적용 체크리스트**:
- `CLAUDE.md`에 팀 컨벤션과 가드레일이 정의되어 있는가?
- hook으로 린트/테스트가 자동 실행되는가?
- `/code-review`가 PR 워크플로우에 통합되어 있는가?
- 에이전트 출력에 대한 검증 루프가 존재하는가?
