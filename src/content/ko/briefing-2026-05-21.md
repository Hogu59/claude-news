## 주요 신규 기능 & 실전 활용

### v2.1.145 — claude agents --json, 플러그인 사전 평가, OTEL 추적 강화 (5/19)

v2.1.145는 자동화·관측 가능성·플러그인 탐색을 크게 개선하는 업데이트입니다.

**`claude agents --json`**: 실행 중인 Claude 세션 목록을 JSON으로 출력합니다. tmux-resurrect, 상태 바, 세션 피커 등 외부 도구와 연동할 때 유용합니다. 여러 에이전트를 병렬로 관리하는 워크플로우에서 스크립트 기반 모니터링이 가능해졌습니다.

```bash
# 실행 중인 세션을 JSON으로 조회
claude agents --json

# jq와 조합하여 특정 세션 필터링
claude agents --json | jq '.[] | select(.status == "awaiting-input")'
```

**`/plugin` 설치 전 상세 정보**: Discover/Browse 화면에서 플러그인을 설치하기 **전에** 해당 플러그인이 제공하는 commands, agents, skills, hooks, MCP/LSP 서버 목록을 확인할 수 있습니다. v2.1.143의 토큰 비용 표시와 결합하면, 설치 전 완전한 영향 평가가 가능합니다.

**OTEL 스팬 개선**: `claude_code.tool` 스팬에 `agent_id`와 `parent_agent_id` 속성이 추가되었습니다. 멀티에이전트 실행에서 어떤 에이전트가 어떤 도구를 호출했는지 추적이 가능해졌습니다. Datadog, Honeycomb 등 관측 도구와 연동 시 디버깅이 용이합니다.

**기타 변경사항**:
- 상태 표시줄 JSON에 GitHub 저장소·PR 정보 포함
- `claude agents` 터미널 탭 제목에 입력 대기 세션 수 표시
- 풀스크린 모드에서 슬래시 명령·@멘션 목록의 마우스 호버/클릭 지원
- Stop/SubagentStop hook 입력에 `background_tasks`, `session_crons` 필드 추가
- 권한 프롬프트 우회, MCP 슬래시 명령, 스피너 멈춤, 크로스프로젝트 resume 힌트 등 다수 버그 수정

## 개발자 워크플로우 팁

### claude agents --json — 멀티 세션 자동화의 시작점

v2.1.145의 `claude agents --json`은 단순한 조회 명령이 아닙니다. 이것을 기반으로 **멀티 에이전트 관리 자동화**를 구축할 수 있습니다.

**실전 활용 패턴**:
1. **tmux 상태 바 연동**: 입력 대기 세션 수를 상태 바에 표시하여, 터미널을 전환하지 않고 에이전트 상태 파악
2. **세션 헬스 체크**: 크론잡으로 주기적으로 세션 목록을 확인하고, 비정상 상태의 세션을 알림
3. **대시보드 구축**: JSON 출력을 파싱하여 팀 전체의 에이전트 활용 현황을 시각화

```bash
# tmux 상태 바에 대기 세션 수 표시 예시
AWAITING=$(claude agents --json | jq '[.[] | select(.status == "awaiting-input")] | length')
echo "Claude: ${AWAITING} waiting"
```

[GitHub v2.1.145](https://github.com/anthropics/claude-code/releases/tag/v2.1.145)

### /plugin 사전 평가 — 설치 전에 영향 범위를 파악하기

v2.1.143의 토큰 비용 표시 + v2.1.144의 최근 업데이트 날짜 + v2.1.145의 상세 구성 정보가 결합되면서, 플러그인 설치 전 **완전한 사전 평가**가 가능해졌습니다. 플러그인이 추가하는 commands, agents, skills, hooks, MCP/LSP 서버를 사전에 확인하면 예상치 못한 동작을 방지할 수 있습니다.

특히 MCP 서버를 제공하는 플러그인은 외부 연결을 열 수 있으므로, TrustFall 같은 보안 이슈를 고려하면 **설치 전 확인이 필수**입니다.

## 생태계 & 플러그인

### Anthropic, Stainless 인수 — SDK/MCP 생성 도구 내재화 ($3억+, 5/18)

Anthropic이 SDK·CLI·MCP 서버 자동 생성 스타트업 **Stainless**를 **$3억 이상**에 인수했습니다. Stainless는 Anthropic, OpenAI, Google, Cloudflare 등의 공식 SDK를 생성해 온 도구입니다.

**개발자 영향**:
- Stainless의 **호스티드 제품이 종료**됩니다. 기존 고객은 생성된 SDK에 대한 완전한 소유권·수정·확장 권한을 유지합니다.
- OpenAI, Google 등은 SDK 생성 도구를 **자체 구축하거나 대안을 찾아야** 합니다.
- Anthropic은 SDK·MCP 생성 파이프라인을 내재화하여, Claude 에이전트의 API 연결 품질을 직접 통제할 수 있게 됩니다.

MCP 생태계의 핵심 인프라를 확보한 전략적 인수이며, 경쟁사에게는 단기적 혼란을 야기할 수 있습니다.

[TechCrunch](https://techcrunch.com/2026/05/18/anthropic-has-acquired-the-dev-tools-startup-used-by-openai-google-and-cloudflare/) | [Anthropic 공식](https://www.anthropic.com/news/anthropic-acquires-stainless)

### Gemini CLI, 6월 18일 종료 → Antigravity CLI 전환 (5/20)

Google이 **Gemini CLI를 2026년 6월 18일에 종료**하고, Antigravity CLI로 전환합니다. Google AI Pro/Ultra 사용자와 무료 사용자가 영향을 받으며, Gemini Code Assist Standard/Enterprise 라이선스 사용자는 **계속 지원**됩니다.

**Antigravity CLI 특징**:
- Go로 작성, 더 빠른 응답 속도
- 멀티에이전트 오케스트레이션 기본 지원
- 터미널 샌드박싱 내장
- Agent Skills, Hooks, Subagents 유지 (Antigravity 플러그인으로 전환)

**주의**: Antigravity CLI는 **오픈소스가 아닙니다**. Gemini CLI가 오픈소스였던 것과 달라, 개발자 커뮤니티에서 비판이 있습니다. Claude Code 사용자에게는 직접 영향이 없지만, AI 코딩 도구 시장 재편의 신호입니다.

[Google Developers Blog](https://developers.googleblog.com/an-important-update-transitioning-gemini-cli-to-antigravity-cli/) | [The Register](https://www.theregister.com/ai-ml/2026/05/20/bye-bye-gemini-cli-google-nudges-devs-toward-antigravity/5243605)
