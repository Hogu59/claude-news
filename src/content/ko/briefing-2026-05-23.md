## 주요 신규 기능 & 실전 활용

### v2.1.149 — /usage 카테고리별 분석, /diff 키보드 내비게이션, 보안 수정 (5/22)

v2.1.149는 비용 가시성과 UI 편의성을 개선하고, 보안 취약점 여러 건을 수정한 업데이트입니다.

**`/usage` 카테고리별 비용 분석**: `/usage`가 이제 전체 사용량뿐 아니라 **skills, subagents, plugins, MCP 서버별** 비용 분석을 보여줍니다. 어떤 구성 요소가 한도를 소비하는지 한눈에 파악할 수 있어, 플러그인이나 MCP 서버의 비용 효율성을 실측 기반으로 판단할 수 있습니다.

```bash
# 카테고리별 사용량 확인
/usage
# Skills: 12%, Subagents: 45%, Plugins: 8%, MCP: 35%
```

**`/diff` 키보드 내비게이션**: 상세 뷰에서 화살표, `j`/`k`, `PgUp`/`PgDn`, `Space`, `Home`/`End` 키로 스크롤할 수 있습니다. 대규모 diff를 마우스 없이 빠르게 탐색할 수 있습니다.

**GFM 체크박스 렌더링**: 마크다운의 `- [ ]` / `- [x]` 태스크 리스트가 일반 불릿 대신 **체크박스로 렌더링**됩니다. Claude가 생성하는 TODO 리스트나 진행 상황 보고가 시각적으로 더 명확해졌습니다.

**엔터프라이즈 설정 — `allowAllClaudeAiMcps`**: 새로운 관리 설정으로, `managed-mcp.json`과 함께 claude.ai 클라우드 MCP 커넥터를 로드할 수 있습니다. 엔터프라이즈 환경에서 관리형 MCP와 클라우드 MCP를 병행 사용할 수 있게 되었습니다.

[GitHub v2.1.149](https://github.com/anthropics/claude-code/releases/tag/v2.1.149)

## 개발자 워크플로우 팁

### /usage 카테고리 분석 — 한도 소비 원인을 추적하기

v2.1.149의 `/usage` 카테고리별 분석은 **"왜 한도가 빨리 소진되는지"**에 대한 답을 줍니다. Uber가 4개월 만에 연간 예산을 소진한 사례에서 보듯, 비용 가시성은 AI 코딩 도구 운영의 핵심입니다.

**실전 활용 패턴**:
1. **플러그인 비용 감사**: 설치한 플러그인 중 컨텍스트를 과도하게 소비하는 것을 식별하여 비활성화
2. **서브에이전트 최적화**: 병렬 에이전트 수와 비용의 상관관계를 확인하여 적정 수준 조정
3. **MCP 서버 비교**: 동일 기능의 MCP 서버 간 비용 차이를 실측하여 효율적인 서버 선택

특히 v2.1.143에서 추가된 마켓플레이스 토큰 비용 표시와 결합하면, **설치 전 예측 비용**과 **실제 사용 비용**을 비교하여 플러그인 투자 대비 효과를 정량화할 수 있습니다.

[GitHub v2.1.149](https://github.com/anthropics/claude-code/releases/tag/v2.1.149)

### Code with Claude London — "PR을 읽지 않고 머지하는 시대"에 대한 경고

MIT Technology Review가 Code with Claude London 행사를 분석하며, Anthropic 엔지니어가 청중에게 "Claude가 작성한 PR을 읽지 않고 머지한 적이 있는지" 물었을 때 **상당수가 손을 들었다**고 보도했습니다.

**교훈**: AI 코딩 도구의 생산성 향상이 **코드 리뷰 생략**으로 이어지면, 기술 부채와 보안 취약점이 누적됩니다. v2.1.147의 `/code-review` 명령이나 `claude ultrareview`를 활용하여, 인간 리뷰 없이도 최소한의 자동 검증을 거치도록 하는 것이 중요합니다.

[MIT Technology Review](https://www.technologyreview.com/2026/05/21/1137735/anthropics-code-with-claude-showed-off-codings-future-whether-you-like-it-or-not/)

## 보안/제한 이슈

### v2.1.149 보안 수정 — PowerShell 권한 우회, worktree 샌드박스 (5/22)

v2.1.149에서 여러 보안 관련 버그가 수정되었습니다.

**PowerShell 권한 우회**: PowerShell 내장 `cd` 함수(`cd..`, `cd\`, `cd~`, `X:`)가 **작업 디렉토리를 감지 없이 변경**할 수 있는 문제가 수정되었습니다. 이전에는 이 명령들이 Claude Code의 권한 시스템을 우회하여 허용되지 않은 디렉토리로 이동할 수 있었습니다.

**worktree 샌드박스 범위 오류**: git worktree에서 샌드박스 쓰기 허용 목록이 공유 `.git` 디렉토리만이 아니라 **저장소 루트 전체**를 포함하는 버그가 수정되었습니다. 이로 인해 worktree 내 에이전트가 의도보다 넓은 범위에 파일을 쓸 수 있었습니다.

**권한 분석 갭**: `PWD`/`OLDPWD`/`DIRSTACK` 환경변수의 오래된 값으로 인한 권한 분석 오류가 수정되었습니다.

**권장 조치**: Windows 사용자는 즉시 v2.1.149로 업데이트할 것을 권장합니다. worktree 격리 기능(`isolation: "worktree"`)을 사용하는 팀은 이전 버전에서 생성된 worktree를 재생성하는 것을 고려하십시오.

[GitHub v2.1.149](https://github.com/anthropics/claude-code/releases/tag/v2.1.149)
