## 主要な新機能と実践活用

### `CLAUDE_CODE_DISABLE_MOUSE_CLICKS` — フルスクリーンのマウスキャプチャを切ってテキスト選択を取り戻す（v2.1.195）

今回のリリースで最も「機能」らしい項目です。新しい環境変数 **`CLAUDE_CODE_DISABLE_MOUSE_CLICKS`** を有効にすると、フルスクリーンモードで**マウスのクリック・ドラッグ・ホバーのキャプチャを切りつつ、ホイールスクロールはそのまま**残します。

```bash
# フルスクリーンでマウスのクリック/ドラッグ/ホバーのキャプチャを切り、ホイールスクロールだけ残す
export CLAUDE_CODE_DISABLE_MOUSE_CLICKS=1
```

これは **6/23（v2.1.187）で追加された「メニューのマウスクリック」の対**です。あのとき権限プロンプトや `/model`・`/config` をマウスで選べるようになりましたが、マウスをキャプチャした途端、**ターミナルのネイティブなドラッグ選択・コピー**が壊れる人がいました（tmux や一部ターミナルで特に顕著）。今回の変数はその off スイッチです — **マウスで出力テキストをドラッグしてコピーする流れを守りたい**なら有効にすればよい、というわけです。小さいですが、「マウス対応を足しつつ、切る自由も与える」という点で、フルスクリーン UI を常用する人に直接効きます。[GitHub v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)

## 開発者ワークフローティップス

### ハイフン入り hook matcher が「完全一致」に修正された — ガード hook・MCP ルールを再検証せよ

CLAUDE.md のルールを **hook で強制**したり、**MCP ツールを matcher でガード**している人は、この一行を必ず押さえてください。v2.1.195 は、**ハイフンを含む識別子**（例：`code-reviewer`、`mcp__brave-search`）を使った hook matcher が**部分一致（substring）で誤って一致**していた不具合を修正しました — これからは**完全一致（exact-match）**です。

```jsonc
// .claude/settings.json — ハイフン matcher の意味が変わった
{
  "hooks": {
    "PreToolUse": [
      // 'mcp__brave-search' は今やその名前のツールだけに正確に一致する。
      // その MCP サーバーの「すべてのツール」に一致させるにはワイルドカードを明示する:
      { "matcher": "mcp__brave-search__.*", "hooks": [ /* ... */ ] }
    ]
  }
}
```

肝心なのは、この修正が**双方向に挙動を変え得る**点です — ① これまで `mcp__brave-search` は*部分一致*だったため、意図に反して**名前が重なる別のツールまで一致**し、ガードが広くかかりすぎていた可能性があります。② 逆に `mcp__brave-search` で**そのサーバーのすべてのツールに一致させる**つもりだった場合、exact-match になったことで**静かに効かなくなり**ます（`mcp__brave-search__.*` に書き換える必要があります）。どちらにせよ、**あなたのガードが実際に何を捕まえていたか**がこのアップグレードで変わります。6/25 の**カンマ区切り matcher の不発火修正**とまったく同じ筋です — *その hook が意図したツールにだけ発火している保証*がなければ、強制も成立しません。ハイフン入り matcher（特に `mcp__` ツール）を使っているなら、**① アップデートし、② 危険なコマンドで hook が正確に発火するか再現**してください。[GitHub v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)

### `claude agents` で background を長く回すなら — クラッシュ復旧が堅くなった

background エージェントを立ち上げて `claude agents` で管理する流れなら、v2.1.195 の安定化のまとまりが直接効きます。6/26 で扱った*background エージェントの挙動改善*とは別物で、今回は**クラッシュ・再起動経路のデータ整合性**修正です。

- **新バージョンが書いたジョブの消失**：より新しい Claude Code バージョンが記録した background ジョブが、`claude agents` で**一覧から消えたりデータを失ったり**していた問題が修正されました — 複数バージョンを行き来して同じジョブを引き継ぐ際の喪失を防ぎます。
- **クラッシュしたジョブの再オープンで空白画面**：クラッシュした background ジョブを開き直すと、再起動画面の代わりに**最大5秒の空白画面**が出ていた問題が修正されました。
- **daemon が到達不能で再起動を阻む**：コントロールソケットの起動に失敗すると、background エージェントの daemon が**到達不能なまま立ち上がり再起動を阻んで**いた問題も修正されました。

肝は、**「夜通し回した background ジョブが朝には消えている/開けない」**たぐいの事故を減らすことです。長時間・複数の background ジョブを `claude agents` で運用しているなら（特に自動アップデートでバージョンが変わる環境では）、このまとまりだけでもアップデートの理由になります。[GitHub v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)

## セキュリティ・制限事項

### 外部プラグインに「明示的なインストール同意」が必要に — settings.json による自動有効化を遮断（v2.1.195）

小さく見えて、**サプライチェーンの観点で押さえる**べき修正です。v2.1.195 は、**プロジェクトの `.claude/settings.json` だけで有効化された外部プラグイン**が、**すべてのローダー経路で明示的なインストール同意を要求**するように修正しました — これまで一部の経路では、その同意を欠いたまま外部プラグインが有効化され得ました。

肝は、**「クローンしたリポジトリが自分の同意なしに外部プラグインを有効化する」**経路を塞ぐことです。誰かのリポジトリを受け取って Claude Code で開いた瞬間、そのリポジトリの `.claude/settings.json` に埋め込まれた外部プラグインが**黙ってロード**されるなら、それは任意コードが自分の環境で走る通り道になります。今後はその有効化が**ユーザーの明示的な同意**を経る必要があります — 6/23 の **`sandbox.credentials`**（サンドボックスのシークレットアクセス遮断）、6/22 の**「ルールは hook で強制」**と同じ並びで、*エージェント・設定ファイルが自動で持ち得る権限の面積を減らす*流れの一手です。他人のリポジトリを頻繁に開く人には即効性があります。[GitHub v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)

### Fable 5・Mythos 5 — 輸出規制15日目、政府は「狭い jailbreak」を理由に・Anthropic は反論（6/27）

連日扱っている件なので短く、ただし**新事実**を中心に。**6/27 時点で輸出規制15日目**（6/12 の命令）でも、**Fable 5・Mythos 5 は依然オフライン**で、6/25 にトラフィック0が確認されました。今回明らかになったのは**政府の理由**です。報道によれば、政府は Fable 5 の **jailbreak（回避）の可能性**を根拠に挙げましたが、その回避とは実質的に**「モデルに特定のコードベースを読ませてソフトウェアの欠陥を直させる」**という狭い（non-universal）もので、しかも**口頭の証拠**にとどまったといいます。

これに対し Anthropic は正面から**反論**しました — 「狭い jailbreak の可能性が見つかったというだけで、数億人に配備された商用モデルを回収することには同意しない。この基準を業界全体に適用すれば、すべてのフロンティアモデル提供者の新規モデル配備が事実上止まるだろう」という立場です。つまり 6/26 で扱った*Tom Brown による交渉の進展*とは別に、**事実関係や基準そのものをめぐって両者が衝突**しているということです。外国籍の開発者（韓国・日本など）の実務対応は変わりません — `fallbackModel`・`availableModels` で **Opus 4.8 のような代替経路**を用意し、モデルの可用性が会社ではなく**地政学と政策に左右され得る**前提で自動化を設計してください。加えて、**6/26 はインシデントなしでクリーン**でした（週前半の相次ぐノイズの後の、久々の無事故の一日です）。[Anthropic 声明](https://www.anthropic.com/news/fable-mythos-access) · [explainx](https://www.explainx.ai/blog/us-government-bans-fable-5-mythos-5-anthropic-export-control-2026)

## エコシステム＆プラグイン

### `/plugin` の Enable/Disable が「名前が異なるとき」効かない問題を修正（v2.1.195）

プラグインエコシステムの運用摩擦を減らす修正です。v2.1.195 は、あるプラグインの **`plugin.json` の `name` がマーケットプレイス登録名と異なるとき**、`/plugin` で **Enable/Disable が動かなかった**問題を修正しました。

肝は、**「名前の不一致のせいで、オン/オフすらできなかった」**プラグインを正常化することです — マーケットプレイス登録名とパッケージ内部の `name` が食い違うと、トグル自体が効かず、そのプラグインを管理できない状況が生じていました。6/25 の**マーケットプレイス `renames` 自動追従**、6/24 の **`/plugin` の整理の促し**と同じ並びで、*プラグインの配布・識別・管理*の粗いエッジをもう一段ならす変更です。複数のプラグインをマーケットプレイスで配布・購読するチーム（特に登録名と内部名が食い違いやすい場合）に実用的です。[GitHub v2.1.195](https://github.com/anthropics/claude-code/releases/tag/v2.1.195)

- **「スタートアップは burn 問題ではなく意思決定の問題を抱えている」（The Next Web）**：失敗したスタートアップ**431社を分析**した記事で、命題は明快です — **資金枯渇は症状であって病ではない。**本当の病は**分断されたデータと可視性の不足の中で下される悪い意思決定**だという診断です。創業者が財務・プロダクト・オペレーションを横断する**単一の真実の源（single source of truth）**を欠いたまま*推測に頼った決定*を繰り返すと、非効率は危機として噴き出すまで隠れてしまう、というわけです。だから解は資金を増やすことではなく、**支出と資源の流れを明瞭に見ること** — 反応的な管理から**証拠に基づく意図的な意思決定**へ移ることです。AI で出力を圧縮できる今こそ、*何を自動化するか*より*何を見て決めるか*が生死を分ける、という筋に読めます。[原文](https://thenextweb.com/news/startups-dont-have-a-burn-problem-they-have-a-decision-problem)
- **「ブログはただ当たり前を言うだけでいい」（Jim Nielsen）**：書くことに**斬新な洞察は必ずしも要らない**という慰めであり、勧めです。核心は — **「みんなが経験しているのに誰もはっきり言わない、当たり前のこと」**を明確に書くだけでも十分に価値がある、ということ。斬新さを追って書けなくなるより、**ありふれた苛立ちを正確に言語化する**方が、むしろ*より多くの*読者に届く、という観察です。技術ブログやドキュメントを**「大きな発見がないと書けない」**という重荷で先延ばしにしてきた開発者に、*書く敷居を下げる*短く明快な処方箋です — 6/26『理解の喜びと力』が*理解を守れ*と言ったのに対し、この記事は*その理解を平易な言葉で口に出せ*と勧めます。[原文](https://blog.jim-nielsen.com/2026/blogging-stating-the-obvious/)
- **「盗むのも技術だ（Stealing Is a Skill）」（ben-mini）**：**純粋な独創性を追うより、よくできた仕事を戦略的に複製する**ほうが、学習と実行の近道だという挑発的な記事です。核心は複製そのものではなく、**「何を・なぜ・どれだけ盗むかを決める眼」**に技術がある、ということ — 他人の成果をそのまま写して**ベースライン**を立て、*何を変えるべきか*をそこから学び、自分の洞察を足して意味ある変奏を作る、という流れです。Virgil Abloh の**「3%アプローチ」**（既存を3%変えれば新しくなる）と、あるマーケティングサイトを丸ごとリデザインした事例が根拠として挙がります。*LLM の出力を写すこと*と*AI 時代の学習*を同時に問う 6/26・6/25 の流れの中で — **「写せ、ただし何を・なぜ写すか分かったうえで」**こそが力だという、別角度の一手です。[原文](https://ben-mini.com/2026/stealing-is-a-skill)

## 注目プロジェクト＆ツール

- **SongRyeon Core — 「LLM が言った判断」と「コードが実際に確認した事実」を分離する**：AI エージェントが**推測をシステムの事実のように提示**したり、**コードが作った結果（fallback）と LLM の推論を混ぜて**報告したりする問題に正面から取り組むローカルエージェントランタイムの実験です。核心は **「LLM が主張したこと」と「コードが事実として検証したこと」を明確に区別**し、*エージェントがどう結論に至ったか*を透明にすること — つまりエージェントの産出を*もっともらしい物語*ではなく*検証可能な根拠*へ引き戻す試みです。6/25『AI 時代のプログラマー：非決定性の制御』、6/24『AI で Google をハック（検証は人が締める）』とまったく同じ筋 — **「エージェントの判断を鵜呑みにせず、コードで検証された部分と分けて見よ」**という原則をランタイムに落とし込んだツールです。[GitHub](https://github.com/Junghoo-developer/SongRyeon)
- **당근전톡 — 당근마켓の物件を「KakaoTalk から自然言語で」読む MCP 実験**：KakaoTalk 上で **당근마켓（韓国の中古・地域マーケット）の中古品・アルバイト・不動産を自然言語で検索**できる MCP（読み取り専用）サービスです。アプリを開く前に**「近所の公開物件」**をメッセンジャー内で直接尋ねて見て回る流れで、作者はこれを**「メッセンジャーに MCP が入ると、生活型の検索はどう変わるか」**を探る**非公式の研究用ツール**だと明言しています（당근마켓との公式提携ではありません）。MCP が開発ツールを超えて**日常のメッセンジャーのインターフェース**へ広がると何が可能になるかを、軽やかに示す事例 — *エージェントがどこに住むか*を一段ずらしてみた発想です。[GeekNews](https://news.hada.io/topic?id=30852)
