## 主要な新機能と実践活用

### dynamic-workflow のトリガーが `ultracode` に変わりました（v2.1.160）

5/28 に導入された Dynamic Workflows は、プロンプトに「workflow」という単語が入ると自動実行され、ドキュメント作成やレビューのセッションで意図せずワークフローが起動する誤作動が頻発していました。v2.1.160 はこれを根本から解決し、**トリガーキーワードが `workflow` から `ultracode` に変更**されました。

- **「workflow」という単語はもう dynamic workflow をトリガーしません。** ドキュメント・コミットメッセージ・レビューコメントで気兼ねなく使えます。
- ワークフローを起動したいときは **`ultracode`** を使うか、自分の言葉でそのまま依頼すればOKです（「このコードベース全体を…に移行して」）。
- 入力欄ではトリガーキーワードが**紫色（violet）でハイライト**されるので、いまワークフローが発動しているかが一目で分かります。

5/30 のブリーフィングで紹介した「`/config` でキーワードトリガーをオフにする」回避策なしで、通常の文章で引っかかることがなくなりました。

[GitHub v2.1.160](https://github.com/anthropics/claude-code/releases/tag/v2.1.160)

### `grep` 一回で Edit 可能に — read-before-edit の緩和（v2.1.160）

これまで Claude はファイルを編集する前に、必ず `Read` でそのファイルを見たことがある必要がありました。v2.1.160 からは、**単一ファイル対象の `grep`/`egrep`/`fgrep` コマンドが read-before-edit の条件を満たします**。

つまり「特定のパターンを grep で探す → その場で修正」という流れで、途中に別の `Read` を挟む必要がなくなります。検索と編集を繰り返す作業で、往復を一回分省けます。

[GitHub v2.1.160](https://github.com/anthropics/claude-code/releases/tag/v2.1.160)

## 開発者ワークフローTips

### スキルが自動ロードされないとき — `UserPromptSubmit` フックで強制ルーティング

スキルは description に基づいて自動アクティベートされますが、実務では**必要な瞬間に肝心のスキルがロードされない**ことが少なくありません。これを決定論的に解決するのが `UserPromptSubmit` フックです。

核となる発想は、**Claude がプロンプトを見る前に、フックが先に適切なスキルを差し込む**ことです。

- `.claude/skills/skill-rules.json` にスキルごとの**キーワード + 正規表現の `intentPatterns`** を定義します。
- `UserPromptSubmit` フックが受け取ったプロンプトをルールと照合し、該当するスキルの推奨をプロンプトに追記します。
- 「Claude は忘れられない — そもそも覚える必要がないから」というのがこのパターンの一言サマリーです。
- 状態は `recommendation-log.json` に蓄積され、**7日後に自動でクリーンアップ**されるので、過剰な推奨なしにカバレッジを保てます。

advisory な `CLAUDE.md` や description ベースの自動アクティベートを**置き換えるのではなく補完する**、決定論的なルーティングです。

[claudefast — Skill Hook](https://claudefa.st/blog/tools/hooks/skill-activation-hook)

### トークンコストをチーム・リポジトリ別に分解 — `OTEL_RESOURCE_ATTRIBUTES`（v2.1.161）

v2.1.161 で、**`OTEL_RESOURCE_ATTRIBUTES` の値がメトリクスのデータポイントのラベルとして含まれる**ようになりました。これにより、使用量・コストのメトリクスを**チームやリポジトリといったカスタム次元でスライス**できます。

```bash
# 使用量メトリクスにチーム/リポジトリのラベルを付与
export OTEL_RESOURCE_ATTRIBUTES="team=payments,repo=checkout"
claude
```

追加の計装コードなしで、ダッシュボードをチーム別・コードベース別にフィルタリングできます。マルチリポジトリや組織規模の導入で「どこがトークンを消費しているか」を追うのに便利です。6/15 の Programmatic Usage Credits 施行を前に、コストガバナンスを整える際に特に実用的です。

[GitHub v2.1.161](https://github.com/anthropics/claude-code/releases/tag/v2.1.161)

## セキュリティ・制限事項

### v2.1.160・v2.1.161 のハードニング — コード実行経路の遮断・シークレットマスキング（6/2）

6月2日の2本のリリースに、**任意コード実行を防ぐ権限ゲート**と**シークレット露出の防止**が入りました。

**v2.1.160 — 書き込み前の確認を追加**:
- `.zshenv`・`.zlogin`・`.bash_login` といった **shell 起動ファイル**と `~/.config/git/` への書き込み前に確認を求めます。これらは意図しないコマンド実行につながり得るためです。
- `acceptEdits` モードでも、**コード実行権限を付与するビルド設定ファイル**（`.npmrc`, `.yarnrc*`, `bunfig.toml`, `.bazelrc`, `.pre-commit-config.yaml`, `.devcontainer/` など）への書き込み前に確認を求めます。

**v2.1.161 — `claude mcp` のシークレットマスキング**:
- `claude mcp` の list/get/add がターミナルにシークレットを出力していた問題が修正されました。`${VAR}` 参照は展開されなくなり、認証ヘッダーや URL 内のシークレットはマスキングされます。

### 6月2日の大規模障害 — IPO 申請の翌日（6/2）

Anthropic が秘密 IPO 申請書を提出した**まさに翌日**、6月2日の 06:00（UTC）頃から Claude に複数サービスの障害が発生しました。**Claude Code・Cowork・Claude.ai・API** が影響を受け、**Vertex AI・Bedrock 経由の顧客は影響を受けませんでした**。本番環境で Claude のアクセス経路を選ぶ際の信頼性の参考データになります。

一部メディアは、**Claude Code のサブエージェントシステムの無限ループ**でサブエージェントが過剰に増殖してトークンを暴走消費し、使用上限を静かに使い切った（"silent quota drain"）ことを原因として挙げています。Anthropic は影響を受けた Pro・Max 加入者に**緊急のクォータリセット**を適用し、16:18（UTC）頃に「完全復旧」を告げました。ただし The Register は障害の事実と時刻のみ確認しており、**サブエージェント原因説は公式には裏付けていない**ため、原因は「報じられているが未確定」と見るのが安全です。

[The Register](https://www.theregister.com/ai-and-ml/2026/06/02/claude_celebrates_anthropics_stock_market_float_with_blockbuster_outage/5250071) | [Storyboard18](https://www.storyboard18.com/digital/anthropics-claude-ai-suffers-disruption-what-caused-the-outage-and-why-user-quotas-were-reset-99951.htm)

### Flatt Security — Claude Code GitHub Actions のサプライチェーン脆弱性を公開（6/1）

GMO Flatt Security（研究者 Ryota K 氏）が、**Claude Code GitHub Actions 連携の脆弱性**の詳細分析を公開しました。核心の欠陥は `checkWritePermissions` が**実際の権限に関係なくすべての GitHub App を信頼**していた点で、攻撃者の App インストールトークンが信頼されたワークフロー入力として扱われ、**悪意ある GitHub の issue/PR ひとつだけ**で、そのワークフローを使うリポジトリを乗っ取れました（CVSS 7.8）。

**重要（タイミング）**: これらの欠陥は 2026年1月12日・2月17日に報告され、**すでにパッチ済み**です（権限バイパスは 1/16 に修正、完全な解消は約4月、GitHub Actions v1.0.94）。つまり新たなライブ脆弱性ではなく、**パッチ済みの欠陥の詳細公開**です。Claude Code GitHub Actions を使うチームは、**v1.0.94 以降**であることを確認するだけで済みます。

[GMO Flatt Security](https://flatt.tech/research/posts/poisoning-claude-code-one-github-issue-to-break-the-supply-chain/) | [GBHackers](https://gbhackers.com/claude-code-github-actions-flaw/)

## エコシステム＆プラグイン

### Anthropic 公式 `security-guidance` プラグイン — リアルタイム脆弱性レビュー（5/27）

Anthropic が、開発中のコードを自動でセキュリティ点検する**無料のファーストパーティ・プラグイン**をリリースしました。3つの層で動作します。

- **編集中の軽量パターン検査**: `eval()`・`os.system()`・`child_process.exec()` などの危険な呼び出しを即座にフラグ。
- **ターンごとのモデルベース git diff 分析**: 認証バイパス・インジェクション・暗号の弱点などをレビュー。
- **コミット/プッシュ時の詳細検証**。

社内ベンチマークでは**セキュリティ関連の PR コメントが 30〜40% 減少**したとのことです。**Claude Code v2.1.144+ と Python 3.8+** が必要で、`/plugins` からインストールします。先に取り上げた v2.1.153/154 のセキュリティ「修正」とは別の、**常駐型の防御ツール**です。

[Help Net Security](https://www.helpnetsecurity.com/2026/05/27/anthropic-claude-code-security-guidance-plugin/) | [Cybersecurity News](https://cybersecuritynews.com/free-security-plugin-for-claude-code/)

### Strava、Claude 向け MCP コネクタを公開（6/1）

フィットネスプラットフォームの Strava が**公式の Model Context Protocol コネクタ**を出しました。サブスクライバーが Strava アカウントを Claude に接続すると、アクティビティデータを対話的に分析できます。**秒単位の心拍・ペース、GPS ルート、サイクリングパワー、クラブ/イベントのメタデータ**を、OAuth 読み取り専用のリモート MCP サーバーとして公開します。Claude Web・Cowork（デスクトップ）・**Claude Code（ターミナル）** のいずれでも設定でき、Strava サブスクリプションに含まれ、段階的に展開中です。

[Let's Data Science](https://letsdatascience.com/news/strava-launches-mcp-connector-for-claude-integration-909e07a0)

### Claude Enterprise アクティベーションプロモ — Code・Cowork の初回メッセージで $1,000 クレジット（6/2）

Anthropic は、**従量課金（usage-based）の Claude Enterprise** 組織の各ユーザーが **Claude Code または Cowork で初回メッセージ**を送ると、**$1,000 の使用クレジット**を付与します（組織あたり最大 $10M、最大 10,000 席）。アクティベーション期間は **6/2〜7/2**、各クレジットは付与後 **90日** で失効し、管理者の操作なしで自動付与されます。レガシーの席ベース・Team・Pro・Max は対象外で、過去にこれらの製品でメッセージを送ったことのあるユーザーも対象外です。

## 知っておくと便利な小さな変更点

- **並列ツール呼び出しの分離**: 同一バッチ内の Bash コマンドが1つ失敗しても、同じバッチの他の呼び出しはキャンセルされず、それぞれが個別に結果を返します（v2.1.161）
- **`/mcp` で未使用コネクタを折りたたみ**: 一度もサインインしていない claude.ai コネクタを「Show unused connectors」の行の後ろにまとめます（v2.1.161）
- **`claude agents` の進捗表示**: 作業がファンアウトすると行に `done/total` が先に表示され、peek では最も時間のかかっている項目が表示されます（v2.1.161）
- **「Reduce motion」への対応**: `/effort` ダイアログ・ワークフローのアニメーション・プロンプトのキーワード shimmer が「Reduce motion」設定に従います（v2.1.161）
- **`CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE` の完全無効化**: 予告どおり削除され、いまや **no-op** です。この環境変数を使う設定が残っていれば整理しましょう（v2.1.160）
- **JetBrains の提案を削除**: 起動画面から JetBrains プラグインのインストール提案がなくなりました（v2.1.160）

## 注目プロジェクト＆ツール

- **AgentDir — エージェントのための mkdir**: 元のファイルを移動も変更もせずに、**タスクごとのフォルダ構成で見せる Rust 製の読み取り専用仮想ファイルシステム**です。AI エージェントを目的に合わせて作られたディレクトリ構造で作業させつつ、元ファイルへの編集は仮想ツリーに自動反映され、copy-on-write でストレージを節約します。macOS・Linux・Windows に対応し、Python・Node.js SDK を提供するので、既存のエージェントパイプラインに組み込みやすいです。[GitHub](https://github.com/nomadamas/agentdir)

- **oh-my-harness — AI コーディングエージェント向けガードレール生成 CLI**: プロジェクトを平文で説明すると（例:「React + FastAPI フルスタック、TDD 強制」）、**Claude Code・Codex が無視できないガードレール**を生成する CLI です。エージェントがよく破る `CLAUDE.md` を手書きする代わりに、**main ブランチへのコミット・テストのスキップ・保護ファイルの編集**といった危険な行為を実際にブロックする git フックとランタイムチェックを作り、自動 lint/format まで付けてくれます。ルールを「助言的なマークダウン」から「強制執行」へ移す点が興味深いです。[GitHub](https://github.com/kyu1204/oh-my-harness)
