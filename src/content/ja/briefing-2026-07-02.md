## 主要な新機能と実践活用

### Claude in Chrome が正式リリース(GA)— ブラウザエージェントがベータを卒業 (v2.1.198)

v2.1.198 の1行目は **Claude in Chrome の正式リリース(GA)**です。これまで有料プラン(Pro・Max・Team・Enterprise)に**ベータ**で開かれていたブラウザ拡張が、**正式サポートの機能**に昇格しました。Claude がサイドパネルで**あなたが見ている画面を一緒に見ながら**、ウェブサイトを読み・クリックし・フォームを埋め、タブをまたぐ**マルチステップのワークフロー**を実行します。

- **何が変わるか**: ベータの札が外れることは、*本番ワークフローに組み込んでよい合図*です — 反復作業を一度実演して録画すると、Claude がその手順を学習して繰り返す機能も含みます。
- **Claude Code との接点**: ターミナルの Claude Code がコードベースを扱うのに対し、Claude in Chrome は*ブラウザ上で起きる作業*(ダッシュボード操作・ウェブ管理コンソール・リサーチ)を自動化します — ローカルのコーディングとウェブ操作を一つのエージェントアカウントでまとめる構図です。

要点は、**エージェントが『住む場所』がターミナルを越えてブラウザへ広がり、それが実験ではなくサポート対象の機能になった**ことです。ただしブラウザエージェントは*ログイン済みセッションや機微なサイト*に直接触れるため、導入前に権限とアクセス範囲をポリシーで固めておきましょう。[Claude for Chrome](https://claude.com/claude-for-chrome) · [GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

### background エージェントが完了時に commit・push・draft PR を — Notification hook も (v2.1.198)

無人実行の観点で、今回の**最大の変化**です。`claude agents` から起動した **background エージェントが worktree でコード作業を終える**と、**止まって人に尋ねる代わりに、自分で commit・push して draft PR を開く**ようになりました — つまり*エージェントを投げておくと、結果がレビュー可能な PR の形で届く*わけです。

さらに **Notification hook** が加わりました — background セッションが**入力を必要とする(`agent_needs_input`)**、または**完了する(`agent_completed`)**と hook が発火します。

```jsonc
// .claude/settings.json — background エージェントの状態遷移を通知で受け取る
{
  "hooks": {
    "Notification": [
      { "matcher": "agent_needs_input", "hooks": [ /* デスクトップ/Telegram 通知スクリプト */ ] },
      { "matcher": "agent_completed",   "hooks": [ /* 例: PR リンクを Slack に送る */ ] }
    ]
  }
}
```

要点は、**「投げて忘れる(fire-and-forget)」フローがついに完結した**ことです — 以前は background エージェントが静かに終わったり途中で止まったりして、*いつ確認すべきか*を人がポーリングする必要がありました。今は終われば PR が残り、詰まれば通知が来ます。6/28 の **git worktree の並列セッション**、6/29 の**サブエージェントのコンテキスト衛生**と同じ流れで — *複数のエージェントを同時に無人で回す*運用をもう一段自動化した変更です。実際の配線は下のワークフローで扱います。[GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

### `/dataviz` スキル — チャート・ダッシュボードのデザインガイド + 実行可能なパレット検証ツール (v2.1.198)

v2.1.198 は新しい組み込みスキル **`/dataviz`** を追加しました — チャート・グラフ・ダッシュボードを作る際の**デザインガイド**を提供し、**実行可能なカラーパレット検証ツール(color-palette validator)**を同梱します。つまり可視化を描く前に、*アクセシビリティ・コントラスト・ライト/ダークの一貫性*をコードで検証するよう促します。

```text
# チャート/ダッシュボード/インフォグラフィックを作る前にスキルを呼び、デザインルールと検証ツールを適用
/dataviz
```

要点は、**AI が作る可視化の「generic AI look」をルールと検証で防ぐ**ことです — 適当な色を並べたダッシュボードではなく、パレットが実際にコントラストや色覚安全性を通過するかをスキルが点検します。今日のおすすめコラムにある Atlassian の **DESIGN.md**(AI スロップを防ぐ移植可能なデザインコンテキスト)とまさに同じ筋 — *デザインの意図を人の記憶ではなく検証可能な成果物として固定する*流れです。[GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

## 開発者ワークフローティップス

### 無人 background を『完結型』に配線する — 終われば draft PR、詰まれば通知

上の新機能を実際のパイプラインに落とすティップスです。v2.1.198 から background エージェントは**完了時に draft PR を残し**、**Notification hook で状態を外に出します**。この二つを組み合わせれば、*人がターミナルを見張らなくても*複数のエージェントを安全に回せます。

```bash
# 1) 作業を background に投げる(worktree で隔離実行)
claude --bg "このマイグレーションを実行してテストを通せ"

# 注意: --bg を --print/-p と併用すると attach 不能なセッションになるため、
#       v2.1.198 からその組み合わせは事前に拒否される(静かに壊れない)

# 2) Notification hook をデスクトップ/Telegram/Slack に配線しておくと、
#    agent_needs_input なら即介入でき、agent_completed なら PR リンクを受け取れる
```

要点は、**ポーリングを通知に、手動の仕上げを自動 PR に置き換える**ことです — 6/29 で扱った **idle watchdog の既定 ON**(止まったストリームを自動中断)と対になります。片方は*静かにハングする失敗*を防ぎ、もう片方は*静かに終わる完了*を通知として可視化します。無人パイプラインがあるなら、アップデート後に **`agent_needs_input`/`agent_completed` の二つのイベントを実際の通知チャネルに配線**しておきましょう。[GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

### Explore・サブエージェントに『より重い仕事』を任せる — 上位モデル・extended thinking を継承

二つ目は、**委譲の品質が静かに上がった**点を活かすティップスです。v2.1.198 は二つを変えました — ① 組み込みの **Explore エージェントが haiku ではなくメインセッションのモデル(opus 上限)を継承**し、② **サブエージェントとコンテキスト圧縮(compaction)がセッションの extended thinking 設定を継承**します。つまり委譲した作業が*より賢いモデルで、より深い思考予算で*動くようになりました。

```text
# 以前: Explore/サブエージェントが haiku 級で動き、調査・要約の品質が浅くなり得た
# v2.1.198: メインセッションのモデル(最大 opus)・extended thinking を継承し、委譲結果が深くなる
```

要点は、**コンテキスト衛生のためにサブエージェントへ委譲しても、品質の損失が減った**ことです — これまでは*メインを軽く保とうと委譲すると結果が浅くなる*トレードオフがありましたが、その幅が狭まりました。6/29 の**「サブエージェントはトークンを分割償却する」**と同じ筋で — *複数ファイルを読む調査・リファクタ・クロスファイル分析*のような重い作業も、安心してサブエージェントに押し込めます。[GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

## セキュリティ・制限事項

### Fable 5・Mythos 5 の輸出規制が解除 — Fable 5 が 7/1 に全世界復帰(ただし 7/7 まで週次上限50%で段階展開) (6/30)

数日追ってきた案件が**解決しました。** **6/30、商務長官 Howard Lutnick の命令で、6/12 から掛かっていた Fable 5・Mythos 5 の輸出規制が解除**され、**Fable 5 は 7/1 から全世界に復帰**しました — Claude Platform・Claude.ai・**Claude Code**・Claude Cowork の全体で再び使えます。約**3週間の凍結**が終わったのです(6/12 発動 → 6/30 解除)。

- **段階的展開**: 復帰は*一度に全面 ON* ではありません。**Pro・Max・Team・一部 Enterprise は 7/7 まで週次使用上限の最大50%**で Fable 5 を使い、その後は使用量クレジットが適用されます。**クラウド(AWS・Google Cloud・Microsoft Foundry)経由のアクセス**は*できるだけ早く*復元されます。
- **復帰の対価 — 4つの約束**: Anthropic は再展開の条件として、① **指定政府パートナーに公開前の独立評価のための拡大 early access** を提供、② **jailbreak 情報を迅速に共有し、政府テスト用の新しい安全策を展開**、③ **共同研究の専任チームを立ち上げ、相当な計算資源を割り当て**、④ **業界共通のセキュリティ標準に向けて協力**、を約束しました。
- **Mythos 5 は別枠**: サイバーセキュリティモデルの Mythos 5 は(6/26 の承認により)**一部の米国企業**にのみ復元され、Anthropic は **Project Glasswing** でアクセスを拡大中です — つまり広い復帰は Fable 5 に限られます。

**外国籍の開発者にとって、今日の実務的含意はこの3週間とは正反対です。** ブリーフィングが繰り返し勧めてきた*「Fable 5 は塞がれているから Opus 4.8 のような代替経路を用意せよ」*という前提は**ひっくり返りました** — Fable 5 のアクセスが(7/7 まで50%上限のランプを経て)再び開きました。ただし教訓は残ります: **モデルの可用性が会社ではなく地政学・政策に左右され得る**ことが今回実証されたので、`fallbackModel`・`availableModels` で代替経路を維持する習慣はそのまま残しておくのが安全です。[Redeploying Claude Fable 5](https://www.anthropic.com/news/redeploying-fable-5) · [CNBC](https://www.cnbc.com/2026/06/30/anthropic-says-trump-admin-has-lifted-export-controls-on-claude-fable-5-and-mythos-5.html) · [Al Jazeera](https://www.aljazeera.com/economy/2026/7/1/us-lifts-restrictions-on-powerful-ai-models-fable-mythos-anthropic-says)

## エコシステム＆プラグイン

### Gateway に Claude Platform on AWS 追加 — failover チェーンがより堅牢に (v2.1.198)

複数プロバイダにまたがってモデルをルーティングする人に届く変更です。v2.1.198 は Claude Code **Gateway のアップストリームプロバイダとして Claude Platform on AWS(`anthropicAws`)を追加**し、あわせて **`model-not-found` 応答が failover チェーンを進行**させるよう修正しました — あるプロバイダで当該モデルが見つからなければ、次の候補へ移ります。

要点は、**モデルの可用性が揺れても自動で次の経路へ落ちる回復力**です。ちょうど今日のセキュリティ節(輸出規制)で見たように*特定経路のモデルが突然塞がれ得る*時代に、`model-not-found` を失敗ではなく*failover のシグナル*として扱うのは実践的です。複数のクラウド・プロバイダを Gateway でまとめて使うチームは、**AWS を候補に加え、failover の順序を再点検**しておきましょう。同時に入った **AWS STS トークン失効時の `awsAuthRefresh` 自動実行**の修正(失効で `Please run /login` に詰まっていた問題を解消)も同じ筋です。[GitHub v2.1.198](https://github.com/anthropics/claude-code/releases/tag/v2.1.198)

## コミュニティニュース

- **Fable 5 の復帰は『前例のない政府協力の条件』付きで来た — フロンティアモデル・ガバナンスの新しい型**: 今回の復帰の本当のニュースは*日付*ではなく*条件*です。Fable 5 は単に点いたのではなく、Anthropic が政府に**公開前の独立評価・jailbreak 情報の共有・共同研究・業界共通のセキュリティ標準**を約束する見返りに戻ってきました(上のセキュリティ節)。引き金は **Amazon の研究者が報告した Fable 5 の安全策のバイパス(jailbreak)**で、政府はこれを根拠に 6/12 の規制を掛け、約3週間後に解除しました。業界的な含意は — **商用フロンティアモデルの展開が、いまや『企業の判断』だけでなく『政府との継続的な協力・報告の体制』と絡み始めた**ことです。6/28 の「アジアの代替モデル」、6/29 の「ハーネスがモデルより重要」と同じ位置で — *どのモデルを使うか*と同じくらい、*そのモデルがどんなガバナンス・地政学の上に立つか*が採用を分ける変数になりました。[The Hacker News](https://thehackernews.com/2026/07/anthropic-restores-claude-fable-5-after.html) · [CoinDesk](https://www.coindesk.com/tech/2026/07/01/anthropic-restores-ai-models-fable-mythos-after-the-u-s-lifts-export-controls)
- **Claude Partner Network — Services Track + Partner Hub 拡張(7/1)**: Anthropic が **Claude Partner Network** に二つを追加しました — **コンサル企業を『Claude を顧客にどれだけ深く展開したか』で格付けする Services Track** と、企業が**有資格の実装パートナーを探す Partner Hub ポータル**です。昇格は**年2回(1/1・7/1)**処理され、今年は **10/1 の追加審査**が一度あります。要点は、Claude を**エンタープライズにより深く押し込む**チャネル戦略です — 個人開発者のツール(Claude Code)を越えて、*組織単位の導入を代行・検証するパートナーエコシステム*を公式化する動きです。導入コンサル・実装パートナーを探す企業チームには参考になります。[Claude Partner Network](https://www.anthropic.com/news/services-track-partner-hub)

## 知っておくと便利な小さな変更点

v2.1.198 で上で扱いきれなかった実用的な変更・修正です。

- **瞬断リトライ**: 応答の途中で起きる短いネットワーク断(`ECONNRESET` などの一時的エラー)がターンをまるごと中断していた問題を修正 — 今は backoff で**リトライ**します (v2.1.198)
- **`/agents` ウィザード削除**: サブエージェントは Claude に頼んで作るか、`.claude/agents/` を直接編集します (v2.1.198)
- **plan mode 起動時に read-only を自動許可**: plan mode でセッションを始めたとき、読み取り専用ツール呼び出しを自動許可しなかった問題を修正 (v2.1.198)
- **agent teams の堅牢化**: API エラーで死んだ teammate が lead に `failed` と報告し、止まった teammate にメッセージすると即座に起きてリトライします (v2.1.198)
- **background タスクが完了後も `Running` に留まる問題を修正**: web・desktop・VS Code のタスクパネルで、完了後やセッション再開後も `Running` のまま止まっていた問題を解消 (v2.1.198)
- **`/branch` の既定名**: 既定の fork 名を圧縮要約ではなく**最初の実プロンプト**から導きます (v2.1.198)
- **構文ハイライトの精度改善**: コードブロック・diff・ファイルプレビューを **highlight.js 11** にアップグレード (v2.1.198)

## おすすめコラム＆読み物

- **『Revised Rules of Engineering Leadership』(Will Larson)**: ハイパーグロースと AI ツール導入を1年余り経て*再調整した*エンジニアリング・リーダーシップの5原則です。鋭い観察がいくつも — **個々のエンジニアがいまや以前の10分の1の時間でマイグレーションを終える**こと、*「動くコード」のコスト*は初期実装よりも**開発インフラ(テスト・CI・レビューのパイプライン)**に左右されること、組織は意思決定を**「速く・良く・堅牢に(fast, good, and robust)」**保ち、**ドメイン知識を持つ安定した高オーナーシップのチーム**を守るときに最も強い、という指摘。AI が実装を安くするほど、*どう決め、どんなチーム構造で支えるか*がリーダーシップの中心に上がってくる、という筋で読めます。[出典](https://lethain.com/revised-rules-of-engineering-leadership/)
- **『Atlassian の DESIGN.md — 移植可能なデザインコンテキストを実地テストして学んだこと』**: 今日の `/dataviz` スキルとまさに共鳴する記事です。**AI が生成する UI がブランドの独自性を失う(AI スロップ)問題**を解くため、Atlassian はデザインシステムの*意図*を**移植可能な Markdown 形式(DESIGN.md)**にエンコードしました。実地テストの正直な結論が価値です — この形式は生成されたダッシュボードをブランドに合わせられましたが、**MCP サーバー方式に比べてトークンを約92%多く消費**するトレードオフがありました。代わりに **クロスプラットフォームの移植性・顧客テーマ・慣れない環境での素早いプロトタイピング**には強かった。教訓は — *デザインコンテキストをどこに置くか(ファイル vs MCP)が、コストと移植性の選択そのもの*であること、そして 6/29 の「CLAUDE.md は短く」と同じ位置で、*どれだけのコンテキストをどこに載せるか*が性能とコストを分ける、という点です。[出典](https://www.atlassian.com/blog/ai-at-work/atlassians-design-md-is-here-what-we-learned-testing-portable-design-context-in-practice)
- **『We Are the Last Generation That Knows How It Works』(私たちは仕組みを知る最後の世代)**: 1990年代のコンピューティングは **`autoexec.bat` の編集・ジャンパ設定・割り込みのトラブルシュート**といった*機械の制約と直接ぶつかる摩擦*を要求し、その**摩擦が本物の理解を生んだ**という省察です。逆に今日の AI アシスタントは*ユーザーの要求に抵抗なく適応*し、かつて技術リテラシーを生んでいたその*格闘*を取り除いてしまう、と診断します — 知識の*保存*は良くなったが、敵対的な問題解決を通じて積み上がる*親しみ(familiarity)*の喪失は、世代が技術と関わる仕方の意味ある変化だ、と。昨日の CUDA カーネル解剖の記事と同じ筋で — *AI が下層を安く処理してくれるほど、その下で何が起きているかを知る能力*が何を意味するのかを問い直させる一編です。[出典](https://unix.foo/posts/last-people-who-know-how-it-works/)

## 注目プロジェクト＆ツール

- **CTX — Claude Code の『セッション間の永続メモリ』(オープンソース)**: セッションごとに文脈を説明し直す摩擦を正面から解くツールです。CTX は**会話履歴・決定・ファイル参照をセッションを越えて自動で保存**し、Claude Code の **`UserPromptSubmit` hook に接続**して、過去の文脈を **BM25 + ベクトルのハイブリッド検索**で低遅延に呼び戻します — つまり*手動で文脈を再ロードしなくても*連続性が保たれます。今日の v2.1.198(サブエージェント・圧縮がセッション設定を継承)、6/30 の Framein(マルチエージェントの共有状態)と同じ位置で — *エージェントの記憶を repo・ローカルに固定しよう*とする最新の試みです。[GitHub](https://github.com/jaytoone/CTX)
- **Branch of Thought — Claude・ChatGPT の会話の『隠れた分岐』をグラフで可視化(Chrome 拡張)**: 長い AI 会話で*どこで話が枝分かれしたか*をサイドパネルに**グラフで描く**拡張です。現在の会話パスを強調し、**代替の分岐をワンクリックで移動**でき、ノードをドラッグして配置を変えたり、グラフを **HTML・PNG・SVG で書き出し**たりできます。Claude と ChatGPT の異なるデータ構造を**正規化レイヤー**で吸収してプラットフォーム非依存に動作し、**すべての処理はローカル**で行われます(任意で自分の API キーを使う Claude Haiku 要約機能あり)。Web ストアではなく**展開済みのコードで配布**して透明性を選んだ点も特徴です — *エージェントとの長い会話を辿り直して管理する*人に実用的です。[GitHub](https://github.com/ijzereen/branch-of-thought)
