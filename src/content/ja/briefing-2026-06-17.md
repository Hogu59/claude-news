## 開発者ワークフローティップス

### Claude Code チームのデザイナーの流儀 — 最初の答えを受け取らず「選んで理由を述べさせる」

Claude Code・Cowork の **Head of Design、Meaghan Choi** が実際の作業方法を公開しました（今週、開発者コミュニティで紹介）。核心は **エージェントの最初の出力をそのまま受け取らないこと**です。自作の **カスタム `/prototype` スキル**で同じ画面の**複数の実装案を HTML プレビュー**として生成させ、その上で Claude に **「最良の案を選び、その理由を説明する（select and justify）」** よう指示してから進めます。

並列セットアップも参考になります — **worktree** で複数の Claude セッションをファイル競合なく同時に走らせ、**auto モード**で繰り返しの承認プロンプトをスキップし、Opus の 1M コンテキストを fast モードで使います。より重要なのは姿勢です — 「**誰でもビルドできるからといって、すべてをデプロイすべきとは限らない。LLM はいまだにデザイン判断が苦手だ**」と述べ、実行は委ねつつ **判断は人間が握る**構造を強調します。漠然と「実装して」ではなく、**選択肢を広げさせ、その一つを根拠とともに選ばせる**という一手が品質を分けます。[YouTube](https://www.youtube.com/watch?v=hKeDfupbA4U)

### エージェントにインフラを直接渡さず「Git PR ゲート」で縛る

あるホームラボ運用者が AI でインフラを管理しながらまとめたパターンが、そのまま実務レシピになります — **AI に本番アクセスを直接与えず**、変更は必ず **Git の feature branch にのみ push** させ、**人間が PR をレビュー**し、承認されたら **自動デプロイ**が走る構造です（OpenCode + GitOps）。AI は **feature branch にしか書き込めず、本番には触れません**。

要点はシンプルです — エージェントのミスのコストが大きい場所（デプロイ・インフラ）ほど、**「AI が提案し人間が承認する」というゲートをツールに強制させる**ことです。この筆者はこの構造でコンテナ更新を数時間から数分に短縮し（リリースノートの自動要約・ヘルスチェック改善）、**未レビューのコードが本番に到達しない**ようにしました。6/15 に取り上げた「after-action フックはブロックせず非ブロックのフィードバックだけ」と同じ筋 — エージェントは速く回しつつ、**取り返しのつかない地点にだけ**人間ゲートを置く設計です。[rsgm.dev](https://rsgm.dev/post/ai-dev-platform/)

## セキュリティ・制限事項

### 輸出規制の余波 — 同盟国の「キルスイッチ」懸念と代替モデルへの殺到 (6/16)

6/12 の **Fable 5・Mythos 5 の回収**が、いまや **同盟国の AI 主権**の問題へと発展しました。Bloomberg と Japan Times によると、米政府が外国籍者の両モデルへのアクセスを一方的に遮断したことで、欧州で「大統領のキルスイッチ」として **理論上の懸念にとどまっていた米国技術依存リスク**が**現実**になりました。反応は即座でした。

- **フランス**: Sébastien Lecornu 首相が、公務員向けツールを自国スタートアップ **Mistral AI ベース**に切り替えると発表しました。
- **Cohere**（Anthropic の小規模な競合）は、規制以降 **問い合わせが「桁違いに」増えた**と明かしています。

Anthropic の上級技術スタッフは、国家安全保障上の懸念について商務省の当局者と協議しています。開発者にとっての要点は変わりませんが、より鮮明になりました — **モデルの可用性が、ベンダーではなく地政学に左右されうる**ということです。韓国・日本などの外国籍開発者は両モデルへのアクセスが断たれた状態なので、`fallbackModel`・`availableModels` で Opus 4.8 のような代替経路をあらかじめ用意しておくのが引き続き実務的な対応です。[Japan Times](https://www.japantimes.co.jp/business/2026/06/16/anthropic-crackdown-ai-alarm/)

### Fable 5 規制の実際の発端は「脱獄」ではなく「fix this code」の三語だった (6/15)

今回の回収を引き起こした第三者の研究論文を **外部で唯一読んだ専門家**である Luta Security CEO の **Katie Moussouris** が、その中身を明かしました。精巧な jailbreak ではありませんでした — 研究者が **既知の CVE と意図的な脆弱性を含むコード**を Fable 5 に与えてセキュリティレビューを求めたところ、モデルが拒否したため、**「fix this code（このコードを直して）」** と言い換えたのです。すると、モデルはこれに応じて **パッチを生成し、追加のプロンプトでパッチを検証するテストスクリプトまで** 作成したといいます。

Moussouris は、これは **セキュリティ専門家が毎日行う正常な防御作業** — 「見つけ・直し・テストする（find-fix-test）ループ」 — にすぎず、**ガードレールの回避ではない**と断言します。**100 人を超えるセキュリティ専門家**が公開書簡でこの措置の撤回を求めています。防御側は脆弱性を見つけて直すために AI を必要とするのに、同じ能力が攻撃にも使われうるという逆説の上で — 今回の制限は **攻撃者よりも防御側に大きな打撃**だ、というのがセキュリティコミュニティの主張です（他国のオープンウェイトの競合モデルが、まもなく同等の能力に達するため）。モデルの安全性と、ありふれたコードセキュリティ自動化の境界はどこにあるのかを、改めて問う出来事です。[The Register](https://www.theregister.com/security/2026/06/15/feds-freaked-over-fable-5-after-simple-fix-this-code-prompt-not-jailbreak-says-researcher/)

## 知っておくと便利な小さな変更点

昨夜の **v2.1.179** の修正です（すべてバグ修正）。

- **接続切断時に部分応答を保持**: ストリーミング中に接続が切れても raw エラーではなく **受信済みの部分応答が残り**、スピナーが「running tool」で固まらなくなりました (v2.1.179)
- **[WSL2] マウスホイールスクロール修正**: Windows Terminal・VS Code 上の WSL2 でホイールスクロールが効かなかった **2.1.172 の回帰**が修正されました (v2.1.179)
- **[Linux] sandbox glob 肥大化修正**: `denyRead`/`allowRead` の glob を **大きなディレクトリツリー**に掛けると Bash ツールの説明が巨大化してセッションが使えなくなる問題が修正されました (v2.1.179)
- **プロモバナー重複表示修正**: ウェルカム画面に複数のプロモバナーが積み重なる現象を **セッションごとに最大1つ**に制限しました (v2.1.179)
- **フィードバック調査の誤作動修正**: ターン終了直後に入力した **1桁の数字の返答**を、セッション評価として誤って取り込んでいた問題が修正されました (v2.1.179)
- **Ctrl+O サブエージェント transcript 修正**: サブエージェント表示時に Ctrl+O で transcript が出なかった問題、プロンプト入力をクリックしても **フォーカスが戻らなかった**問題が修正されました (v2.1.179)
- **リモートセッションのバックグラウンドタスク固着修正**: ターン間でタスクが「still running」のまま固まって見える問題が修正され、**リモートセッションのプラグイン読み込み性能**が改善されました (v2.1.179)

## 注目プロジェクト＆ツール

- **token-router — クラウド LLM のトークンを約99%削減するローカルルーター**: 大きなファイルを扱う際のトークンコストを **最大99%**削減するハイブリッドコンテキストルーターです（「ローカル探索は無料、クラウド推論は高性能」）。① **ローカルの Ollama + Gemma 4 2B** がクエリに合う **関連行番号**をまず抽出し、② その座標どおりに **原文の断片だけをディスクから切り出し**、③ **ノイズを除去した断片だけをクラウドモデル**（Claude・Codex など）に送ります。インフラログ 2,000 行を 41,711 → 131 トークン（99.69%減）、レガシーコード 2,155 行を 7,520 → 70 トークン（99.06%減）にした例を挙げます。`error_log`・`heavy_code`・`agent_context` の3モード、MIT ライセンス。プログラム的利用が別計測になったいま、「どこまでローカルで絞り、何をクラウドに送るか」を扱う実用ツールです。[GitHub](https://github.com/sleeplesshan/token-router)
- **CodeBurn — 25種の AI コーディングツールの支出をローカルで一望**: Claude Code・Cursor・Copilot・JetBrains・Gemini CLI・Cline・Devin など **25種の AI コーディングツールのトークン使用量・コスト・性能を一箇所で**表示するオープンソースです。ラッパー・プロキシ・API キーのアップロードなしに **すべてローカルで**動作し、ディスク上のセッションデータを直接読み取って **LiteLLM でコストを計算**します。タスク種別・モデル・ツール・プロジェクト・プロバイダー別に支出を分解でき、**CLI ダッシュボード・macOS メニューバーアプリ・MCP サーバー**（エージェントがコストを問い合わせ）まで提供します（MIT）。6/15 のクレジット分離以降、「どの自動化がどこでクレジットを消費しているか」を直接覗くのに最適なツールです。[codeburn.app](https://codeburn.app)
