## 開発者ワークフローティップス

### 大きな作業は Plan モードで「決定」を先に終わらせる（Shift+Tab）

エージェントにいきなり「実装して」と投げる代わりに、**Shift+Tab** で Plan モードに入り、まず計画を立てさせるパターンです。要点は単なる「考えてから動く」ではなく、**曖昧な決定を、レビュー済みのスペックとして先に固定する**ことにあります。何を・どこに・どう作るかを実装前に人間が一度確定しておけば、実行段階でエージェントが毎回推測で分岐していたポイントが消え、各決定の正確さがほぼ100%に収束します（すでに人間が答えを決めているからです）。

本日から自動化の利用が別建てクレジットで計測される以上（下のセキュリティ参照）、これはコストにも直結します。計画なしに突っ込んだエージェントが誤った方向へ長々とトークンを燃やして戻ってくる事態を、**5分の plan レビュー**が防ぎます。曖昧な指示ほど、Plan モードでスペックへ絞り込んでから実行させましょう。[smart-webtech](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/)

### after-action フックは「ブロック」ではなく「フィードバック」に徹する

6/13 に扱ったパス限定フック（`Edit(src/**)` など）を実戦投入する際の注意点です。ファイル編集後に linter・formatter を走らせる **after-action フックは、結果をノンブロッキングでモデルへ返す**用途に限り、**作業の途中で書き込み（write）を止めない**のが賢明です。Claude が複数ステップにまたがる推論をしている最中にフックが書き込みを遮ると、エージェントが「今どのステップだったか」を見失い、多段階の作業がまるごと崩れてしまうからです。

ルール化すると——フォーマット・lint の出力は**編集後に自動でモデルへ食わせ**（ノンブロッキングのフィードバック）次の判断に反映させ、「これは絶対ダメ」という類のハードなブロックは、`.env` の読み取り拒否のように**明確に危険な事前（PreToolUse）の地点**だけに掛けます。after-action 段階で重く止めるほど、エージェントの多段階作業は壊れやすくなります。[Builder.io](https://www.builder.io/blog/claude-code-tips-best-practices)

## セキュリティ・制限事項

### 輸出規制、今度は「責任論争」へ — 「Anthropic の不注意」vs 反論（6/14 報道）

昨日扱った **Fable 5・Mythos 5 の全面回収（6/12）**が、**政治的な責任論争**に発展しました。トランプ政権の高官は、今回の輸出規制が **「Anthropic の不注意（recklessness）」**によって引き起こされたと主張しています——会社が連邦政府の初期の懸念を軽く扱い、問題修正に真剣でなかったため、政府との信頼が損なわれた、というものです。政府側は、**Amazon を含む6つのテスト機関**が Fable 5 のサイバー攻撃制約を回避する手法（jailbreak）を見つけ、これは「その能力は十分に制限している」という Anthropic の主張と矛盾すると述べています。

Anthropic はこの「不注意」という筋書きを真っ向から否定します——経営陣は**15分以内に連絡が取れる**状態で、Dario Amodei がウェルネス・リトリートに行っていたという報道も事実ではない、というものです。両者は初期接触以降、**毎日ビデオ会議**を続けており、Anthropic のシニア技術スタッフがホワイトハウス関係者と面会しています。

開発者にとっての要点は変わりません——**モデルの利用可否が、ベンダーではなく政府・政治の手に左右されうる**ことが一段と明確になりました。韓国・日本など外国籍の開発者は両モデルへのアクセスが断たれた状態なので、`fallbackModel`・`availableModels` で Opus 4.8 のような代替モデルへ安全にフォールバックするよう設定を点検しておくのが、引き続きの実務対応です。[Fox Business](https://www.foxbusiness.com/politics/trump-admin-says-anthropics-recklessness-triggered-export-controls-latest-ai-models)

### 本日施行：Programmatic Usage Credits 発効（6/15）

数日カウントダウンしてきた **Programmatic Usage Credits** が、**本日（6/15）発効**しました。これより **Claude Agent SDK・`claude -p`・Claude Code GitHub Actions・サードパーティエージェント**は、サブスク枠ではなく**別建ての月次クレジット（フルAPIレート・繰り越しなし）**から消費されます（Pro **$20**・Max5x **$100**・Max20x **$200**）。**対話型ターミナルの Claude Code・チャット・Cowork はこれまで通り**です。

2つのメーターが実際に分離された以上、**overflow 課金を有効にしていなければ、クレジット枯渇時に自動化が停止する**可能性があります。最後のチェックリストはシンプルです——`/usage` でどの自動化がどのモデルでクレジットを燃やしているかを確認し、overflow ポリシーを**本日中に**意識的に決めておきましょう。[The Decoder](https://the-decoder.com/claude-subscriptions-get-separate-budgets-for-programmatic-use-billed-at-full-api-prices/)

## エコシステム＆プラグイン

### オリジナル Claude 4 モデル、本日 API から引退（6/15）

本日 6/15 には**もう1つの締め切り**が重なります——**オリジナルの Claude 4 モデル2種が Claude API から引退**します。対象は2025年5月版の **`claude-opus-4-20250514`** と **`claude-sonnet-4-20250514`** で、現行の Opus 4.8・Sonnet 4.6 ではなく、その**第一世代**です。**猶予期間なし**で、午前9時（PT）から両モデルIDへのリクエストは **404** で失敗します。

注意すべきは、**Agent SDK・自動化が、引退したモデル文字列を後継へ自動エイリアスしない**点です。スクリプト・CI・`availableModels`・サブエージェントの `model:`・GitHub Actions の設定にこの2つのIDを固定していれば、**本日そのまま壊れます**。移行先はそれぞれ **Sonnet 4.6・Opus 4.8** です。

```bash
# 設定・スクリプトに残った旧モデルIDをまず点検
grep -rn "claude-opus-4-20250514\|claude-sonnet-4-20250514" .
```

上のクレジット施行と重なり、本日は**固定したモデルIDを一気に点検する好機**です——コストの帰属（クレジット）と利用可否（引退）をまとめて整理しておきましょう。[ChatForest](https://chatforest.com/guides/anthropic-claude-sonnet-4-opus-4-deprecation-june-15-2026/)

## 注目プロジェクト＆ツール

- **Electronic Clam — 蓋を閉じても AI エージェントを動かし続ける macOS アプリ**: クラムシェルモード（ノートを閉じた状態）でもエージェントの作業が止まらないようにするメニューバーアプリです。ターミナルコマンドや再認証なしにディスプレイスリープをトグルで防ぎ、**バッテリーと温度を監視してしきい値を超えるとタスクを一時停止**し、トランスクリプトの更新時刻を見て**実際に作業中か idle かを区別**します。**Claude Code・Codex・Cursor・opencode・Antigravity** を認識し、SSH・画面共有・Tailscale セッションの間も起きたままに保ちます（Swift/AppKit、Apple Silicon・macOS 13+、Developer ID 署名、オープンソース）。長時間の無人エージェントを回す人にうってつけの実用ツールです。[GitHub](https://github.com/jadhvank/eclam)
- **Grit — エージェントで Git を Rust に書き直す**: Git をコマンド群ではなく**再入可能でリンク可能なライブラリ**として、ゼロから Rust で再実装したプロジェクトです。Git のテストスイートで **99% 通過（42,001 件中 41,715 件）**を達成していますが、まだ本番向けではありません。面白いのは作り方——**Claude Code・Cursor のクラウドエージェント・動的ワークフロー**を大量に使い、約 **$10,000〜15,000・450億トークン**で **36万行・500超の PR** を生み出しました。上の Boris Cherny の「数万のエージェント」が、実際のコードベース規模でどう現れるかを示す事例です（現状は性能・API の安定性・Windows 非対応などの限界も明記）。[GitButler ブログ](https://blog.gitbutler.com/true-grit)
