## 主要な新機能と実践活用

### v2.1.140 — subagent_type柔軟マッチング、/goal安定性改善、プラグイン診断強化 (5/12)

v2.1.140は、v2.1.139の大規模機能追加に続く安定性と使いやすさの改善アップデートです。

**subagent_type柔軟マッチング**: Agentツールで`subagent_type`を指定する際、大文字小文字とセパレータ（ハイフン、アンダースコア、スペース）を無視してマッチングします。`"Code Reviewer"`、`"code-reviewer"`、`"code_reviewer"`のいずれも同じ`code-reviewer`エージェントに解決されます。CLAUDE.mdやプロンプトでエージェント名の正確な形式を覚える必要がなくなりました。

**`/goal`フック無効時の修正**: `disableAllHooks`または`allowManagedHooksOnly`が設定された環境で`/goal`が無限待機していた問題が修正されました。解決しないインジケーターの代わりに明確なメッセージを表示します。

**プラグインフォルダ警告**: `plugin.json`で明示的にキーを設定した場合、デフォルトコンポーネントフォルダ（例：`commands/`）がサイレントに無視される状況を`/doctor`、`claude plugin list`、`/plugin`で警告します。プラグイン開発時の「なぜコマンドがロードされないのか」というデバッグ時間を削減します。

**バックグラウンドサービス安定性**: `claude --bg`実行時にバックグラウンドサービスのidle-exit直前に"connection dropped"エラーが発生していた問題と、エンタープライズエンドポイントセキュリティソフトウェアによる起動失敗が修正されました。

## 開発者ワークフローティップス

### Planモードを「リサーチフェーズ」として分離活用する

最近のClaude Codeベストプラクティスガイドが共通して強調するパターンは、**リサーチ・計画と実装の明確な分離**です。Planモードでコードベースを探索しアプローチを設計した後、実装フェーズに移行する2段階ワークフローです。

**なぜ効果的か**: Planモードなしで直接実装に入ると、Claudeが「間違った問題を自信を持って20分間解いてから」やり直す状況が発生します。計画を先に立てれば、コード作成前に誤った前提を発見でき、計画の修正は半完成の機能を巻き戻すよりはるかに低コストです。

**適用基準**:
- 複数ファイルの変更 → Planモード必須
- 不慣れなコードベース → Planモード必須
- アーキテクチャ決定が必要な作業 → Planモード必須
- 単一ファイルのバグ修正 → 直接実装可能

[Claude Code Best Practices](https://code.claude.com/docs/en/best-practices) | [Smart WebTech](https://smart-webtech.com/blog/claude-code-workflows-and-best-practices/)

### 並列エージェントセッション + Agent Viewでマルチタスク管理

v2.1.139の`claude agents`コマンドと組み合わせると、4〜5個のClaudeセッションを別々のgitブランチで並列実行するチームワークフローが可能になります。成功の鍵は**各セッションに明確な計画を先に提供すること**です。計画なしの並列エージェントは「並列混乱」を生むだけです。

```bash
# Agent viewで全セッション状態を確認
claude agents

# 各セッションは独立したworktreeで実行
# settings.jsonでworktree基準点を設定
```

```json
{
  "worktree": {
    "baseRef": "head"
  }
}
```
