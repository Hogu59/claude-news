## 主要な新機能と実践活用

### エンタープライズクラウドでauto modeがopt-inなしに有効化、デフォルトモデルがOpus 4.8に (v2.1.207)

v2.1.207は、**Amazon Bedrock・Google Cloud Vertex AI・Foundry**を使うチームを狙ったリリースです。変化は2つあります。

- **auto modeに明示的なopt-inが不要に**: これまでauto mode(権限プロンプトなしにブランチを切り、ファイルを編集し、テストを走らせ、PRまで開く無人実行モード)は、`CLAUDE_CODE_ENABLE_AUTO_MODE=1`で明示的に有効化する必要がありました。この3クラウドでは今後、**環境変数なしでauto modeを使え、不要なら`disableAutoMode`で無効化**します。参入のあり方がopt-inからopt-outへ反転したわけです。
- **デフォルトモデルがOpus 4.8に**: Bedrock・Vertex・AWS上のClaude Platformが、今後**デフォルトでClaude Opus 4.8**を使います。

```bash
# Bedrock/Vertex/Foundry: CLAUDE_CODE_ENABLE_AUTO_MODE なしでも auto mode が利用可能に
# (不要ならリリースノートが示す disableAutoMode 設定で無効化する)

# 新しいデフォルト(Opus 4.8)ではなく安価なティアが必要なら明示的に固定する
claude --model <安価なモデル>
```

要点は、**無人のコーディングエージェントが、すでに承認済みのクラウド境界の内側に入ってきた**ことです。法務・セキュリティチームが本番ワークロード向けにBedrockやVertexを承認済みなら、*データが境界を越える*という最も一般的な反論は消えます。ただし裏を返せば、**誰も明示的に有効化していないのにauto modeが利用可能になる**状況が生じ得るため(下のワークフローティップス参照)、エンタープライズ配備なら本日一度、姿勢を点検する価値があります。 [GitHub v2.1.207](https://github.com/anthropics/claude-code/releases/tag/v2.1.207)

## 開発者ワークフローティップス

### 無人auto-modeの「安全姿勢」を再点検する — これは有効化ではなく無効化の問題になった (v2.1.207)

v2.1.207でauto modeがBedrock・Vertex・Foundryで**default-on(opt-out)**になったことで、*auto modeを使うか*は有効化の問題ではなく、**無効化するかを選ぶ問題**になりました。これらのクラウドで無人パイプラインを回すなら、そのままチェックリストです。

- **まず意図しない有効化を確認**: チームがauto modeを明示的に承認していないなら、`disableAutoMode`で切っておき、パイロット範囲でのみ有効化しましょう。業界の評価も *構造化パイロットには適するが、無管理の広範なロールアウトにはまだ* という線を引いています。
- **v2.1.205の整合性ガードと重ねて見る**: auto modeを有効化するなら、7/9のブリーフィングで扱った**偽承認防止・トランスクリプト改ざんブロック・未解決変数への`rm -rf`ガード**がそのまま防衛線になります。独自の承認・監査ロジックを載せているなら、二重にならないか点検してください。
- **カスタムランチャー保護**: v2.1.207は、**自動アップデータが`~/.local/bin/claude`のカスタムランチャースクリプトを上書きする問題**を修正しました。`claude`をシェルラッパーで包んで無人セッションを起動しているチームなら、アップデート時にラッパーが消えていた原因不明の事故はここに由来するかもしれません。

要点は、**デフォルトが安全側から便利側へ動いたので、安全はいまや明示的に守るもの**になった、ということです。7/9の整合性ガードが*実行の安全*を支えたなら、今回のdefault-onは*その安全を有効にするか否か*をチームの判断へ戻したことになります。 [GitHub v2.1.207](https://github.com/anthropics/claude-code/releases/tag/v2.1.207)

### クラウドのデフォルトモデルがOpus 4.8に変わった — コスト・挙動の基準を測り直す (v2.1.207)

同じリリースで、**Bedrock・Vertex・AWSのデフォルトモデルがOpus 4.8**になりました。デフォルトに頼って無人バッチや評価を回すチームにとっては、*昨日と同じコマンドが、今日は別のモデルで*実行され得るということです。

- **コスト・レイテンシを測り直す**: Opus 4.8はピーク性能のモデルです。大量の反復作業にデフォルトのまま当たると、トークンコストが想定外に跳ねます。日常作業は安価なティアに明示的に固定し、Opus 4.8は難しい問題に温存する2段構成は依然有効です。
- **挙動の回帰を点検**: モデルが変わればプロンプトやツール利用の癖も微妙に変わります。パイプラインの代表的なタスクでred→greenをもう一度回し、デフォルトの入れ替えが静かな回帰を生んでいないか確認してください。

これは最近のブリーフィングが繰り返す**「デフォルトやベンチではなく、自分の作業でモデルを固定せよ」**(7/9のSWE-Bench Proの欠陥、GPT-5.6のベンチ)とまさに同じ位置です。今回はベンダーが数字を差し替えたのではなく、*リリースがデフォルトを変えた*ことが同じ点検を要求しています。 [GitHub v2.1.207](https://github.com/anthropics/claude-code/releases/tag/v2.1.207)

## セキュリティ・制限事項

### v2.1.207、信頼・同意の境界を締め直す修正群 (7/10)

v2.1.207には、目立つ新機能と並んで、**権限・同意・信頼の境界**を狙った修正が入っています。プラグインやリモート管理設定を使う組織なら、読み飛ばしてよい項目ではありません。

- **リモート管理設定の「サイレント同意」を修正**: リモートで配布されたmanaged settingsが、**セキュリティダイアログを表示せずに同意済みとして記録**される問題が修正されました。管理型の配備でユーザー確認を飛ばして設定が適用される経路を塞いだものです。
- **プラグインのシェルインジェクションをブロック**: プラグインのhooks・monitors・MCP `headersHelper`において、**シェル形式コマンドで使われる`${user_config.*}`が拒否**されるようになりました。ユーザー設定値がそのままシェルコマンドに紛れ込み、インジェクションに至る経路を封じます。
- **プラグインのオプション値の信頼範囲を縮小**: プラグインのオプション値を、**プロジェクト単位の`.claude/settings.json`から読まなくなりました**。リポジトリがプラグインの挙動を任意に変えられる余地を狭めます。

一貫した筋は、**利便性のために広げていた信頼境界を、一層ずつ狭め直している**ことです。7/8のSkillCloak(悪性スキルのスキャナ回避)や7/9のGitLost(エージェントを騙したデータ漏えい)が*外部から来る脅威*だったのに対し、今回の修正群は*CLI自身が信頼していた入力経路*を締めます。サードパーティのプラグインを使う、あるいはClaude Codeを管理型で配備しているなら、v2.1.207へ更新し、特に`${user_config.*}`をシェルコマンドに渡していたプラグインがあれば挙動が変わったか確認してください。 [GitHub v2.1.207](https://github.com/anthropics/claude-code/releases/tag/v2.1.207)

## 注目プロジェクト＆ツール

- **レトロゲームを韓国語化する方法論を収めたエージェントスキル**: セガサターン・スーパーファミコン・ゲームギア・メガドライブ・ドリームキャストといった**レトロゲームの韓国語化の方法論を、Claudeのエージェントスキルとしてまとめた**プロジェクトです。専用のレガシーツールに頼るのではなく、**AIがROMファイル・ファイルシステム・フォント・テキストエンジンを直接解釈し、ゲームごとに独自の解法を開発**させるアプローチです。制作者は**Claude Opus 4.6で約2か月かけて**複数プラットフォームの韓国語パッチを作り、コストのかかる実装失敗を減らすために**静的なエラー検出と概念実証(PoC)テスト**を前面に置いたと述べています。7/8のSkillCloak(悪性スキルのスキャナ回避)が*スキルサプライチェーンの脅威*を突いたのに対し、本プロジェクトは*エージェントスキルがドメインの難題(バイナリのリバースエンジニアリング・フォント・エンコーディング)にどこまで踏み込めるか*を示す好例です。 [GitHub](https://github.com/mcpads/create-retro-game-kr-patch)
- **htmldrop — AIが作ったHTMLをリンク1つで共有 (MCP対応)**: **HTMLを投げると即座に共有リンクが出てくる**サービスです。APIで直接使うこともでき、**MCPサーバー統合**を通じて、AIエージェントが生成したHTMLを**自動でリンク化して共有**できます。リンクはデフォルトで**7日後に失効**し、**MITオープンソース**なのでセルフホスティングも可能です。Claude CodeやCoworkがレポート・ダッシュボード・プロトタイプをHTMLとして次々に吐き出す流れの中で、*エージェントの成果物を人へ手渡す最後の一歩*を埋めるユーティリティです。7/8のgh-attach(エージェントが作った証跡をCLIでPRに添付)と同じ筋の、*エージェント出力の共有チャネル*の一片です。 [htmldrop](https://htmldrop.link)
