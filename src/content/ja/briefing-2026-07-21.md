## 主要な新機能と実践活用

### `sandbox.filesystem.disabled` — ファイルシステム隔離だけを無効化し、ネットワークegress制御は維持する (v2.1.216)

v2.1.216で**`sandbox.filesystem.disabled`設定**が追加されました。これまでサンドボックスはファイルシステム隔離とネットワークegress制御が一体化していましたが、今回**ファイルシステム隔離だけを選択的にオフ**にできるようになりました。

- **なぜ必要か**: ファイルシステム隔離が特定のビルドツールやシンボリックリンク依存のワークフローと衝突し、サンドボックスをまるごと無効化せざるを得なかったチームにとって、**ネットワークegress制御という中核の防御ラインは維持したまま**摩擦だけを取り除けるスイッチができました。
- **適用範囲**: 隔離をオフにするのはファイルシステムアクセスに限定され、アウトバウンドのネットワークルールは従来通り強制されます — 7/18の権限チェック大改修が示した*信頼境界は細かく分割してそれぞれ調整できるべきだ*という方向性の延長です。

```json
// .claude/settings.json
{
  "sandbox": {
    "filesystem": { "disabled": true }
  }
}
```

[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

### 長時間セッションの「静かな低速化」が解消 — メッセージ正規化コストの二乗増加を修正 (v2.1.216)

v2.1.216で**長時間稼働するセッションでメッセージ正規化コストがターン数に対して二乗で増えていく低速化**が修正されました — 数秒単位の停止や再開の遅さの原因でした。

- **何が起きていたか**: セッションが長くなるほど毎ターン支払う正規化コストが*ターン数に比例*ではなく**ターン数の二乗に比例**して膨らんでいました。会話が積み重なるほど応答がだんだん遅くなり、最終的には再開そのものが目に見えて鈍くなるパターンでした。
- **誰が影響を受けたか**: `/loop`やワークフローで一日中続くセッション、圧縮なしに増え続ける無人パイプラインが最も大きな影響を受けました — 表面上は「なんとなく遅くなった」としか見えなかった問題の正体が、今回明らかになった形です。

要点は7/16のヒントにあった*無人・バックグラウンドセッションの認証寿命を点検せよ*と同じ系統だという点です — あのときは*認証*が長時間セッションの静かな死因でしたが、今回は*正規化コスト*が静かな低速化の原因でした。長時間セッションが一日の後半になるほど遅く感じていたなら、原因候補リストからこの項目を消せるようになりました。[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

## 開発者ワークフローティップス

### AskUserQuestionが「待ってほしい」という答えをきちんと理解するように (v2.1.216)

v2.1.216で**AskUserQuestionツールがユーザーの自由回答を誤解釈していたバグ**が修正されました — ユーザーが*待ってほしい*、*先に説明してほしい*と答えたのに、Claudeには**「進めていい」という合図として誤って伝わっていた**問題です。

- **7/17〜18のAuto-Continue騒動とは別のバグ**: 7月初めにv2.1.198が無告知で導入し、2日で opt-in に差し戻した*60秒無応答での自動進行*というミスフィーチャー(7/18ブリーフィング参照)とは別物です — 今回はユーザーが**実際に回答を入力したにもかかわらず**、その意図(待機・説明の要求)が逆に伝わっていた、自由テキストのパース段階における信頼性バグです。
- **何が変わるか**: 自由回答に**中立的な文言**を適用し、ユーザーが明示的に止めてと言ったときにClaudeが実際に止まるよう修正されました — AskUserQuestionを確認ゲートとして使う無人パイプラインで、人の答えが無視されているように見えていた体験が解消されます。

要点は**「回答可能性(answerability)の些細に見えて致命的な部分まで直す」**という姿勢です — 人が介入できるゲートを用意していても、そのゲートの信号伝達自体が漏れていたなら、無人自動化の信頼性は見た目より脆いということになります。AskUserQuestionで承認ゲートを設けているチームは、v2.1.216で実際に待機・説明のリクエストが尊重されるか確認してみてください。[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

### 再開したバックグラウンドサブエージェントが自分のアイデンティティを取り戻す (v2.1.216)

v2.1.216で**再開したバックグラウンドエージェントセッションがデフォルトエージェントに戻ってしまう問題**が修正されました — 今後は**エージェントのプロンプトとツール制限が再開時にそのまま復元**されます。

- **何が起きていたか**: `.claude/agents/`で定義したカスタムサブエージェント(レビュー専用・読み取り専用の調査エージェントなど)をバックグラウンドで起動し、後で再開すると、そのセッションが**カスタム定義を失い汎用のデフォルトエージェントとして動作**してしまうケースがありました — 絞り込んでいたはずのツール権限が、再開とともに広がりうるということです。
- **なぜ重要か**: 7/17の`/fork`↔`/subtask`分離以降、バックグラウンドセッションを複数走らせて後から開き直すワークフローが増えました — 再開が*同じエージェントの続き*ではなく*別のエージェントへのひそかな切り替え*になりうるというのは、ツールごとに権限を絞って使うチームほど静かに漏れていた穴でした。

要点は7/17ブリーフィングの*セッション位相の再整理*がまだ完全には終わっていないというサインです — `/fork`・`/resume`でセッションを使い回せる資産として扱うチームなら、再開後にアクティブなエージェントが意図したものかどうか確認する習慣は、v2.1.216以降は手放して構いません。[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

### `/context`が上限超過を直接警告するように

v2.1.216で`/context`が**会話がコンテキストウィンドウを超えたときに明示的に警告**するようになり、失敗した`/compact`は**エラーとして表示**されるようになりました。

- **これまでは**: コンテキスト超過や圧縮失敗が静かに見過ごされると、セッションの挙動がおかしい原因を後になって会話の長さの問題だと突き止める羽目になっていました。
- **今日からの習慣に**: 長時間セッションで応答品質がぶれてきたと感じたら、まず`/context`を確認するのをルーチンに入れましょう — 上記の信頼性修正群と同じく、**長時間セッションの状態を自分で診断できるシグナル**がまた一つ増えました。

[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

## セキュリティ・制限事項

### v2.1.216 — ハードニング連鎖の5番目の輪: worktree隔離バイパスとシンボリックリンク (7/20)

v2.1.216に、今週ずっと続いてきた**信頼境界のハードニング**の5番目の輪が実装されました。今回の標的は*隔離されていると信じられていたworktree*と*パス操作によるバイパス*です。

- **worktree隔離バイパスの遮断**: **worktreeで隔離されたサブエージェントが`git -C`、`--git-dir`、`GIT_DIR`/`GIT_WORK_TREE`を通じてgitコマンドを共有チェックアウトへリダイレクトできた**問題、そして**作業ディレクトリが選択したプロジェクトと異なる場合にworktreeセッションが別プロジェクトの残存worktreeに誤って着地していた**問題が同時に修正されました — 並列エージェントをworktreeで分けて動かす構成の隔離という前提そのものを締め直す修正です。
- **シンボリックリンクバイパスの遮断**: **workflowの保存とscheduled taskの書き込みが`.claude`内のシンボリックリンクをたどってプロジェクト外への書き込みが漏れていた**問題、そして**`/rewind`がシンボリックリンク・ハードリンクを通じて追跡対象パス外のファイルを復元・削除できた**問題が同じリリースで同時に修正されました(スキップしたパス数を報告)。
- **パーサー不整合3種**: **Bash複合コマンド文**(`&&`リストや否定内のリダイレクト)における権限チェック漏れ、**非ASCII文字**が実際のシェルの単語境界と異なる形でパースされていた問題、**PowerShellツールが不可視のUnicode文字を含むコマンドの権限チェックをすり抜けさせていた**問題 — 7/18の権限チェッカー大改修で取りきれなかった残存パース不整合を続けて狭めました。
- **Windows自動承認の縮小**: **読み取り専用コマンドがWindows上でネットワークパスにアクセスする際、プロンプトなしで実行されていた**問題も修正されました。

要点は7/15(インジェクション)・7/16(承認UIスプーフィング)・7/17(plan mode)・7/18(権限チェッカー大改修)に続く**5番目の輪**だという点です — 毎回*人が安全だと思い込んでいた境界*を一つずつ実際に安全にしており、今回の対象は**worktree隔離とパス(シンボリックリンク)操作**でした。worktreeでサブエージェントを並列隔離している、あるいはhook・worktreeを安全装置として使っているチームは、v2.1.216へアップグレードして前提を確認し直してください。[GitHub v2.1.216](https://github.com/anthropics/claude-code/releases/tag/v2.1.216)

### Fable 5新構造発効初日の摩擦 — Maxプランのクレジット誤請求インシデント、すべて解消 (7/20)

昨日(7/19〜20)確定・発効した**Fable 5新構造**(Max・Team Premiumは永久込み、Pro・Team Standardはクレジット換算)の**初日にインシデントが3件**発生し、いずれも解消されました。

- **Fable 5クレジット誤請求**: **Maxプランの一部ユーザーが、プランに元々含まれているFable 5を使っているにもかかわらず使用量クレジットを要求される**不具合がありました — 影響を受けたユーザーには**Claude Codeの再起動**が推奨され、解消しました。新構造に初めて触れたMaxユーザーが想定外の課金プロンプトを見たなら、このインシデントだった可能性が高いです。
- **モデルのelevated errors 2件**: **Opus 4.8**と**Haiku 4.5**でそれぞれelevated errorsが発生し、いずれも同日中に解消されました。

要点は**「構造が確定したからといって施行がスムーズとは限らない」**という点です — 7/19〜20ブリーフィングが扱った*18日間のシーソーの結末*が昨日は政策レベルの話だったとすれば、今日はその政策が実際の請求・権限ロジックとして実装される過程の最初の摩擦です。MaxプランでFable 5使用中にクレジットのプロンプトを見た場合は、再起動で解決したか確認し、繰り返すようならインシデント履歴と照合してください。[Claude Status](https://status.claude.com/history) · [StatusGator](https://statusgator.com/services/claude)

**Sonnet 5の導入価格は8/31に終了**し、9/1から入力3ドル・出力15ドル(+50%)に上がります — 詳細は7/13ブリーフィングを参照してください。

## おすすめコラム＆読み物

- **「LangGraph導入後、なぜコードがより複雑になったのか」**: *状態管理とフロー制御がすっきりするはず*という期待でLangGraphを導入したところ、逆にコードが複雑化してしまったという失敗談です。診断してみると**グラフのノードが実質1つしかない**にもかかわらず、フレームワークの状態・エッジ・条件分岐ルーティングの概念をすべて背負い込んでしまい、実際の問題(単純な逐次ロジック)よりもフレームワークが要求する抽象化コストの方が大きかったというのが要点です。エージェントオーケストレーションフレームワークの導入を検討している開発者に向けて — *問題の実際の複雑さがフレームワークの標準的な複雑さに満たない場合、フレームワーク自体が負債になる*という反例のケーススタディです。7/16のワークフローティップにあった*dynamic workflowsは繰り返し可能なオーケストレーションにしか価値がない*という原則と並べて読む価値があります。[GeekNews](https://news.hada.io/topic?id=31628)
- **「AIの助言を受けると精度は3分の1に下がり、自信は2倍以上に高まる」**: AIの助言を受けた参加者は**「わからない」と答える割合が44%から3%へ急減**し、**精度は27%(対照群に比べて低下)まで下がった一方、自信は76%まで跳ね上がった**という実験結果です。研究チームは*合理的なツールへの委任*と区別するため、あえて**AIがよく間違える種類の質問**(映画の視覚的な細部など)を選んでテストしました — AIが自信満々に間違えると、人間も一緒に自信満々で間違えるようになる過信の伝播現象を示しています。上のWordPress RCE事例が*きちんと設計されたワークフロー下でのAIの実戦成果*を示したとすれば、この研究はその対極 — **検証なしに受け入れるAIの助言のリスク**をデータで裏付けます。`/verify`がopt-inになったv2.1.215(7/19)以降、*証拠を要求せよ*という原則がなぜ今もなお人間の役割であり続けるのかを示す根拠資料として読む価値があります。
- **「中国のオープンウェイトAI戦略が先行している」**: AIモデルは**ブランドロイヤルティと表面的な乗り換えコスト以外にはさほど大きな防御力を持たない**ため、中国のオープンウェイト戦略が米国のクローズド型モデルよりも広い展開エコシステムを作り出しているという分析です。核心の論点は — 企業向けAIの実質的な堀は**モデルそのものではなく、契約・社内システム連携・利便性機能**といった周辺サービスから生まれるということです。7/18のコラム*State of Open Source AI*(オープンウェイトがコーディング・指示追従でクローズドモデルに収斂しつつある)と本日のSupabaseレポート(ハーネス・プラットフォーム層へ移る競争)をつなぐ、*モデルはコモディティ化しつつある*という今週繰り返されてきたテーゼの別の角度です。[GeekNews](https://news.hada.io/topic?id=31634)
