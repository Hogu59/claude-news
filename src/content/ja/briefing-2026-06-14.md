## 開発者ワークフローティップス

6/15のProgrammatic Usage Credits移行は**明日**です。昨日までに `/usage` での内訳確認や `enforceAvailableModels`（組織レベルのallowlist）を扱いましたが、今日はその下の**repoレベルで今すぐ直せる2つ**です。

### サブエージェントのデフォルトモデルをYAMLに明記し、repoにコミットする

各サブエージェント定義（`.claude/agents/*.md`）の **YAML frontmatter に `model:` を明記**し、その設定を **repo にコミット**しておくパターンです。コードレビューは Sonnet、lint やフォーマット確認のような機械的作業は Haiku というように、役割ごとに適切なモデルを**デフォルトとして固定**すれば、すべてのエージェントが惰性で最も高価なモデルに落ちるのを防げます。

```markdown
---
name: lint-checker
description: 変更ファイルの lint・フォーマット違反のみを確認
model: claude-haiku-4-5-20251001
tools: Read, Bash
---
```

昨日扱った `enforceAvailableModels` が**組織から下に押さえつける** allowlist だとすれば、これは **repo の中で各エージェントのデフォルトモデルを決める**補完策です。両方を併用すれば「誰がどのモデルでクレジットを消費するか」を上下両方から制御できます。クレジットが分離される明日の前に、よく使うサブエージェントの `model:` から点検しておきましょう。[CloudZero — Claude Code Agents](https://www.cloudzero.com/blog/claude-code-agents/)

### 並列サブエージェントは3〜5個が現実的な上限

複数の分析が同じ経験則に収束しています — **並列サブエージェントは3〜5個で効用が最大**になり、**10個を超えるとコストだけ増えて追加の利益はほとんどない**、というものです（厳密な制限ではなく実務上のヒューリスティックです）。6/9に扱った「サブエージェント中心のワークフローは単一スレッドの最大約7倍のトークンを使う」点の、実務的な上限線にあたります。

各サブエージェントが自分のコンテキストを別々に抱えて動くため、fan-out の幅を広げるとトークンはほぼ線形に増える一方、分解の利益はすぐに飽和します。「もっと多く立ち上げれば速くなる」ではなく、**本当に独立して分割できる3〜5個の大きな塊**に分けるのが、コスト・精度の両面で有利です。6/10のネストされたサブエージェント（最大5段）と併用する場合は、段ごとに倍率が掛け算されるため、なおさらです。[Tembo — Claude Code Subagents](https://www.tembo.io/blog/claude-code-subagents)

## セキュリティ・制限事項

### 米輸出管理の指示でFable 5・Mythos 5を全面回収 — 5日での反転 (6/12)

6月12日、Anthropic は **Claude Fable 5 と Claude Mythos 5 へのアクセスを全面的に無効化**しました。米商務省が**国家安全保障を根拠に輸出管理の指示**を出し、**すべての外国人（米国内外を問わず、外国籍の Anthropic 従業員を含む）**による両モデルの使用を停止するよう命じたためです。Anthropic はこれを順守するには**モデルそのものを丸ごと取り下げるしかない**と述べました（指示はET 6/12 17:21に受領）。

開発者にとって重要な理由は明白です — **5日前（6/9）にv2.1.170でClaude Codeから使えるようになったばかりのFable 5が、6/22までのPro・Max・Team・Enterprise無料提供期間の途中で消えた**のです。外国籍の開発者（韓国・日本を含む）にとっては、事実上アクセスが断たれました。

- **理由は明示されず**: 政府の書簡は**具体的な懸念を示していません**。ただし当局者は、**Fable 5の安全装置を回避する手法**（本来はMythos級のサイバー能力へのアクセスを防ぐためのもの）を把握した後に決定したと伝えています。
- **Anthropicの異議**: 同社はFable 5には強力な安全装置があったとし、**狭い脆弱性ひとつでモデルごと取り下げることは、業界全体のフロンティアモデル展開を止める悪い前例になる**として、命令に同意しないと表明しました。

6/10〜6/11に扱ったFable 5の**3重分類器・distillationブロック・Project Glasswing**が「強力なモデルほど誰が使えるかを狭く制御する」方向だったとすれば、今回の回収はその制御権が**企業ではなく政府の手にある可能性**を露わにしました。Fable 5をワークフローに組み込んだばかりなら、**`fallbackModel`・`availableModels` でOpus 4.8のような代替モデルに安全にフォールバックする**設定を点検しておくのが当面の実務対応です。[Anthropic 公式声明](https://www.anthropic.com/news/fable-mythos-access)

### D-1: 明日(6/15) Programmatic Usage Credits 施行 (6/15施行)

数日間追ってきた **Programmatic Usage Credits** が**明日**施行されます（5/14予告）。最後に要点だけ — **Claude Agent SDK・`claude -p`・Claude Code GitHub Actions・サードパーティエージェント**がサブスクリプション上限から分離され、**別建ての月次クレジット（全額APIレート・繰り越しなし）**で課金されます。**対話型ターミナル・チャット・Cowork は影響を受けません**（Pro $20・Max5x $100・Max20x $200、使い切ると overflow を設定していなければ自動化が停止）。

本日の最終チェックリストはシンプルです — ①`/usage`で漏れを確認（6/13）、②上のティップス通りサブエージェントのモデルとfan-out幅を整理、③overflow課金ポリシーを意図的に決めておく。[TechTimes まとめ](https://www.techtimes.com/articles/317625/20260602/anthropic-ends-subscription-subsidy-agents-june-15-credit-pool-replaces-flat-rate-access.htm)

## 知っておくと便利な小さな変更点

昨日扱えなかった **v2.1.176** の実用的な修正です。

- **Remote Control のモデル無断切替を修正**: Web/モバイルから Remote Control で接続したときにセッションモデルが**無断で切り替わる**問題が修正されました（v2.1.176）
- **Remote Control の切断通知を改善**: 切断時に**数値コードではなく人間が読める理由**が表示され、アカウントサインイン時にリモートセッションが正しく切断されます（v2.1.176）
- **`/cd`・worktree 移動のブランチ表示を修正**: ディレクトリ移動後も**以前のディレクトリの git ブランチ**を表示していた問題が修正されました（v2.1.176）
- **`footerLinksRegexes` 設定を追加**: フッター行に **regex でマッチするリンクバッジ**を入れられます（user・managed 設定の両方）（v2.1.176）
- **[Linux] sandbox のシンボリックリンク修正**: `.claude/settings.json` が絶対パス先のシンボリックリンクのとき sandbox の起動に失敗する問題が修正されました（v2.1.176）
- **クラウドセッションの idle 認証エラーを修正**: 長時間 idle のクラウドセッションが "Could not resolve authentication method" で失敗する問題が修正されました（v2.1.176）

## おすすめコラム＆読み物

- **「FFmpegの21件のゼロデイ」(depthfirst)**: あるセキュリティチームの **AIエージェントが約 $1,000** でFFmpegのCコード約150万行を走査し、**再現可能なPoC付きのゼロデイ21件**を発見しました。一部は **15〜20年**、1件は **23年（2003年のコード）**眠っていたバグで、最も深刻なDFVULN-127はAV1 RTP depacketizerのヒープオーバーフローにより、**183バイトのパケット1つ（RTSP経由）でリモートコード実行**が可能です。AIがセキュリティ研究の経済性を丸ごと変えつつある生々しい証拠であり、まさに上で米政府がMythos級モデルのサイバー能力を警戒する理由と同じ場所に位置する記事です。[depthfirst](https://depthfirst.com/research/21-zero-days-in-ffmpeg)
- **「Every Frame Perfect」(tonsky)**: UIの品質は、開始・終了の状態ではなく**その間のすべてのアニメーションフレームが完璧かどうか**で決まる、という記事です（6/13公開）。コンポーネントが異なる速度で動いたり、画面遷移に白いちらつきが混じったりした瞬間にユーザーはアプリを「詰めが甘い」と感じます。「ユーザーはコードを見られないので、**UIこそが品質を判断する唯一の表面**」だからです。6/10に扱った「Linearはなぜ速いのか」と同じ筋の職人的ディテール論で、エージェントがコードを大量に吐き出す時代に**最後の1%の仕上げ**が信頼を作ると説きます。[tonsky](https://tonsky.me/blog/every-frame-perfect/)
- **「AIがソフトウェアエンジニアを置き換えなかった理由」(normaltech)**: AIがエンジニアを置き換えられなかったのは、**そもそもコーディングがボトルネックではなかった**からだという分析です。本当の制約は**何を作るかを決め、結果を検証し、責任を負う**こと — AIは実行レイヤーを圧縮しましたが、**意思決定・デリバリーのレイヤーは依然として人間の領分**として残った、というものです。6/9に扱った「LLMが私のキャリアを侵食している」という悲観と対で読むと、「では人はどこに残るのか」への落ち着いた反対側の答えになります。[normaltech](https://www.normaltech.ai/p/why-ai-hasnt-replaced-software-engineers)

## 注目プロジェクト＆ツール

- **LOSLES — 「金額そのもの」を識別子に使う自動決済システム**: PG社の加入・承認・手数料なしで、個人開発者や小規模プロジェクトが決済を受けられるオープンソースです。発想が新鮮です — 1万ウォンの商品に注文ごとに9,901ウォン・9,900ウォンのように**わずかに異なる金額**を割り当て、Androidアプリが**KakaoPayの入金通知をリアルタイムで検知**し、受け取った金額からどの注文かを照合します。「決済の識別」という馴染みのある問題を、**金額をキーに変えて**解く点が印象的です（大規模サービス向けではないと作者も明記）。[GitHub](https://github.com/ai1023dev/Losles)
- **MiniLex — 自然言語で法令を検索するRAGデモ**: 「性暴力処罰法第14条」のような条文番号を知らなくても、**日常語で尋ねて関連条項を出典付きで**受け取れる法令検索ツールです。7種の法令文書を対象に **39.7万+のMarkdown文書・136万+の条項**をインデックス化し（nomic-embed-text-v2-moe 埋め込み、内部ベンチでR@5 0.86）、Next.js・FastAPI・pgvector構成、MITライセンスです。ここ数日扱ってきたRAG・エージェントメモリの系譜を、**公共の法令検索**という実生活の問題に適用した事例です。[GitHub](https://github.com/wellsa-ai)
