## 主要な新機能と実践活用

### パススコープ付き hook 条件がようやく正しくマッチする (v2.1.176)

hook を `Read`/`Edit`/`Write` の**特定パスだけ**に掛けていた人は、文書どおり書いたのに静かに発火しないケースに遭っていたかもしれません。v2.1.176 がこれを修正し、`Edit(src/**)`・`Read(~/.ssh/**)`・`Read(.env)` のような**パスパターン条件が文書どおりマッチ**します。

意味は明確です — 「**Claude がこのパスに触れたときだけ**」走る hook を安定して書けるようになりました。たとえば `src/**` を編集したら自動フォーマッタを走らせ、`.env` を読もうとしたらブロック/警告する、といった**パス単位の決定的ガード**です。

```json
// .claude/settings.json（スキーマ/キーの正確な形は公式 hooks ドキュメント参照）
{
  "hooks": {
    "PreToolUse": [
      {
        "if": "Read(.env)",
        "command": "echo '⚠️  .env の読み取りをブロック' && exit 2"
      }
    ],
    "PostToolUse": [
      {
        "if": "Edit(src/**)",
        "command": "prettier --write \"$CLAUDE_FILE_PATH\""
      }
    ]
  }
}
```

昨日扱った「**CLAUDE.md は勧告（約80%遵守）、hook は決定的（100%）**」という区別の、実戦的な完成形です — その決定的な hook を、今や**パス単位で正確に狙える**ようになりました。これまでパス条件を入れて「効いているはず」と思っていたなら、実際は素通りしていた可能性があるので、一度点検してみてください。

[GitHub v2.1.176](https://github.com/anthropics/claude-code/releases/tag/v2.1.176)

### `enforceAvailableModels` — Default・エイリアス・`/fast` まで覆うガバナンス (v2.1.175/176)

6/11 に扱った `availableModels`（選択可能なモデルの許可リスト）に、**実質的な続報**が来ました。これまで許可リストは「選べるモデル」だけを絞っており、**Default として使われるモデル**や環境変数エイリアスには抜け道がありました。その穴をふさいでいます。

- **`enforceAvailableModels`**（管理設定、v2.1.175）：有効にすると allowlist が **Default モデルまで**制約します。Default が許可外モデルに解決される場合は**リストの最初の許可モデルへフォールバック**し、user/project 設定が **managed リストを広げられなくなります**。
- **エイリアス迂回の遮断**（v2.1.176）：`ANTHROPIC_DEFAULT_OPUS_MODEL` のような `ANTHROPIC_DEFAULT_*_MODEL` 環境変数で、エイリアスを**許可外モデルへリダイレクトする経路が塞がれ**、`/fast` も切替結果が allowlist の外になる場合は**トグルを拒否**します。

```json
// 管理 settings.json（組織ポリシー）
{
  "enforceAvailableModels": true,
  "availableModels": ["claude-sonnet-4-6", "claude-haiku-4-5-20251001"]
}
```

**2日後（6/15）の Programmatic Usage Credits 施行**と正確にかみ合う変更です — クレジットが分離された瞬間、サブエージェント（ネスト含む）や自動化がこっそり上位モデルへ乗り換えることが実コストになります。これを Default・エイリアス・`/fast` のどの経路からも迂回できないよう、**一つのスイッチでまとめて**抑えられるようになりました。

[GitHub v2.1.175](https://github.com/anthropics/claude-code/releases/tag/v2.1.175)

## 開発者ワークフローティップス

### `/usage` が cache miss・long context・サブエージェントで内訳表示 — 6/15 前に点検 (v2.1.174)

v2.1.174 から `/usage` ダイアログが使用量を **cache miss・long context・サブエージェント**で分解して表示し、**VSCode の帰属**も示します。単なる「合計」ではなく、**どこでトークンが漏れているか**を指し示します。

これは単なる UI 改善ではなく、**6/15 クレジット施行を控えた診断ツール**です。自動化やサブエージェントのツリーがクレジットを速く食うなら、まず `/usage` で **long context・cache miss・サブエージェント**のどれが原因かを確認しましょう。cache miss の比率が大きければプロンプト構造を安定させてキャッシュヒット率を上げ、サブエージェントの比率が大きければネスト深度を削る — **施行前に**削れる余地を見つけられます。

[GitHub v2.1.174](https://github.com/anthropics/claude-code/releases/tag/v2.1.174)

### 「必ず毎回」なら CLAUDE.md ではなく hook へ昇格させる

複数の実務ガイドが同じ結論に収束しています — **CLAUDE.md は勧告（advisory）で Claude は約80%しか従わず、hook は決定的（deterministic・100%）**です。フォーマット・lint・セキュリティチェックのように**例外なく毎回起きるべき**ことは、CLAUDE.md の一文ではなく **hook にすべき**です。

これまで hook は「すべての Edit」のように広くしか掛けられず負担でしたが、上記 v2.1.176 の**パススコープ条件**がこの負担を軽くします — `Edit(src/**)` でソースディレクトリだけ、`Read(.env)` でシークリットアクセスだけに掛ければ、決定的な保証を保ったまま**不要な場所では走らせない**ことができます。「このルールは絶対に外せない」項目を CLAUDE.md から選び、**パススコープ付き hook へ移す**のが今週の実戦的な整理です。[Builder.io — Claude Code Tips](https://www.builder.io/blog/claude-code-tips-best-practices)

## セキュリティ・制限事項

### D-2：6/15 Programmatic Usage Credits 施行 — 自動化は別建てクレジットへ分離される (6/15 施行)

数日間、通りすがりに触れてきた **6/15 Programmatic Usage Credits** が、いよいよ**2日前**になりました（2026-06-15 施行、5/14 予告）。要点を再整理すると — **プログラム的な利用がサブスク枠から分離され、別建ての月次クレジットで**課金されます。

- **対象**：**Claude Agent SDK、`claude -p`（ヘッドレス）、Claude Code GitHub Actions、サードパーティエージェント**。これらはサブスク枠ではなく**フル API レートの月次クレジット**でメータリングされ、**繰越（rollover）なし**。
- **影響なし**：**対話型ターミナルの Claude Code、チャット、Claude Cowork** などインタラクティブ利用はそのままです。
- **クレジット枠**：Pro **$20**、Max 5x **$100**、Max 20x **$200** — 毎月失効。
- **使い切ると**：**overflow ビリングを有効にしていないと自動化が停止します。** また Opus 4.8 トークナイザーがプロンプトあたりのトークンを**最大35%**多く数える場合があり、体感の消費が想定より速いことがあります。

今日の新しい局面は、**施行間近（D-2）**であることと、上で扱った **v2.1.175/176 のガバナンス強化がまさにこの変更への応答**だという点です — クレジットが分離された瞬間、「どの自動化がどのモデルでいくら使うか」が即コストになるため、①`/usage` で帰属を確認し、②`enforceAvailableModels` で自動化のモデルを縛り、③overflow ポリシーを意図的に決めておく、が施行前のチェックリストです。[Anthropic 課金変更（TechTimes まとめ）](https://www.techtimes.com/articles/317625/20260602/anthropic-ends-subscription-subsidy-agents-june-15-credit-pool-replaces-flat-rate-access.htm)

## おすすめコラム＆読み物

- **「仕事で何もしない（Doing Nothing at Work）」（Sean Goedecke）**：エンジニアのインパクトは**常に忙しいこと**からではなく、**決定的な瞬間に使える余力を残しておくこと**から生まれる、という論考です。平時の稼働率を **80%** に保つからこそ、大型契約の成立・インシデント対応・重要機能のローンチといった、**予告なく訪れる高インパクトの機会**をつかめる — ルーティンに100%注ぐと、肝心な瞬間に動かす手が残らないという逆説です。エージェントがルーティンを高速に片付ける時代に、「人はどこに余力を置くべきか」を問い直します。GN+ 27ポイント。[Sean Goedecke](https://www.seangoedecke.com/doing-nothing-at-work/)
- **「生産的な個人が生産的な企業を作るとは限らない」**：AI が**個人の生産性を10倍**にしても、企業価値は比例して上がらない、という分析です。1890年代の紡績工場が蒸気機関を電気モーターに**ただ置き換えた時ではなく、工程全体を電気モーター中心に再設計した時**に初めて利益を出した、という歴史的教訓を引き、ツールを導入するだけでなく**技術と組織構造を一緒に再構想**する「制度的知性（Institutional Intelligence）」が必要だと説きます。6/10 に扱った「AI は減速している」と同じ筋で、「能力は上がるのになぜ企業成果につながらないのか」への構造的な答えです。GN+ 23ポイント。[原文（@gsivulka）](https://x.com/gsivulka/status/2031797989908627849)
- **「Software is Built Between Commits」（Zed, DeltaDB）**：コードは**コミットの時点ではなく、コミットの間**で作られる、という視点から、AI エージェントとの**会話と細かな編集の全過程**を追跡する新しいバージョン管理システム **DeltaDB** を紹介します。完成したコミットを待ってレビューする代わりに、作業が展開される間に**会話とコード変更を一緒に記録**し、リアルタイムの協働を狙います — 「会話こそがソース」となるエージェント開発の時代に、バージョン管理そのものを作り直そうとする試みで、最近扱った「ループエンジニアリング」「エージェントメモリ」の流れと同じ場所に置かれます。[Zed Blog](https://zed.dev/blog/introducing-deltadb)

## 注目プロジェクト＆ツール

- **OCTOMO — SMS の流れを反転させた携帯電話認証 API**：従来の認証はサービスがユーザーにコードを**送る（MT）**方式で送信コストがかかりましたが、OCTOMO はユーザーが指定番号にコードを**送る（MO）**方式に流れを反転させました。送信コストが消え、**ユーザーが実際に自分の端末からメッセージを送った**という点で所有認証がより強くなります。サービスは単一の API に「直近5分以内に一致するコードが届いたか」を問い合わせるだけです — 馴染みのある問題を、方向を変えて解く発想が新鮮です。GN+ Show 16ポイント。[octomo.octoverse.kr](https://octomo.octoverse.kr)
- **turbo-graph — グラフメモリを載せた RAG インデックス**：turbovec の上に**グラフメモリとメタデータフィルタリング**を加えた制約型 RAG インデックスレイヤーです。テナント ACL・タグ・ソース・時間ウィンドウ・グラフ隣接といった**多次元フィルタリング**を、アプリケーションコードではなく**インデックスレイヤー**でグラフコンパイル・キャッシュ再利用・リランキングとして処理します。現在 alpha で、本番 RAG に向けた API 設計のフィードバックを募集中です — 最近扱った「memorize」やエージェントメモリ系に、**RAG インフラの側**から迫るもう一つの試みです。[GitHub](https://github.com/bigmacfive/turbo-graph)
