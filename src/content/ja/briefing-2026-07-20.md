## 主要な新機能と実践活用

### `/verify`・`/code-review`、Claudeが勝手に回さなくなった — 検証がopt-inに（v2.1.215）

v2.1.215の唯一の変更は短いですが、日々の流れを作り替えます。**Claudeは`/verify`と`/code-review`スキルを自ら実行せず、あなたが明示的に呼び出したときだけ**動くようになりました。

- **何が変わったか**: これまでClaudeはスキルの自動選択の過程で、検証やコードレビューを *勝手に* 差し込むことがありました。便利な場面もありましたが、この2つは重いスキルです — `/code-review`はサブエージェントを扇状に展開し、`/verify`は実際にアプリを起動して挙動を確認します。望まない瞬間に起動するとトークンと時間を消費し、実行の流れを驚かせていました。これで **起動する主体が人間に固定**されました。
- **使い方**: 変更をコミットする前にランタイム挙動を目視するなら`/verify`、作業ブランチのdiffを精読させるなら`/code-review`を自分で打ちます — 自動トリガーが消えたので、この2つは *意図的に開けるゲート* になりました。

```text
/verify          # アプリを起動して変更が実際に動くか確認
/code-review     # 現在のdiffを精読 — 必要なときに明示呼び出し
```

これは昨日（7/19）の *主張ではなく証拠を要求せよ* の正確な続きです。検証が *自動* なら人間は気にする必要がありませんが、*opt-in* になると **検証を呼ぶこと自体が忘れやすい一手** になります。下のワークフローTipsで、無人ループがこの変更後に検証を落とさないための方法をまとめます。[GitHub v2.1.215](https://github.com/anthropics/claude-code/releases/tag/v2.1.215) · [Claude Code changelog](https://code.claude.com/docs/en/changelog)

### Claude Enterprise Admin APIベータ — 組織メンバー管理をコードで（7/14）

Anthropicが **Claude Enterprise組織のユーザー管理をAPIで開くAdmin APIベータ**を公開しました（7/14）。コンソールをクリックして人を管理していた作業が、**スクリプトで自動化**できます。

- **できること**: **メンバー一覧・メールでの検索、ロール変更、メンバー削除、招待の送信・取り消し、グループとその所属の管理、カスタムロールの読み取り**をAPIで処理します。オンボーディング・オフボーディングをIdPや人事システムのフローに繋いで自動化したいチームに直接届きます。
- **ベータヘッダーとスコープ**: グループ・カスタムロールのリクエストには`anthropic-beta: ce-user-management-2026-07-13`ヘッダーが必要で、メンバー・招待のリクエストはヘッダー不要です。`read:org_audit`スコープを持つAdmin APIキーは、すべてのユーザー管理GETエンドポイントを呼び出せます。

これは最近の流れと同じ場所にあります — 7/14のHIPAAセルフサーブ、7/1のモデル単位エンタイトルメントに続き、**エンタープライズ運用のレバーがコンソールからAPIへ降りてくる**一片です。組織規模でClaudeを回し、アカウントのライフサイクルを自動化したいチームなら、ベータ段階からパイプラインに載せてみる価値があります。[Anthropic Developer Platform リリースノート](https://releasebot.io/updates/anthropic/claude-developer-platform)

## 開発者ワークフローTips

### v2.1.215以降 — 無人ループに検証を「もう一度」明文化せよ

上の新機能の裏面が、そのまま本日の行動項目です。**`/verify`・`/code-review`が自動実行されなくなったため、これまでClaudeの自動検証にそれとなく依存していた無人・自動トリガーのパイプラインは、検証ステップが静かに抜け落ちる可能性があります。**

- **CLAUDE.mdに明記**: *完了を宣言する前に必ず`/verify`を実行し、その出力を添付せよ* のようなルールをCLAUDE.mdに置けば、自動トリガーが去った場所を指示で埋められます — 7/19のTips *証拠で報告させよ* を、v2.1.215以降の版に移したものです。
- **強制はhookで**: 例外なく強制すべきなら、完了直前に検証を要求するStop hookが指示文より確実です（7/13のTips） — auto modeがhookの`ask`を尊重するよう修正されたのが7/15なので、その前提も今や成立します。
- **明示呼び出しをルーチンに刻む**: レビューが要るブランチなら、作業終了マクロに`/code-review`を明示的に入れましょう — 自動からopt-inに変わったスキルは、呼ぶ習慣を作って初めて再び生きます。

```markdown
# CLAUDE.md の例（v2.1.215以降）
- タスク完了を宣言する前に必ず /verify を実行し、その出力を添付すること
- 検証なしに「動きます」と報告しないこと
```

繰り返しの命題は **「便利なデフォルトが消えたら、安全は明示的に取り戻す」** です — 自動検証があった場所に *呼ぶルール* を置くことが、v2.1.215アップグレードの本当の宿題です。[GitHub v2.1.215](https://github.com/anthropics/claude-code/releases/tag/v2.1.215)

### 予備のMac 1台を「Claude Code専用の実行端末」に — 隔離レシピ

`--dangerously-skip-permissions`を気楽に使いたいが、メイン端末の個人データが気になる。そんなときに、**物理的に分離した予備MacをClaude Code専用の実行環境に**する詳しいガイドが公開されました。コンテナのサンドボックスと完全ローカル実行の中間値です。

- **なぜ別ハードウェアか**: コンテナは結局メイン端末で動くため完全隔離ではなく、Unity・GUIアプリ・グラフィック作業までやらせるには実機が要り、何より **`--dangerously-skip-permissions`の爆発半径を別の物理端末の中に閉じ込められます。**
- **隔離の原則**: 古い予備Macを完全に消去し、**Apple IDなしの新規ローカルアカウント**を作り、個人データを取り除いた上で **SSH（LAN）やiPhoneのClaudeアプリで遠隔操作**します。Ed25519鍵でパスワードなしSSH、パスワードなし`sudo`でエージェントのプロンプトを排除、スリープ・自動ロック解除で接続維持が主要な準備です。
- **実戦の拡張**: 暗号化P2Pのクリップボード同期（`clip.sh`）、tmux + LaunchAgentベースのcomputer-use（GUI自動化）、家の外から繋ぐTailscale、ブラウザ自動化のChrome拡張まで付きます — 画面収録・アクセシビリティ権限は（Claudeではなくtmuxバイナリに対して）人間が一度承認します。

これは7/17の *中国連携キャンペーンが示した二重用途リスク*、7/18の *権限チェッカー大改修* と同じ場所にあります — エージェントの権限を広く開くほど、**開いた権限をどこに閉じ込めるか**が次の設計課題になります。複数の自律エージェントを回すチームに、爆発半径を端末単位で隔離する実物のリファレンスです。[GeekNews](https://news.hada.io/topic?id=31591) · [claude-controls-mac](https://ykdojo.github.io/claude-controls-mac/)

- **Transcribe.cpp — クラウドなしでアプリに組み込むクロスプラットフォーム音声認識**: Mac・Windows・Linuxのアプリに **ローカル音声認識（STT）を直接組み込む** オープンソースライブラリです。**ggml** ベースで **16のASRファミリー・60以上のモデル** をサポートし、**Vulkan・Metal・CUDA・TinyBLAS** アクセラレーションに **Python・JS/TS・Rust・ObjC/Swift** の公式バインディングを提供します。ストリーミング・バッチの2モード、既存の`whisper.cpp`の`.bin`モデル互換、RK3566のような低電力端末でもリアルタイム以上の速度が特徴で、数千の発話に対してWERで精度を検証しています — ドキュメントは *意図的にAIを使わずに* 書いたと明かしています。音声をクラウドに送らずオンデバイスで処理したいデスクトップアプリ開発者にとって、*個人開発者でも本格的なインフラツールを作れる* という実物の事例です。[GeekNews](https://news.hada.io/topic?id=31600) · [GitHub](https://github.com/handy-computer/transcribe.cpp)
