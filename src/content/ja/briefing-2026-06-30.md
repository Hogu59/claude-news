## 主要な新機能と実践活用

### Claude in Microsoft FoundryがGA — 「Azureアカウントそのまま」本番エージェントへ（GA、6/29）

CLI外で最大の発表です。**6/29、ClaudeがMicrosoft FoundryでGA（一般提供）**になりました — Azure上でホストされ、企業が**すでに信頼しているAzureアカウントの認証・課金・ネットワーク・ガバナンス・データ制御のまま**Claudeでビルドできるようになります。要点はGAであること、つまり実験ではなく**正式にサポートされるサービス**だという点です。

- **モデル・機能**: **Claude Opus 4.8・Haiku 4.5**がMessages APIで提供され、**prompt caching・extended thinking**などのコア機能もそのまま付いてきます — コーディング・エージェント作業・複雑な推論までカバーします。
- **データレジデンシー・課金**: 推論を**どこで処理するか選べ**（データレジデンシーが必要なチーム向けの**USデータゾーン**を含む）、請求は**単一インボイス**に統合され、資格のある**Microsoft Enterprise Agreement**顧客はClaudeの使用分を**Azureのコミットメントに充当**できます。
- **2つのホスティング**: **Azureホスティング**（ガバナンス統合が完全）と**Anthropicホスティング**（機能の利用範囲が広い）から選べます — コンプライアンスを取るか、最新機能を先に取るかの選択です。

開発者視点の意味は明確です — **エージェントの実験から本番までの経路が短くなります。** 別のベンダーオンボーディングなしに、*すでに使っているAzureアカウントの中で*ガバナンス・監査・ネットワークポリシーを保ったままClaudeエージェントを立ち上げられます。とくにモデルへの直接アクセスが**地政学・政策に揺れる**今（後述の輸出規制を参照）、*マネージドクラウド経由のガバナンスされた経路*は、韓国・日本などのエンタープライズチームにとってもう一つの安全弁になります。 [Claude in Microsoft Foundry](https://claude.com/blog/claude-in-microsoft-foundry) · [Microsoft Azure Blog](https://azure.microsoft.com/en-us/blog/claude-in-microsoft-foundry-is-now-generally-available/)

## 開発者ワークフローティップス

### v2.1.196で無人実行がより安く・より堅牢に — `/code-review` −25% + idle watchdogデフォルトON

Claude CodeをCI・cron・backgroundで**無人実行**しているなら、v2.1.196の2つの変更がすぐ効きます。**① `/code-review`のトークン効率が約25%改善**し、変更レビューを自動パイプラインに組み込んだチームの呼び出しコストが下がります。**② stream idle watchdogが全プロバイダーでデフォルトON**になりました — 切れたソケットや止まったストリームに**無限にぶら下がる代わりに自動で中断**します。

```text
# 以前: レスポンスストリームが死ぬとセッションが止まり、無人ジョブ全体がハングした
# v2.1.196: idle watchdogがデフォルトで有効、止まったストリームを自動中断 → リトライ/失敗へきれいに抜ける
```

要点は、**「静かに止まり一晩ハングする」失敗モードをデフォルトでガードする**ことです。これまではwatchdogを自分で有効にするか、外部タイムアウトで包む必要がありましたが、いまそのガードが標準になりました — 6/22で扱った**`CLAUDE_CODE_MAX_RETRIES`（上限15）・リトライ/バックオフ**、6/27の**backgroundのクラッシュ復旧の強化**と同じ流れで、*無人実行の信頼性*をもう一段引き上げる束です。無人パイプラインがあれば更新し、自分で入れた外部watchdogと二重になっていないか確認しておきましょう。 [GitHub v2.1.196](https://github.com/anthropics/claude-code/releases/tag/v2.1.196)

### `--dangerously-skip-permissions`が「静かにフォールバック」していたバグの修正 — ヘッドレスの権限モードを再検証せよ

2つ目は**権限モードの落とし穴**を突くティップスです。v2.1.196は、**`--dangerously-skip-permissions`が実際には適用されず、静かに別のモードにフォールバック**していたバグを修正しました。つまり、このフラグでヘッドレス・CIの権限プロンプトをスキップしていると*信じていた*自動化が、実は**そのバイパスが効いていない**まま動いていた可能性があります。

```bash
# このフラグをCI/ヘッドレスで使っていたなら:
claude -p "..." --dangerously-skip-permissions
# v2.1.196以前: 静かにフォールバックし、意図したbypassが適用されていなかった可能性
# v2.1.196以後: bypassが実際に適用される → 挙動が変わりうるので再検証が必須
```

要点は、**「自分がかけた権限モードが本当にそのモードだったか」を再確認せよ**ということです。この修正で*挙動が双方向に変わりえます* — フォールバックのおかげで（意図せず）より安全に動いていた作業がいまや**本物のbypass**で実行され始め、逆にbypassが効かず止まっていた作業はいま通過します。6/25の**カンマhook matcherの不発**、6/27の**ハイフンmatcherの修正**と同じ流れで、*自分のガード・フラグが実際に何をしていたか*が更新で変わる種類の変更です。`--dangerously-skip-permissions`を自動化に埋め込んでいるなら、更新後に**危険なコマンドで実際の挙動を一度再現**してみてください。 [GitHub v2.1.196](https://github.com/anthropics/claude-code/releases/tag/v2.1.196)

## セキュリティ・制限事項

### v2.1.196のサプライチェーン・リモート制御ハードニング — 信頼されないrepoのMCP遮断 + non-AnthropicホストのRemote Control無効化

このリリースでセキュリティ観点から最も押さえるべき2点です。

- **`claude mcp list`/`get`が信頼されていないrepoからサーバーをspawnしなくなりました。** これまでは、MCPサーバーの一覧・情報を照会するだけでも、cloneしたrepoの設定が**サーバープロセスを起動**しうる経路がありました — つまり*他人のrepoを開いて見るだけ*で外部コードが走る通路でした。いまその照会はサーバーを起動しません。6/27の**外部プラグインのインストール同意の強制**（settings.jsonによる自動有効化の遮断）と完全に同じ**サプライチェーン**の流れです。
- **Remote Controlがnon-Anthropicホストで無効化されます。** ローカルのClaude Codeセッションをリモートで閲覧・操作するRemote Controlが、Anthropicホストでない環境ではオフになります — 6/25の**Trusted Devices for Remote Control**（リモート操作前のデバイス認証）と同じ場所で、*リモートでセッションに届きうる表面*をもう一段狭める変更です。

他人のrepoを頻繁に開く人や、リモート制御ワークフローを使うチームには即効性があります。どちらも、*エージェント・設定ファイルが自動で持ちうる権限の表面を縮める*最近の流れの延長線上です。 [GitHub v2.1.196](https://github.com/anthropics/claude-code/releases/tag/v2.1.196)

### 輸出規制の続報 — Fable 5の復帰は「無制限」でないかもしれない: 追加料金・本人確認の変数（6/29）

何日も扱った件なので、**新しいディテールだけ**を。6/29時点で**Fable 5は依然、全一般ユーザー（消費者・API・Claude Code・海外サブスクライバー）にオフライン**です。報道（Axios・Capacity）は**政権の姿勢が軟化**し*数日以内*の復帰が取り沙汰されると伝えますが — これは6/27〜29ですでに扱った「within days」の流れの続きです。**本日の新しい変数は*復帰の形***です。

- **フラットな「ON」でないかもしれない**: 報道は、サブスクライバーが**無制限アクセスを取り戻す**のか、それともモデルが**追加料金や本人確認を伴って**戻るのか*まだ分からない*と伝えます。*方向*は改善しましたが、より厳しい*条件*が新たに俎上に載りました。
- **依然としてゲート**: 全面再有効化の前に**Pentagon・NSAの承認**が残っています — 最終スイッチは安全保障当局に握られています。

外国籍の開発者（韓国・日本など）の実務対応は変わりません — `fallbackModel`・`availableModels`で**Opus 4.8のような代替経路**を敷き、モデルの可用性が会社ではなく**地政学・政策、さらには復帰の*条件*（料金・本人確認）に左右されうる**前提で自動化を設計しましょう。今回の新しい教訓は — 復帰を待ちつつ、*「戻る＝以前のまま戻る」と仮定するな*ということです。 [Capacity](https://capacityglobal.com/news/fable5-return-imminent/) · [Axios](https://www.axios.com/2026/06/27/anthropic-fable-5-return-soon) · [Anthropic声明](https://www.anthropic.com/news/fable-mythos-access)

## エコシステム＆プラグイン

### Claude Mythos Previewのサポート終了 — 本日（6/30）引退、移行先（Mythos 5）自体が輸出規制下

**本日（6/30）**、`claude-mythos-preview`モデルが**引退**します — Anthropicのdeprecationスケジュールに従い、この識別子へのリクエストはこれから失敗します。推奨される移行先は**Claude Mythos 5（`claude-mythos-5`）**です。

落とし穴は、**移行経路そのものが塞がれているかもしれないという逆説**です。Mythos 5は6/12の輸出規制でオフになり、6/27時点でライセンスなしで復元されたのは**Annex Aの約100社の重要インフラ企業、米政府、Anthropic自社の外国籍従業員**に限られます。つまりMythos Previewを使っていた多くの外部開発者にとって、*公式の移行先*であるMythos 5は**依然アクセス不可**かもしれません。6/28の**Opus 4.7 fast modeのサポート終了予告**（7/24に削除）と同じ場所で、*モデル・モードのライフサイクルが次第に明示的に管理される*流れですが、今回は**終了と輸出規制が重なり**、「次のバージョンに移るだけ」が全員に開かれてはいません。Previewに依存するスクリプト・CIがあるなら、**まず代替モデルにアクセスできるかどうか**を確認してください。 [Claudeモデルのdeprecation文書](https://platform.claude.com/docs/en/about-claude/model-deprecations)

## 注目プロジェクト＆ツール

- **Framein — 複数のAIコーディングエージェントの*下に敷く*共有作業状態レイヤー**: Claude・Gemini・Codexを行き来して*毎回コンテキストと目標を再要約*する摩擦を解く軽量ツールです。仕掛けは、エージェントの上ではなく*下*に敷くこと — **task contract（作業契約）・決定記録（decision record）・検証結果**を**リポジトリ（repo）に置き**、どのエージェントが来ても*同じ権威ある状態*を読んで引き継げるようにします。つまりモデル間のハンドオフ時に*コンテキストを説明し直す必要がなくなり*ます。まだ**プリリリースv0.0.6**、MITライセンス、`npm install -g framein`でインストールします — 6/29の**サブエージェントのコンテキスト衛生**、6/25の**Monet（ローカルメモリ）**と同じ場所で、*マルチエージェントの共有状態*をrepoの次元に引き上げた試みです。 [GitHub](https://github.com/framein-dev/framein) · [framein.dev](https://www.framein.dev)
- **Orchterm — 複数のコーディングエージェントを*一つの窓*で同時に管理するデスクトップ**: **ターミナル・コードエディタ・ブラウザ・Gitツールを一つのウィンドウに集め**、その上で**複数のAIコーディングエージェントを同時に**動かすデスクトップアプリです。各エージェントを**隔離されたインスタンス**で動かしつつ、**共有タスクボード（task board）**で互いに調整させ、*並列開発*を一つの作業空間で処理します。6/28の**git worktreeの並列セッション**（3〜5個を同時に）、6/24の**gh-orbit**（複数worktreeのPR・CIを一つのダッシュボードに）と同じ流れ — *エージェントを一つずつではなく複数同時に動かす*パターンをGUIに移したツールです。 [Orchterm](https://zendy00.github.io/orch-term-pages/)
