## 開発者ワークフローティップス

### ファイルを丸ごと読ませず「コードグラフ」を渡す — @ttsc/graph でトークン10分の1

**@ttsc/graph** は、**TypeScript コンパイラが既にパースした構造を再利用し、ファイル全文の代わりにコード依存グラフをエージェントに渡す MCP サーバー** です。着眼点が鋭いです — Claude Code がコードを理解しようとファイルを繰り返し grep・read すると、トークンはリポジトリの規模に比例して膨らみます。ここでは **名前(names)・エッジ(edges)・シグネチャ(signatures)・file:line スパンだけ** を送ります。その結果 **応答サイズがリポジトリ規模と無関係** になり、作者は Claude Code で **トークン約10倍の削減** を報告しています。

もう一段あります — エージェントがリクエストを投げる前に **question → draft → review チェーン** を完了するよう強制し、*まずファイルを大量に読んでしまう* 近道を封じます。つまり節約はデータ表現(グラフ)だけでなく **リクエスト前の構造化された推論** にも由来します。

```text
# アイデア(ttsc/graph の解説より)
- 渡すもの: シンボル名 ・ 呼び出し/依存エッジ ・ 関数シグネチャ ・ 定義位置(file:line)
- 渡さないもの: 関数本体の全文(必要なときだけ該当 span を狙って読む)
- 効果: payload サイズがコードベース規模に比例しない → 大規模 TS リポで特に大きな削減
```

7/5 の **pxpipe**(密なコンテキストを画像トークンとして安く入れる)と同じ問題意識 — *モデルに必要なのは正確なバイトではなく理解* という前提 — を、今回は **AST 構造** で解いた例です。TypeScript リポジトリで Claude Code を回すチームなら、大規模リポでコンテキスト予算が厳しいときに試す価値があります(報告値なので代表的なタスクで自ら検証してください)。[dev.to](https://dev.to/samchon/i-made-ts-compiler-graph-mcp-10x-fewer-tokens-in-claude-code-1aea)

### コーディング実行者から「コンテキスト・検証・製品化の設計者」へ

エージェントが **課題特定 → 作成 → テスト → 修正** を自ら一周する時代になり、開発者が価値を生む場所は *コードを打つ手* から *ループを設計する席* へ移ります。ある velog の記事は、これを3つの実務に具体化します。

- **問題定義・受け入れ基準を成果物として残す**: 何を作るか・何が「完了」かを口頭の指示のままにせず、エージェントが読み返せる **ドキュメントやチェックリストとして固定** します(CLAUDE.md・issue・spec)。曖昧な指示は曖昧なコードで返ってきます。
- **コードを書く前に検証手段を敷く**: テスト・型・lint・受け入れ基準といった **外部の真実の軸** を先に置けば、コンテキストが埋まるにつれ鈍るモデルの自己判断の代わりに、*レッド→グリーン* という明確なフィードバックがセッションを通じて保たれます。
- **生成物を「下書き」として扱う**: 動くコードがそのまま良い製品ではありません — UX・構造・保守性は人が見直す必要があります。レビューの焦点を *動くかどうか* から *1か月後に直せるか* へ移します。

肝は **「実行をエージェントに任せるほど、人は問題・基準・検証をより鮮明に所有しなければならない」** という逆説です — 7/4 の *答えはより強い検証ハーネス*、7/5 の *ループエンジニアリング* とまったく同じ場所です。この3つは、ツールなしで今日すぐ CLAUDE.md とレビュー習慣に入れられる実務でもあります。[velog](https://velog.io/@teo/ai-era-developer-role)

## エコシステム＆プラグイン

### shadcn/ui、新規プロジェクトの既定を Radix → Base UI へ変更(7月)

**shadcn/ui が新規プロジェクトの既定コンポーネント基盤を Radix から Base UI(v1.6.0)へ変更** しました。根拠は Base UI の安定性(1.6.0)と採用の流れ — **新規プロジェクトで2:1の選好** と **週間ダウンロード600万超** です。既存の Radix 利用は **維持** されるため急いで乗り換える必要はなく、新規プロジェクトの既定だけが変わります(併せて新しい chat コンポーネントと GitHub registry 機能も追加)。

開発者にとってより興味深いのは **移行の届け方** です — Radix → Base UI の移行を、従来の **codemod ではなく AI エージェントスキル** として提供します。つまりライブラリが *変換スクリプト* ではなく *エージェントが実行するプレイブック* を配布チャネルに選んだわけです。これは Claude Code のようなコーディングエージェントが **大規模なコード移行の第一の道具** になりつつある兆候であり、フロントエンドチームにとっては *エージェントで回す移行* を前提にリポジトリを整えておく理由になります。[出典](https://ui.shadcn.com/docs/changelog)

## 知っておくと便利な小さな変更点

リリースノートで見落としやすい、今週終盤の実用的なディテールです。

- **`.claude.json` の MCP サーバー設定が配列でないと起動クラッシュ → 修正**(v2.1.200): `disabledMcpServers`・`enabledMcpServers` を文字列など **非配列の値** で書くと起動時にクラッシュしていた問題が直りました。この2つのキーを手編集するなら **配列** にしてください。
- **`claude agents` のセッション行が PR リンクを `#N` で表示**(v2.1.199): background エージェントが開いた draft PR のリンクが、セッション一覧に **bare `#N`** として出ます — どのセッションがどの PR を出したか一目で追いやすくなりました。
- **7/8 本人・年齢確認の発効まで D-2**: 6/23・7/5 で扱った消費者プライバシーポリシーの確認条項が **明後日(7/8)に発効** します。対象は **個人 Free・Pro・Max のフラグ付き少数アカウント** のみで、**Enterprise・Team・API・Claude for Work は除外** です — 個人アカウントでクリティカルな自動化を回している場合、2日後に確認プロンプトが流れを止め得る点だけ意識してください。
- **Sonnet 5 の導入価格は 8/31 まで**: 既定モデル Sonnet 5 の導入価格 **100万トークンあたり2ドル/10ドル** は **8月31日** まで、以降は3ドル/15ドルです — 大量バッチ・評価ジョブの日程を組む際の参考に。

## おすすめコラム＆読み物

- **「何かを学んでみるのは良いかもしれない」**: 大人でも **1日30〜45分** あれば、ピクセルアート・タイピング・3D・音楽・言語といった実用スキルを **意味あるかたちで身につけられる** という記事です(marginalia.nu)。核心の洞察は *向上は練習時間そのものではなく、その合間の睡眠で固まる* という点 — だから毎日少しずつが詰め込みに勝り、時間はスクロールや受動的な消費からほんの少し削ればよい、というものです。学びを *回復力と主体性への生涯投資* と捉える見方は、エージェントが実行を代替する時代の *何を学び続けるか* という問いと重なって読めます。[原文](https://www.marginalia.nu/log/a_112_learning/)
- **「ひとり起業家の時代」**: 米国・世界で **高所得の1人事業(年10万ドル超)が急増** している現象が、統計の歪みや詐欺ではなく **実際の経済活動** だとする分析です(Stripe の経済チーム)。核心の論旨は *AI と進化したソフトウェアが、かつて人を雇って埋めていたスキルの空白を埋め* 、起業の参入障壁が根本的に下がった、というもの — 上の「設計者としての開発者」の実務と併せて読むと、*ひとりでもチーム規模の成果を出す* 個人が増える流れのマクロな背景です。[原文](https://www.stripe.com/economics/the-age-of-the-solopreneur)

## 注目プロジェクト＆ツール

- **OpenTag — Slack 版 Claude Tag のオープンソース・セルフホスト代替**: Anthropic が Team・Enterprise に提供する **Slack 内の Claude Tag**(Slack の会話で Claude をタグ付けしてタスクを委任)に対する **オープンソースの代替** です(CopilotKit)。*ベンダーロックインを避けたい* という問題意識から生まれ、**ランタイムを自ら所有し LLM を選べる** 構成をとります — Slack スレッドを読み・質問に答え・ツールを呼び出し、**結果を会話の中に直接レンダリング** し、同じコードが **Discord・Telegram・WhatsApp** でも動きます。**human-in-the-loop 承認** フローと **generative UI** を備え、**MIT ライセンス** なのでローカル配備と全面カスタマイズが可能です。Anthropic が SaaS として売るチャネル型エージェントを、*自分のインフラで自分のモデルで* 動かしたいチームに直接届く道具です。[GitHub](https://github.com/CopilotKit/OpenTag)
