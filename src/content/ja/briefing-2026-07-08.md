## 主要な新機能と実践活用

### Claude Cowork が Web・モバイルへ — デスクで始めてスマホで引き継ぐ (7/7)

**7/7、Anthropic が Claude Cowork を Web とモバイルに拡張**しました。これまでデスクトップに縛られていた Cowork のセッションとファイルを、**どのデバイスからでも**続けられるようになります — **Max ユーザーから**数週間かけて beta としてロールアウトされます。

- **クロスデバイスの継続性**: デスクで作業を始め、スマホで進捗を確認し、完成した成果物をどこでも引き継げます — ノートPCを閉じていても。
- **バックグラウンド・スケジュール実行**: デバイス接続がなくても作業が続き、予約したタスクは自律的に回ります(例: 月曜午前6時のクライアント向けブリーフィングが、寝ている間にメールスレッド・議事録・ニュースリサーチを集めて準備)。
- **判断は人に**: 人の決定が必要な場面では、Claude が**スマホに質問を上げて**、会議中でも作業の方向を修正できるようにします。

通底しているのは、**エージェントの実行がデスクトップセッションから離れ、どこでも引き継げるバックグラウンド作業へ移っている**という点です — Claude Code のバックグラウンドエージェントや Remote Control(ローカルセッションをスマホ・ブラウザから操作)とまさに同じ方向です。ひねりはこうです: Anthropic 自身が**Cowork 利用の90%以上は非開発業務**(業務運営・コンテンツ)だと明かしています。コードで評価を得たエージェントが、オフィス全体へ広がっている流れのプロダクト側の兆候です(業界のフレーミングはコミュニティニュース参照)。 [Anthropic 発表](https://claude.com/blog/cowork-web-mobile)

### v2.1.203 の新規項目 — ログイン失効警告と MCP 作業ディレクトリ (v2.1.203)

v2.1.203 が加えたユーザー向け機能は小粒ながら実用的です。

- **ログイン失効警告**: ログインがまもなく失効するときに**事前に警告**し、バックグラウンドセッションが認証失効で切れる前に再認証できるようにします — 7/6 の claude.ai 障害で OAuth ログインも一緒に塞がった件(7/7 のブリーフィング)を思い出すと、無人セッションの認証断を減らすうれしい安全策です。
- **手動権限モードの ⏸ バッジ**: フッターにグレーの ⏸ バッジを表示し、**いま手動権限モードかどうか**を常に見えるようにします — 既定の権限モードが Manual に変わった v2.1.200(7/4 のブリーフィング)の後続で、モードの混同を減らします。
- **MCP `roots/list` への追加作業ディレクトリ**: セッションの追加作業ディレクトリを MCP `roots/list` に公開し、集合が変わると `notifications/roots/list_changed` を送ります — MCP サーバーがマルチディレクトリのワークスペースを正しく認識するようになります。
- **VSCode の全セッション Remote Control トグル**: VSCode 設定に「全セッションで Remote Control を有効化」トグルが追加されました。

[GitHub v2.1.203](https://github.com/anthropics/claude-code/releases/tag/v2.1.203)

## 開発者ワークフローティップス

### バックグラウンド・worktree の自動化を新しい修正に合わせて再点検する (v2.1.203)

v2.1.203 は、バックグラウンドエージェントと git worktree の利用者に直接届く修正を大量に含みます。無人・並列で Claude Code を回しているなら、そのままチェックリストです。

- **`ANTHROPIC_BASE_URL` 脱落の修正**: バックグラウンド・agent-view セッションがシェルで export した `ANTHROPIC_BASE_URL` を落とし、**API キーが既定エンドポイントへ行って401で失敗**する問題が修正されました — カスタムゲートウェイ・プロキシを使うチームなら、原因不明の401はここが出所かもしれません。
- **Windows PATH 継承の修正**: バックグラウンドエージェントがディスパッチしたシェルではなくデーモンの古い `PATH` を継承し、**ツールが消える**問題を解消。
- **worktree 分離の修正**: worktree 分離されたサブエージェントが自分の worktree ではなく**親チェックアウトでシェルコマンドを実行していた**問題、マルチリポのワークスペースでネストしたリポの worktree 作成が拒否される問題を、いずれも修正。
- **デーモン自動アップグレードの安定化**: バックグラウンドデーモンの自動アップグレード失敗が**実行中のすべてのバックグラウンドセッションを黙って落とす**問題を解消。
- **macOS ストールの修正**: 誤った低メモリ検知でバックグラウンドセッションの起動・切り替えが**15〜20秒止まる**リグレッション(2.1.196)を修正。

パターンはおなじみです — **無人パイプラインの静かな失敗ポイントがもう一層剥がされた**ということで、7/4 のブリーフィングのサブエージェント静かな失敗修正と同じ場所です。カスタム base URL や多数の worktree を使う自動化があれば今回のリリースへ上げておき、特に原因不明の401やツール消失を経験していたら再現するか確認してください。 [GitHub v2.1.203](https://github.com/anthropics/claude-code/releases/tag/v2.1.203)

### サブエージェントの再委譲を再び信頼しつつ、リサーチは Haiku で回す

v2.1.203 は**サブエージェントが自分のタスクを丸ごと別のサブエージェントに再委譲する傾向を減らしました。** これまでサブエージェントに重いリサーチを任せると、*仕事をする代わりにまた他へ回す*無駄がありましたが、この修正で委譲が一段と予測可能になりました。ここにパワーユーザーの実践2つを重ねれば、実戦レシピになります。

- **リサーチ・探索系サブエージェントは Haiku で**: コードベースをなめるサブエージェントはファイルを大量に読んでコンテキストを食います — `model` フィールドで **Haiku を指定**してリサーチを安く回し、**Opus はメインスレッド**に温存してください。コンテキストが根本的な制約である状況では、サブエージェント自体がコンテキスト節約のツールです。
- **成功の主張ではなく証拠を求める**: **テスト出力、実行したコマンドとその結果、スクリーンショット**といった機械的証拠を要求すれば、見ていなかったセッションも検証が速くなります — 再実行するより証拠を確認するほうが速いのです。

要点は**委譲は信頼性が確保されて初めて得になる**ということです。リリースがインフラ(暴走的な再委譲の抑制)は直してくれますが、*何を委譲し何を証拠として受け取るか*は依然として人の設計です — 7/6 の「設計者としての開発者」、7/5 の検証ループと同じ場所です。 [Claude Code パワーユーザーティップス](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)

## セキュリティ・制限事項

### SkillCloak — 悪性 Agent Skill が静的スキャナを90%超回避 (7月)

香港科技大学(HKUST)の研究チームが、論文 **「Cloak and Detonate: Scanner Evasion and Dynamic Detection of Agent Skill Malware」** で、悪性 **Agent Skill** が静的スキャナをすり抜けるように設計されたフレームワーク **SkillCloak** を公開しました。Agent Skill がスタック型で組み合わさり(v2.1.199)、ライブラリマイグレーションまでスキルとして配布される(7/6 の shadcn)流れの中で、**スキルのサプライチェーンセキュリティ**を正面から突く結果です。

- **2つの回避技術**: ① **構造的難読化** — ペイロードの目立つ指標を*意味的に等価な形*へ書き換え、パターンマッチ規則をかわします。② **self-extracting packing** — ペイロード全体を**スキャナがスキップするディレクトリ(例: `.git/`)** へ移し、無害に見えるデコーダが**エージェント実行時にだけ**スキルを再構築します。
- **実測の威力**: **実在の悪性スキル1,613件**でテストした結果、一部スキャナを**90%超回避**しつつ、マルウェアの機能はそのまま保持しました。偽装されたスキルは **Claude Code・Codex のようなプロダクションエージェントで正常動作**しました。
- **対策**: 論文はあわせて **SKILLDETONATE** を提案します — **サンドボックス内でファイルアクセス・データフロー・ネットワーク活動を実行時に監視**する監査器です。

要点は**サードパーティ製スキルを静的検査だけで信用するな**ということです — 7/4 の「AI と共にした冒険」の結論(*静的解析・サンドボックスのような検証ハーネスに投資せよ*)とまさに同じ場所であり、下の「注目プロジェクト」の Mirror Stack(事後検証)とも同じ問題意識です。外部から受け取った Agent Skill・プラグインを使うなら、出所の不確かなスキルは**まず隔離・サンドボックスで動かして**ネットワーク・ファイルアクセスを観察してください。 [The Hacker News](https://thehackernews.com/2026/07/new-skillcloak-technique-lets-malicious.html)

## 知っておくと便利な小さな変更点

v2.1.203 の細かいながら実用的な修正と、リマインダーです。

- **`claude agents` に戻ってもサブエージェントの作業が引き継がれる**: `claude agents` に戻ると、実行中だったサブエージェントが**黙って中断されプロンプトを最初から再実行**する問題を修正 — いま作業が引き継がれます (v2.1.203)
- **コンテキスト指標のメモリ・CPU リグレッションを修正**: コンテキスト使用量の指標が**毎ターン、トランスクリプト全体を再解析**する問題を解消 (v2.1.203)
- **古いセッショントークンが自動回復**: デーモンのセッショントークンが古くなり attach・応答・stop に**恒久的に無反応**になっていたバックグラウンドセッションが、自ら回復します (v2.1.203)
- **バイナリ・起動メモリを約7MB 削減**: 大きなバンドル依存をインライン展開せず遅延ロード (v2.1.203)
- **多数 worktree 時の Bash 失敗を修正**: worktree が多いリポで Bash が「argument list too long」で失敗する問題を解消 (v2.1.203)
- **ビュークローズのキー変更**: 左矢印でバックグラウンド・diff・workflow 詳細ビューを閉じる動作を廃止 — いまは **Esc** (v2.1.203)
- **起動時警告の移動**: 起動時の「claude command missing or broken」警告を削除 → **`/doctor`・`/status`** へ移動 (v2.1.203)
- **本人・年齢確認は本日(7/8)発効**: 対象は個人 Free・Pro・Max のフラグ付き少数アカウントのみで、Enterprise・Team・API は除外 — 背景は 6/23・7/5 のブリーフィング参照
- **Fable 5 クレジット移行は昨日(7/7)発効**: 使用量クレジットを有効化していなければ Fable 5 へのアクセスが停止 — 詳細は 7/7 のブリーフィング参照

## 注目プロジェクト＆ツール

- **gh-attach — CLI で GitHub の Issue・PR に画像・ファイルを添付**: GitHub にはファイルアップロードの公式 API がなく、Issue・PR にスクリーンショットを貼るには Web UI を開く必要がありました。**gh-attach** はこの隙間を巧みに回避します — **ローカルブラウザに保存されたログインクッキーを読み、Web と同じアップロード API を呼んで**、ターミナルからそのまま添付します。Chrome・Firefox・Safari に対応し、ログイン中のアカウントとセッションを自動で合わせ、JSON・テンプレートのオプションで**スクリプト化**もできます。Go で書かれ、`gh` 拡張としても単体バイナリとしても配布され、Homebrew でインストールします。Claude Code が draft PR を自動で開く(v2.1.198)流れの中で、*エージェントが作った PR に証拠スクリーンショットを CLI で貼る*ワークフローに直結するユーティリティです。 [GitHub](https://github.com/sudosubin/gh-attach)
- **Mirror Stack — ループエージェントの誠実さを「約束」ではなく「証明」へ**: AI エージェントのセキュリティを*事前のゲートキーピング*から**事後検証**へ移す、ローカルファースト・Apache-2.0 のフレームワークです。不正を防ごうとする代わりに、**誠実な行動だけが改ざん不可能な監査記録を残す**ように設計します — 4つのツールが*主張(claim)・行動(action)・成果物(artifact)・周辺の証人(witness)*をそれぞれ検証します: **measure-mirror**(23の統計・アンチゲーミングプローブで主張を検証)・**action-mirror**(チェーン連結された行動履歴)・**provenance-mirror**(生成物の来歴を機械的に検証)・**mirror-witness**(GitHub Actions・CI 環境を別インフラなしで分散証人として活用)。**MCP を公式サポート**し、pip インストールだけで Claude・Cursor・Windsurf に組み込めます。上の SkillCloak(静的検査の脆弱性)と 7/4 の「検証ハーネスに投資せよ」が投げた問題を、*事後に証明する*側から受ける事例です。 [GitHub](https://github.com/mirror-stack)
