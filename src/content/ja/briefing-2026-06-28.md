## 主要な新機能と実践活用

### Trusted Devices for Remote Control — ローカルセッションをリモート操作する前にデバイスを認証させる (6/25)

CLIリリースは止まっていますが、Claudeアプリ側で**Claude Codeのリモート制御**に直結する管理者機能が追加されました。**6/25**より、Team・Enterpriseプランの管理者が**Trusted Devices for Remote Control**を有効化すると、メンバーが**ローカルのClaude Codeセッションをリモートで閲覧(view)・操作(steer)する前に、自分のデバイスを認証する**ことを要求できます。

要点は、**誰が・どのデバイスで自分のローカルセッションに遠隔から触れるか**をゲートで制御することです。Claude Codeには、席を外している間でも**ローカルで動くセッションを別デバイスから覗き、方向づける**フローがありますが、その入口に**デバイス信頼(device trust)**の一段を挟みます — 認証されていないデバイスはセッションを閲覧も操作もできません。直近の流れと同じ位置づけです — 6/23の**`sandbox.credentials`**(サンドボックスからのシークレットアクセス遮断)、6/27の**外部プラグインのインストール同意の必須化**と並んで、*エージェントやセッションに到達しうるアクセス面を管理者が中央から狭める*方向の一歩です。チームでClaude Codeセッションを共有・操作するワークフローを使うなら、導入前に**どのデバイスを信頼するか**をポリシーとして定めておきましょう。[Claudeリリースノート](https://support.claude.com/en/articles/12138966-release-notes)

## 開発者ワークフローティップス

### 最大のアンロックは並列セッション — git worktreeで3〜5個を同時に回す

Anthropicのパワーユーザーガイドが挙げる**第一の生産性アンロック**は、機能ではなく運用の仕方です — **3〜5個のClaudeセッションを、それぞれ自前のgit worktreeで並列に**回すこと。あるセッションが考えて実装している間、別のセッションが別の作業を進めるので、*待ち時間*がそのまま*別タスクの進行時間*になります。

```bash
# 各セッションを隔離されたworktreeで起動し、名前を付けて区別する
claude --worktree

# 大規模なマイグレーションは /batch — 各エージェントが隔離されたまま
# 自分の変更をテストし、PRを独立して作成する
/batch
```

要点は**衝突しない並列化**です。worktreeで分けておけば、複数のセッション・エージェントが同じファイルを同時に触っても互いの作業を壊しません。6/24の**gh-orbit**(複数worktreeのPR・CIを1つのダッシュボードに)、6/20の**autoモード + worktree隔離**と対になる運用です — エージェントを*1つずつ順番に*ではなく*複数のレーンで同時に*走らせたいなら、worktreeの分離が出発点になります。[Claude Code power-user tips](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)

### 検証は作業したモデルではなく別のモデルに任せる — 作業者と採点者を分ける

二つ目は**検証の信頼性**を高めるティップスです。エージェントが自ら「完了しました」と報告しても鵜呑みにせず、**結果を作ったモデルではなく、新たに起動した別のモデルにその結果を反証(refute)させ**ましょう。*作業したエージェントが自分の作業を採点しない* — この分離が肝心です。人間のコードレビューで作成者とレビュアーを分けるのと同じ原理です。

```text
# 変更分を自動チェック(再利用・品質・効率・CLAUDE.md準拠)
/simplify

# その上で、検証専用のサブエージェント/ワークフローでもう一度:
#  - 結果を作ったセッションではなく、新しいモデルに
#  - 「この結論は誤りだと仮定して反証せよ」と指示する
```

理由は明快です — 同じモデルが同じコンテキストで自分の成果を見直すと、**自分が見落としたものを同じように見落とす**可能性が高いのです。新しいモデル・新しいコンテキストで*反証を試みさせる*と、その偏りが崩れ、誤った「完了」報告がふるい落とされます。6/25の**ハーネスエンジニアリング**(テスト・リンター・承認ゲートで非決定性を統御)、6/24の**Slackのエージェンティックテスティング**(目標達成の検証)と同じ筋です — *エージェントの出力を、もっともらしい物語ではなく検証可能な根拠へ戻す*一手です。[Claude Code best practices](https://code.claude.com/docs/en/best-practices)

## エコシステム＆プラグイン

### Opus 4.7のfast mode終了予告 — 6/25にdeprecated、7/24に削除(Opus 4.8へ)

fast modeを使う人が必ず押さえるべき**サポート終了予告**です。**Opus 4.7のfast modeが6/25付でdeprecated**となり、**7/24に削除**されます。削除後は、**`claude-opus-4-7`に`speed: "fast"`で送ったリクエストがエラーを返し**、標準のOpus 4.7に**自動フォールバックしません**。

```text
# 7/24以降: claude-opus-4-7 + fast mode = エラー(標準4.7へのフォールバックなし)
# 対応: fast modeのワークロードをOpus 4.8 fast modeへ移す
```

要点は**「静かに壊れる」自動化を未然に防ぐ**ことです — fast modeをOpus 4.7に固定したスクリプト・CI・MCPワークフローがあれば、7/24を境にその呼び出しは*フォールバックなしでそのまま失敗*し始めます。6/24の**組織単位のモデル制限**、6/20の**モデルの自動入れ替え/deprecation**と同じ位置づけです — モデルとモードのライフサイクルが次第に明示的に管理されるなか、**どのモデル・どのspeedに依存しているか**を今のうちに点検し、Opus 4.8へ移しておきましょう。[Claudeモデルのdeprecation文書](https://platform.claude.com/docs/en/about-claude/model-deprecations) · [fast mode文書](https://code.claude.com/docs/en/fast-mode)

## おすすめコラム＆読み物

- **「オープンウェイト対クローズドウェイトLLMの差」**: オープンモデルが閉鎖モデルに*どこまで追いついたか*を問う記事ですが、核心の洞察は**「測る指標で結論が変わる」**です — **単一の知能指数(intelligence index)**で見ればオープンは**2026年末には追いつく**ように見えますが、**18のベンチマーク**に広げると平均の差は**約5か月でほぼ横ばい**で、その改善すら**コーディング課題に集中**しています。つまり同じデータから*合理的な人々が正反対の結論*に至りうるということ — 「オープンがもうすぐ追いつく」という物語を*どの指標で描いたのか*を問い直させる記事です。本日の輸出禁止・アジア代替モデルの流れと併せて読むと、**「フロンティアの差」を1つの数字で断じるな**という重しになります。[原文](https://blog.doubleword.ai/frontier-os-llm)
- **「物理メディアの所有がなお重要な理由」**: デジタルで何かを*購入*するとは、実は**所有ではなく、いつでも取り消されうるライセンス**であることを突く記事です — サービス終了・ポリシー変更・アカウント停止があれば、*買ったと思っていたもの*が消えます。根拠が具体的です: **Sonyの2026年のStudio Canal作品の削除**(返金なし)、**UbisoftのThe Crew**(サーバー終了でディスク所有者もプレイ不可)、**Netflixの250本超の削除**(自社の「オリジナル」を含む)。開発者の文脈では、本日の2つの話題と正確に共鳴します — 輸出規制で**モデルへのアクセスが一夜で消える**(上記Fable 5)ことと、**接続したサービスが自分の資源に手を出す**事例(上記のClaude決済詐欺)。*自分が「使っている」と思うもののうち、実際に自分の制御下にあるのは何か*を問わせます。[原文](https://dervis.de/physical/)
- **「フィンテックエンジニアリングハンドブック」**: お金を扱うシステムを設計する際の**3つの原則 — データを作り出さない(重複処理の禁止)・データを失わない(全取引を監査)・何も信頼しない(すべて検証)**をまとめた実践ガイドです。処方が具体的です — **金額は浮動小数点ではなく整数(最小単位)・BigDecimal・有理数で**、**不変の監査証跡**(value time・booking time・settlement timeを分離)、**外部連携はすべて不信**(webhookは*トリガー*であって真実の源ではない)、**冪等(idempotent)な操作と永続的なステートマシン**で処理途中のクラッシュからも復旧、**整合性照合(reconciliation)ジョブ**で台帳と外部プロバイダのドリフトを検出。AIがコードを速く吐き出すほど、**「何が絶対に間違ってはいけないか」を知るドメイン知識**の価値が増す — エージェントに決済・精算コードを任せる前に一読しておきたいチェックリストです。[原文](https://w.pitula.me/fintech-engineering-handbook/)

## 注目プロジェクト＆ツール

- **CerberusBeacon — 「必要なときだけ開き、アイドルで閉じる」セルフホスト型Webターミナル**: サーバーへリモート接続する際、**ポートフォワーディング・固定IP・VPNなしで**、**Cloudflareトンネル経由で必要なときだけWebターミナルを開き、アイドルになると自動で閉じる**セルフホストツールです。ブラウザベースの**ターミナルとファイル管理**を提供し、**Telegram/Slackボットによる遠隔操作**もできます。要点は**「常時開いたアクセス面」を減らす**こと — 常時稼働のSSH/Webコンソールの代わりに、*使うときだけ一時的に開く*モデルで露出を最小化します。本日のTrusted Devicesや決済詐欺の事例と同じ筋で、**リモートアクセスの常時露出を減らしたい**セルフホスターに実用的です。[GitHub](https://github.com/TaewonyNet/CerberusBeacon)
- **macSandbox — Apple Silicon Mac向けの「使い捨てWindows 11サンドボックス」**: **QEMUとHypervisor.framework**を使い、Apple Silicon Macで**ARM64のWindows 11 VMを数秒で**起動するツールです。VMは**埋め込みRDP**で表示され、**終了するとすべての変更が破棄**されます — macOS版の「Windows Sandbox」というわけです。出所の怪しいインストーラや文書を、**捨てて終われる隔離環境で確認**したいときにそのまま役立ちます(上記の決済詐欺の事例のように、*信頼できないものは隔離して扱う*という原則のデスクトップ版)。検証・テスト用の使い捨てWindowsが必要なMac開発者に実用的です。[GitHub](https://github.com/yourtablecloth/macSandbox)
