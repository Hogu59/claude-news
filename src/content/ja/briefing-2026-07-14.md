## 主要な新機能と実践活用

### LTM が Anthropic と戦略提携 — Claude Code を BlueVerse に載せて企業配送へ (7/13)

Anthropic 自身の新機能・リリースはない日ですが、*導入を実行する側*が動きました。IT サービス企業 **LTM が Anthropic と戦略的提携**を結び、**Claude・Claude Code・Claude Cowork を自社のデリバリーワークフローに組み込む**と発表しました(7/13)。

- **どこに載せるか**: LTM のエンタープライズ実装プラットフォーム **BlueVerse AI Delivery Fabric** を土台に、ソフトウェアエンジニアリング・アプリケーション近代化・SRE 運用へ Claude を載せます。
- **ガバナンスを製品化**: 専用の **Center of Excellence(CoE)** が、再利用可能なスキルやリファレンスアーキテクチャとともに、*責任ある利用・エージェントのライフサイクル・モデルガバナンス・データプライバシー/レジデンシー準拠*のフレームワークを作ります。
- **人材を付ける**: **AI1000 人材プログラム**で **Claude 認定アーキテクトと Forward Deployed Engineer を数千人**育成・投入し、顧客を評価→実装→継続的改善の段階へ導きます。対象は **BFSI(金融)・ハイテク・コンシューマー・製造**の大企業で、掲げるのは*パイロットから本番へ*です。

実践活用の観点では、**これは『機能』ではなく『導入経路』が広がったニュース**です — 規制産業(金融・製造)で Claude Code の採用を検討する組織にとって、*モデルや CLI をどう使うか*と同じくらい、*誰がガバナンス・リファレンスアーキテクチャ・現場エンジニアを供給するか*が実際のボトルネックです。7/11 の **UST 提携**とまったく同じ形で、アルバータ(7/6)・カリフォルニア(7/5)に続く、**Anthropic の SI チャネルが密になっていく**今週のもう一片です。自前で CoE を立てる余力のないチームには、こうした提携が束ねる*認定アーキテクト・FDE・ガバナンステンプレート*が、導入リスクを下げるレバーになります。[Yahoo Finance](https://finance.yahoo.com/technology/ai/articles/ltm-partners-anthropic-accelerate-claude-113000192.html)

## 開発者ワークフローティップス

### 7/19 まで延びた『窓』を自分のワークロードのベンチマークに使う

Fable 5 のサブスク込み分と週次+50%制限が **7/19 まで延長**された(下記セキュリティ)ということは、今週が実質的に**フロンティアモデルを自分の作業で無料に試せる最後の窓**だという意味です。締め切りをストレスではなく計測の機会に反転させましょう。

- **自分の作業で A/B する**: パイプラインの代表的なタスク(リファクタリング・テスト生成・不慣れなアルゴリズム)を **Fable 5 と安いティアの両方**で回し、実際の red→green 通過率や修正の往復回数を計測します。7/12 の*タスク種別でモデルをルーティング*、7/10 の*ベンチではなく自分の作業で測り直せ*と同じ位置です — 表面レートではなく*自分のトラフィックでの成果*が判断材料です。
- **使い続けるなら今準備する**: 7/19 以降、Fable 5 は**クレジット専用(入力10・出力50ドル)・猶予なし・自動フォールバックなし**です。クレジットを先払いし、残高が尽きたときにどのモデルへ落とすかの**明示的なフォールバックを今のうちに設定**しましょう — 切替当日に無人パイプラインが静かに失敗するのを防ぎます。

```bash
# 同じ代表タスクを2つのモデルで計測(例)
claude --model claude-fable-5   # フロンティアティア — 不慣れ/高難度タスク
claude --model claude-sonnet-5  # 日常作業のベースライン
/usage-credits                  # 残高確認・クレジット先払い
```

要点は、**延長は決定を先送りする合図ではなく、決定をデータで下すための窓**だということです。下のコラム*外側のループを所有せよ*の精神そのままに — モデル選択という判定を、ベンダーの告知ではなく自分の計測で握りましょう。[Forbes](https://www.forbes.com/sites/sandycarter/2026/07/13/claude-fable-5-extends-to-july-19-7-days-7-power-moves/)

### チェックポイントと `/rewind` — 無人・エージェントセッションの『安価な取り消し』

auto mode がクラウドのデフォルトになり(7/11)、エージェントの書き込み権限が増える流れの中で、*誤った一歩を安価に取り消せること*が安全の基礎になります。Claude Code はこれを標準で提供します。

- **プロンプトごとのスナップショット**: Claude Code は変更前の状態を**自動でチェックポイント**します。**ダブル Esc**(または `/rewind`)で rewind メニューを開き、**会話だけ・コードだけ・両方**を選んで以前の地点へ巻き戻せます。
- **エージェントセッションで特に価値がある**: 無人実行が6手目で方向を外したと気づいたとき、最初からやり直したり手で巻き戻したりせず、**望むチェックポイントへ復元**すればよいのです — 大胆に委譲しつつ、失敗の原状回復コストをほぼゼロにする道具です。

```text
Esc Esc      # (または /rewind) rewind メニューを開く
→ 巻き戻す範囲を選択: 会話 / コード / 両方
→ 復元するチェックポイントを選択
```

これは下の*外側のループを所有せよ*と同じ原理です — 人間がデプロイの判定と責任を握るには、*エージェントの一歩をいつでも取り消せる*必要があります。rewind はその『取り消し権限』を UI にしたものです。[Claude Code パワーユーザー向けヒント](https://support.claude.com/en/articles/14554000-claude-code-power-user-tips)

## おすすめコラム＆読み物

- **「外側のループを所有せよ(Own the Outer Loop)」**: エージェントが自律的な作業をこなすほど、エンジニアは**システムの境界で責任を握り続けねばならない**という Addy Osmani の記事です。核心は、**内側のループ**(エージェントが調査→実装→検証を繰り返す、ツール・メモリ・権限・サンドボックス・テストに囲まれた実行空間)と、**外側のループ**(人間が証拠を検討し、*この成果をデプロイすべきか*を判定し、結果に責任を負う監督空間)の区別です。実践指針は三つ — ① **品質シグナルと判定**: 型チェック・テスト・監査ログ・モニターが back-pressure を生み、*モデルではなく人間*が本番投入前の最終判定を下す。② **趣味判断を批評で操作化**: 基準を明示し、事例で練習し、根拠を文書化して、直感的な選好を*教えられるフレームワーク*に変える。③ **説明責任を設計**: 何を確認し、どの証拠が決定を正当化したかの監査証跡を残す。一文がこの記事を要約します — *誰かが、何が変わったのか、なぜ安全だったのか、間違っていたら何が起きるのかを正確に説明できねばならない。* 7/13 の*plan mode の段階ゲート*、そして上の `/rewind` ティップスとまったく同じ位置です — *速度はエージェントに委ね、判定と取り消しは人間が握る。* [addyo.substack.com](https://addyo.substack.com/p/own-the-outer-loop)
- **「コードベースを完全に理解できなくてもよい理由」**: *エンジニアはコードベース全体を完全に理解すべきだ*という通念に反論する Sean Goedecke の記事です。数百万行規模のシステムでは**完全な理解はそもそも不可能**であり、*部分的だが正確な理解*で効果的に働くことは失敗ではなく**現実的な標準**だ、というのです。二つの文化が分かれます — 低離職・小規模のコードベースで働く人は*当然、完全に理解すべき*と言い、高離職・大規模で働く人は*全部は無理だから、自分のローカル領域で最善を尽くすだけ*と言います。方法は**一つのフローを端から端まで理解し、そこから少しずつ枝を伸ばす**ことです。上の*外側のループを所有せよ*と対にして読むと命題が鮮明になります — 全体を頭に収められないなら、規律は*全知*ではなく、**境界での判定と検証を所有すること**へ移ります。エージェントが大規模リポジトリを代わりに読む時代に、とりわけ実務的です — 人間の仕事は*すべての行を知ること*ではなく、*変わったものが安全かを判定すること*です。[seangoedecke.com](https://www.seangoedecke.com/in-defense-of-not-understanding-your-codebase/)

## 注目プロジェクト＆ツール

- **LDBD — AI ボット12個に2か月間の株価方向を予測させ、全て公開検証**: *機械学習が本当に単純な戦略に勝つのか*を、**変更不可・タイムスタンプ付きの予測**で公開検証したプラットフォームです。予測は投稿時点で封印されて後から編集できず、実際の修正後終値と照合して**自動採点**され、全検証履歴が残る透明なリーダーボードで公開されます。作者は **AI ベースのボット12個とベースラインボット18個**(例: 無条件の強気ベットのような単純戦略)を2か月間並走させ、*ML が本当に素朴なベースラインを上回るか*を観測しました。最近のブリーフィングが繰り返した*報じられた数字は自分の目で検証せよ*(SWE-Bench Pro の欠陥 7/10、TryAI の12モデル比較 7/12)と同じ精神で、今回の対象はコーディングのベンチではなく*予測そのものの公開反証可能性*です。上の*外側のループを所有せよ*が説く**説明責任(answerability)**を、株価予測という騒がしいドメインでインフラとして実装した小さな事例と読めます。[LDBD](https://ldbd.app)
