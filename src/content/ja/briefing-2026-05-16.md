## 主要な新機能と実践活用

### v2.1.143 — プラグイン依存性管理、バックグラウンドセッションの柔軟性強化 (5/15)

v2.1.143はプラグインエコシステムの安定性とバックグラウンド作業の柔軟性を向上させるアップデートです。

**プラグイン依存性管理**: `claude plugin disable`が、他の有効なプラグインが依存しているプラグインを無効化しようとした場合に拒否するようになりました。依存チェーンを一括で解除するコピー＆ペースト可能なコマンドヒントも表示されます。逆に`claude plugin enable`は必要な依存関係を自動的に有効化します。

**マーケットプレイストークンコスト表示**: `/plugin`マーケットプレイスのブラウズ画面に、各プラグインの推定コンテキストコスト（トークン見積もり）が表示されるようになりました。インストール前にコンテキストウィンドウへの影響を事前に把握できます。

**`worktree.bgIsolation: "none"`設定**: バックグラウンドセッションがworktreeを作成せずに作業ディレクトリを直接編集できるようになりました。サブモジュールが多い、またはworktreeが実用的でないリポジトリで有用です。

```json
{
  "worktree": {
    "bgIsolation": "none"
  }
}
```

**PowerShell `-ExecutionPolicy Bypass`**: PowerShellツールがデフォルトで`-ExecutionPolicy Bypass`を渡すようになりました。既存の実行ポリシーを維持するには`CLAUDE_CODE_POWERSHELL_RESPECT_EXECUTION_POLICY=1`環境変数で無効化できます。

**バックグラウンドセッションのモデル/エフォート保持**: バックグラウンドセッションがアイドル状態から復帰した後も、設定済みのモデルとエフォートレベルを正しく保持するようになりました。

[GitHub v2.1.143](https://github.com/anthropics/claude-code/releases)

## 開発者ワークフローティップス

### CLAUDE.mdは「アドバイス」、Hookは「強制」— ルール分離戦略

Claude Codeで最も多い失敗の一つは、すべてのルールをCLAUDE.mdに入れることです。2026年のベストプラクティスガイドが共通して強調する重要な区別があります。

**CLAUDE.md（アドバイザリー、約80%遵守）**: コーディングスタイル、アーキテクチャガイド、命名規則など「守れると良い」ルール。コンテキストが複雑になるとClaudeが忘れる可能性があります。

**Hook（決定論的、100%実行）**: リント、フォーマット、危険なコマンドのブロックなど「必ず守るべき」ルール。終了コード0は許可、2はブロックで、例外なく毎回実行されます。

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write",
      "hooks": [{
        "type": "command",
        "args": ["prettier", "--write", "$CLAUDE_FILE_PATH"]
      }]
    }]
  }
}
```

**実務基準**: 「Claudeがこのルールを一度でも無視したら問題になるか？」— Yesならフック、NoならCLAUDE.mdです。

[SmartScope](https://smartscope.blog/en/generative-ai/claude/claude-code-best-practices-advanced-2026/) | [Claude Code Hooks Docs](https://code.claude.com/docs/en/hooks)

### サブエージェントコンテキスト分離 — 「探索汚染」を構造的に防止

大規模コードベースでClaudeが探索と実装を同じセッションで行うと、失敗したアプローチがコンテキストに蓄積され、後続の試行品質が低下します。`.claude/agents/`ディレクトリに探索専用サブエージェントを定義すると、探索が別のコンテキストウィンドウで行われ、**構造的に汚染を防止**します。

核心は「並列化」ではなく「コンテキスト分離」です。失敗した探索パスが実装セッションに影響しないため、複数のアプローチを試した後でも実装セッションはクリーンな状態を維持します。

[Claude Blog: Large Codebases](https://claude.com/blog/how-claude-code-works-in-large-codebases-best-practices-and-where-to-start)
