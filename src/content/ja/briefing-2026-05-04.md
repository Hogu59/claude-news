## 開発者ワークフローティップス

### 「CLAUDE.mdは提案、Hookが法律」 — 遵守率70%対100%

Claude Codeコミュニティで繰り返し確認されている重要な教訓があります。**CLAUDE.mdの指示は約70%しか遵守されません**。スタイルの好みなら問題ありませんが、「mainにpush禁止」や「本番データ削除禁止」のようなルールで70%は、本番障害を待っているのと同じです。

**核心的な違い**：
- `CLAUDE.md` = モデルへの**教育**（助言的、確率的遵守）
- `settings.json` + Hook = システムレベルの**強制**（決定論的、100%遵守）
- HookはLLMの推論チェーンの外で実行されるため、モデルが無視することは不可能です

```json
// settings.json — mainブランチpushブロック（Hookで強制）
{
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "if echo \"$CLAUDE_TOOL_INPUT\" | grep -q 'git push.*main'; then echo 'BLOCKED: main push forbidden' >&2; exit 1; fi"
      }]
    }]
  }
}
```

まとめると：**スタイルと慣習はCLAUDE.mdに、絶対ルールはHookに**設定してください。

### セッション終了時のレトロ + lessons.mdパターン

コミュニティで広がっているプラクティスがあります。セッション終了時にClaudeに「今回のセッションで何を学んだ？」と聞き、その結果を`lessons.md`に保存するパターンです。Claudeがミスを修正されるたびに該当ルールを`lessons.md`に追加すると、時間の経過とともに**プロジェクト特化の学習が蓄積**されます。

保存先の基準：
- 一般的なプロジェクトコンセプト → `CLAUDE.md`
- コーディング規約・良い慣行 → ルールファイル
- 技術的な能力 → Skillファイル
- プロジェクトの詳細 → Markdownドキュメント
- アーキテクチャの決定 → ADR（Architecture Decision Records）

```bash
# セッション終了時のレトロ
claude "今回のセッションで学んだことをまとめて。
一般ルールはCLAUDE.mdに、コーディング規約はlessons.mdに追加して。"
```

### ロシアの組織的Wikipedia汚染 — AI学習データ汚染の脅威（5/3）

Atlantic Councilがロシアの組織的Wikipedia操作キャンペーンを分析した報告書を発表しました。「Pravdaネットワーク」と呼ばれる80以上の偽ニュースポータルが親ロシアの言説を拡散し、ウクライナ関連48件のWikipedia記事で組織的な編集パターンが検出されました。

**AI開発者にとって重要な理由**：WikipediaはLLM学習の核心データソースです。偽の親ロシア出典がWikipediaに植え込まれると、学習した**AIチャットボットのユーザーが反ウクライナ・反西側メッセージに晒される**ことになります。データポイズニングが単なるサイバー攻撃ではなく、国家レベルの認知戦ツールとして活用されているという警告です。

Claude CodeでRAGやナレッジベースシステムを構築する際、Wikipediaデータの信頼性検証レイヤーの導入を検討する必要があります。

[Atlantic Council](https://www.atlanticcouncil.org/)

## エコシステム＆プラグイン

### CTX — Claude Codeクロスセッションメモリプラグイン

Claude Codeのセッション間コンテキスト喪失問題を解決するプラグインです。プロンプト送信時に3種類のコンテキストを自動注入します：

- **G1**：Gitログベースの意思決定タイムライン（過去の推論の理解）
- **G2**：BM25セマンティック検索による関連ファイル自動注入
- **CM**：SQLite FTS5ベースの会話履歴検索（オプションのベクターサポート）

10,000+ターンで**0.880のメモリリコール精度**、コンテキスト注入の遅延は**1ms以内**です。クラウド依存なしで完全にローカルで動作します。

```bash
# インストール
pip install ctx-retriever && ctx-install
# またはClaude Codeで
/plugin install ctx@jaytoone
```

[GitHub](https://github.com/jaytoone/CTX)

### VibeFrame — コーディングエージェント向けストーリーボードベース動画CLI

Claude Codeのようなコーディングエージェントが**マークダウンファイルで動画を制作**できるCLIツールです。`STORYBOARD.md`と`DESIGN.md`がエージェントが読み書きするインターフェースとして機能します。

- `vibe build`：アセット生成とシーン合成
- `vibe render`：MP4出力
- `vibe inspect`：レンダー結果のレビュー
- HTML/CSS/JSベースの決定論的合成、FFmpeg編集

画像・動画・ナレーション・音楽生成のマルチプロバイダーをサポートし、dry-runで事前テストが可能です。

[GitHub](https://github.com/vericontext/vibeframe)

## おすすめコラム＆読み物

- **「オープンソースがオープンコミュニティを意味するわけではない」**：GitHubがオープンソースのメンテナンスにIssue管理、PRレビュー、コミュニティモデレーションという無給労働を追加したと分析しています。小規模プロジェクトではIssue・PRを無効化し、信頼できる少数と作業するか、ソロで運営しながらもオープンソースであり得るという主張です。「現代的なオープン開発への期待は膨らみすぎており、選択事項だ」という結論が挑発的です。[blog.feld.me](https://blog.feld.me)

- **「DO_NOT_TRACK — CLIツールのためのプライバシー標準」**：`NO_COLOR`のように`DO_NOT_TRACK=1`一つですべてのCLIツールのテレメトリを無効化しようという提案です。広告追跡、使用量レポート、クラッシュレポーティング、非必須のサードパーティリクエストを包括します。開発ツールを作る際にこの標準を尊重することが、ユーザーの信頼を高める方法です。[donottrack.sh](https://donottrack.sh)
