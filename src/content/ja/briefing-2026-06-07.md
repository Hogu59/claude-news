## 開発者ワークフローティップス

### Skills・Subagents・CLAUDE.md —「どれだけコンテキストを隔離するか」で見分ける

この3つを混同して使うと、ツールセットが散らかります。最も分かりやすい見分け方は、**「その内容がメインの会話コンテキストをどれだけ占有・汚染するか」** です。

- **CLAUDE.md = 常にオンのルール**：毎セッション必ずロードされます。「すべてのセッションで常に真であること」（ビルドコマンド、コアの規約、絶対禁止事項）だけを置きます。
- **Skill = 現在の会話にロードされる知識**：作業の文脈に合わせて入り込み、その場にとどまり、Claude が他の文脈と並べて見ます。**たまにしか必要にならない**再利用ワークフローや深い知識に向いています。
- **Subagent = 隔離されたコンテキスト**：独自のコンテキストウィンドウで作業し、**要約だけ**をメインに返します。内部で何が起きてもメインセッションを汚染しません。

サブエージェントに回すべきサインは明確です — **(1)** 作業が冗長な出力を生むが、それがメインのコンテキストには不要なとき、**(2)** ツールや権限を絞っておきたいとき、**(3)** 自己完結していて要約を受け取れば済むとき。「毎セッションのルール → CLAUDE.md、たまに使うワークフロー → Skill、コンテキストを膨らませる作業 → Subagent」と覚えておきましょう。

[Claude Code Docs — Subagents](https://code.claude.com/docs/en/sub-agents)

### 静的な deny ルールの先へ — PreToolUse フックの `permissionDecision` で呼び出しごとに権限をコードで判断

昨日紹介した `deny: ["*"]` のような**静的な**権限ルールは強力ですが、「条件付き」のポリシーを表現するのは苦手です。より動的には、**PreToolUse フック**がツール実行の直前に、**呼び出し単位で** allow/deny/ask を判断できます。フックが stdout で `hookSpecificOutput.permissionDecision` を返すと、その呼び出しが自動承認・自動拒否されるか、ユーザー確認へエスカレーションされます。

```json
{
  "hookSpecificOutput": {
    "hookEventName": "PreToolUse",
    "permissionDecision": "deny",
    "permissionDecisionReason": "本番 DB のパスへの書き込みは禁止します"
  }
}
```

たとえば「特定のパス/コマンドのパターンなら自動拒否、安全な読み取りコマンドなら自動承認」のように、静的な allow/deny リストでは組みにくいポリシーをシェルスクリプトで実装できます。昨日の deny glob が「一行で面（めん）をロックする」ツールなら、これは「呼び出しごとに判断を差し込む」ツールです。正確な出力スキーマは公式 hooks ドキュメントに従ってください。

[Claude Code Docs — Hooks](https://code.claude.com/docs/en/hooks)

### 遅いシェルは Claude Code を蝕む — 足すのではなく削る

Claude Code はターミナルで動き、Bash ツールのシェルを**ユーザーのプロファイル（`~/.zshrc` など）で初期化**します。重いシェル起動設定（`oh-my-zsh` のような肥大化したフレームワーク、同期式の git プロンプト）は、その初期化や日常的なコマンド実行全体に遅延を加えます。ある開発者はシェル起動を **約30ms** まで短縮しましたが、鍵は「機能を足すのではなく削ること」でした。

- `oh-my-zsh` を丸ごと有効化する代わりに、**本当に必要なプラグイン3つだけ**を選択的に source
- `compinit` をキャッシュし、セキュリティ監査を**1日1回**だけ実行
- `nvm` のようにめったに使わないツールは**遅延ロード（lazy-load）**
- **非同期プロンプト**で git の参照が入力応答を妨げないように
- GPU アクセラレーション対応のターミナルエミュレータを使う

本日のおすすめ読み物「Code is Cheap」とまったく同じ筋です — **最適化とは、たいてい何かを取り除くこと**です。

[mijndertstuij.nl — Life is Too Short for a Slow Terminal](https://mijndertstuij.nl/posts/life-is-too-short-for-a-slow-terminal/)

- **「Code is Cheap（コードは安くなった）」 — htmx.org**：AI コーディングがコードの*生産*コストを劇的に下げた一方で、生成されたコードを*理解*するコストを同時に押し上げた、というエッセイです。LLM の出力は決定論もソースの保存もなく、コンパイラの出力のようには扱えません。複雑さはシステム規模に応じて**少なくとも指数的に**増えます。その答えとして、コードを積み上げる「ビルダー」ではなく、不要なコードを削る **「引き算のエンジニア（subtractive engineer）」**、彫刻家のような態度を提示します。生成をエージェントに委ねるほど、人間の仕事は「取り除くこと」へ移っていく、という洞察です。[htmx.org](https://htmx.org/essays/code-is-cheap/)

- **「C++: The Documentary — 世界で最も重要なプログラミング言語」 — Herb Sutter（6/4）**：Bell Labs を起点に、標準化を経て、ゲーム・金融・AI 計算に至る **C++ の40年史**を、言語を作った中心人物たちとともに辿るドキュメンタリーが6月4日に公開されました。エージェントがコードを大量に生み出す時代に、数十年を生き延びる言語と、その背後の人々・決断を振り返ることは、「長く生きるソフトウェアとは何か」を改めて考えさせます。[herbsutter.com](https://herbsutter.com/2026/06/04/c-the-documentary-released-today/)

## 注目プロジェクト＆ツール

- **tabyAgent — より軽く、セルフホストしやすいエージェントプラットフォーム**：OpenClaw・Hermes の代替で、ホストマウントなしに Docker 内で動作し、Telegram で対話する軽量エージェントです。OpenClaw に比べて **RAM 消費が約9分の1**、単一のインストールスクリプトでセットアップでき、マルチメッセンジャー対応や画像生成といった複雑な機能はあえて省いて保守性を高めています。Playwright によるブラウザ自動化、自律的なスケジュールタスク、ファイル・ターミナル操作、会話をまたいだメモリ管理を提供します（AGPL 3.0、v0.3.0）。重いエージェントフレームワークが負担だった人向けのミニマルな選択肢です。[GitHub](https://github.com/gpdir16/tabyAgent)

- **KOLongDoc — 韓国語の長文ドキュメント VLM ベンチマーク**：ChatGPT・Claude・Gemini のようなマルチモーダルモデルが **韓国の公共機関の文書**をどれだけよく読めるかを評価するオープンソースのベンチマークです。複数ページにわたる高解像度文書の処理、ページ間の情報を統合する**マルチホップ推論**、長文コンテキストの理解を200の評価項目で測定し、データセットは HuggingFace で公開されています。エージェントが実際の行政文書を扱う機会が増える中、韓国語の長文理解を具体的に検証しようとする試みです。[GitHub](https://github.com/Marker-Inc-Korea/KOLongDoc)
