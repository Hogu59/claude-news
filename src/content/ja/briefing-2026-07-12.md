## 主要な新機能と実践活用

### Claude Code Desktopに内蔵ブラウザ — エージェントがWebを直接読んでクリックする (7/10)

Anthropicが**Claude Code Desktopアプリの内蔵(in-app)ブラウザ**を公開しました。Claudeがドキュメント・デザイン・任意のサイトを**直接開いて読み、クリックし、操作**します — ローカルdevサーバーのプレビューを扱ってきたのと同じやり方が、外部Webにも届くようになりました。

- **開き方**: macOSは`Cmd+Shift+B`、Windowsは`Ctrl+Shift+B`、またはViewsメニュー。チャットに外部リンクが現れると、**内蔵パネルで開くか既定ブラウザで開くかの選択**が出ます。
- **サンドボックス隔離**: **個人の資格情報・履歴と分離された別のブラウジングプロファイル**で動き、**セッション(クッキー・ローカルストレージ)を再起動後も保持するかはユーザーが選択**します。外部サイトでの操作は**安全分類器(safety classifier)がレビュー**します。
- **ログイン対応**: **Google OAuthポップアップを含むWebサイトログイン**が使えるため、認証付きWebアプリをエージェントに直接テストさせるフローにそのまま使えます。

```text
# Claude Code Desktop
Cmd+Shift+B (macOS) / Ctrl+Shift+B (Windows)
→ 内蔵ブラウザパネルを開く (Viewsメニューからも可)
→ セッション保持(persist)の有無は設定で選択
```

要点は**「ブラウザが追加ツール(MCP・拡張)からエージェントハーネスの標準部品に入った」**ことです — 7/9のブリーフィングで*Claude Browser・Claude Preview MCP名の予約*を「デスクトップ側にブラウザ統合が準備中というシグナル」と読みましたが、まさにそれが出てきました。ドキュメントを読ませるためのURLコピペの往復も、認証付きステージング確認のためのPlaywright MCPの用意も、デスクトップでは一段短くなります。[9to5Mac](https://9to5mac.com/2026/07/10/anthropic-highlights-claude-codes-in-app-browser-on-the-desktop/) · [ClaudeDevs](https://x.com/ClaudeDevs/status/2075635283211772279) · [Desktopドキュメント](https://code.claude.com/docs/en/desktop)

### Claude for Government — Claude Code・CoworkがFedRAMP Highのパブリックベータに (7/7)

Anthropicが**Claude CodeとClaude CoworkをClaude for Government Desktopのパブリックベータ**として提供開始しました。商用顧客が使うものと**同じアプリケーション**を、**FedRAMP High認証環境**で届ける構成です — 公共部門のチームがClaude Codeで公共サービスを支えるシステムを近代化し、Coworkでメモ・RFPレビュー・ケースワーク・資料作成を委任する絵です。

- **データ境界**: 会話履歴は**機関管理デバイスにローカル保存**され、推論はFedRAMP High認証環境で実行されます。
- **管理統制**: 部門レベルの管理者が下位機関にシート・前払い使用量を配分し、**SCIMグループマッピングでシート階層ごとのレート制限・支出上限・モデル制限**を適用します。請求は割当予算に合わせた固定増分・超過不可(not-to-exceed)上限の構造です。
- **監査可能性**: すべての管理操作が**ハッシュチェーン監査ログ**に残り、Anthropic側の機微な操作は**2人承認**を要求、ペネトレーションテスト要約はNDA下で提供されます。標準MDMで配布します。

要点は**「エンタープライズチャネル拡大の政府版が、制度(調達)からプロダクトに降りてきた」**ことです — カリフォルニア調達(7/5)、アルバータ州事例(7/7)、UST提携(7/9)と続いた流れの中で、今回は*規制要件(FedRAMP High・監査ログ・予算上限)をプロダクト機能として吸収*した形です。規制産業(金融・医療)でClaude Code導入を検討する組織なら、ここにある統制リストがそのまま参考リストになります。Anthropicが契約・請求当事者なので、別途クラウド契約なしに始められます。[Claude発表](https://claude.com/blog/bringing-claude-code-and-claude-cowork-to-government)

## 開発者ワークフローティップス

### モデルは「タスク種別ごと」に選べ — 12モデル×4アプリ×5回の実測が示したもの (7/12)

TryAIが**12個のモデルに同じアプリ4種(Doom風レイキャスター迷路・3Dルービックキューブ・電卓・ライフゲーム)を各5回ずつ**作らせた比較実測を公開しました。GPT-5.6の3ティア(Sol・Terra・Luna)・GPT-5.5・Claude Opus 4.8・Fable 5・Grok 4.5・Muse Spark 1.1に、オープンウェイト(Qwen 3.7 Plus・DeepSeek V4 Pro・Kimi K2.6・GLM-5.2)まで含まれます。

- **最上位ティアが常に勝つわけではありません**: 課題ごとの逆転が繰り返されました — レイキャスターはGPT系が先行しましたが、**ルービックキューブはFable 5が唯一の5/5**、**電卓はClaude(Opus・Fableとも)が全勝**でした。
- **オープンウェイトはドキュメント豊富な課題でコスト競争力**: Qwen・GLMはよくあるパターンの課題はこなしましたが、目新しい課題で崩れました。**Grok 4.5はOpus 4.8のコスパ代替**と評価され、コスト・速度の差は極端でした(Lunaは実質タダ同然)。
- **適用法**: パイプライン全体を1つのモデルに固定せず、**課題種別ごとにモデルをルーティング**しましょう — 反復的でドキュメントの多い作業は安いティアに、未知のアルゴリズム課題はフラッグシップに。

```bash
# タスク種別ごとの明示的な固定の例
claude --model claude-sonnet-5      # 日常の反復作業
claude --model claude-fable-5      # 未知のアルゴリズム・最難関
```

要点は7/10ブリーフィングの*SWE-Bench Proの30%に欠陥 — ベンチではなく自分の作業で測れ*と同じ場所にあります — 今回の実測はそのテーゼの**データ版**です: 実際の性能は単一のリーダーボードではなく、*課題×モデルの行列*の形をしています。[TryAI](https://www.tryai.dev/blog/gpt-5.6-build-off-12-models)

### コンテキスト予算を監査せよ — /doctorの未使用スキル・MCP検出とTool Searchの遅延ロード

MCPサーバー・スキル・プラグインを積み上げたままのセッションは、**使わないツール定義が静かにコンテキストを食っています**。最近のツールがこの問題を正面から支援します。

- **`/doctor`で監査**: `/doctor`(別名`/checkup`)がインストール状態の点検に加え、**未使用のスキル・MCPサーバー・プラグインをコンテキストコスト対比で指摘**します — 7/10の肥大化CLAUDE.mdトリミング提案と合わせると、*コンテキスト予算*が診断可能な項目になったわけです。指摘された項目は削除するか、プロジェクトごとに分離しましょう。
- **MCP Tool Searchの遅延ロード**: サーバーが多い構成なら、ツールスキーマを事前に全部積まず**必要時に検索してロードするTool Search**がコンテキスト使用量を大幅に減らします(最大〜95%削減の報告)。大きなMCPスタックをコンテキスト上限の心配なく維持する現実的な経路です。

要点は7/9の*CLAUDE.mdは長さがそのまま性能予算*と同じ原理が**ツール定義にも適用される**ことです — 文であれスキーマであれ、コンテキストに載るすべてが注意予算を消費します。長らく触っていない`.mcp.json`があるなら、今日の`/doctor`一発がそのまま性能チューニングです。[MCPドキュメント](https://code.claude.com/docs/en/mcp) · [Releasebot](https://releasebot.io/updates/anthropic/claude-code)

## セキュリティ・制限事項

### Console APIキーに有効期限設定が追加 — 長寿命キーの衛生をツールが支援 (7/11)

Anthropicが**Claude ConsoleのAPIキー・Admin APIキー作成時に有効期限を設定する機能**を追加しました(7/11)。

- **期限オプション**: 作成時に**プリセット・カスタム期間・Never**から選択します。
- **期限前通知**: **寿命7日以上のキーは、期限前に作成者へメール**が届きます。
- **Admin API連携**: 各キーの期限が**`expires_at`フィールド**で報告され、キーのインベントリを自動監査できます。

要点は**「漏えい事故の定番原因である『期限なしの長寿命キー』を、プラットフォームが標準ツールで防いでくれるようになった」**ことです — CI・無人パイプライン用キーは短い期限+ローテーションが定石ですが、これまでは手動管理頼みでした。実務のアクションは2つ: ① 新しいキーには**必ず期限を設定**し(Neverは例外を正当化できる場合のみ)、② 既存キーはAdmin APIの`expires_at`で**無期限キーをインベントリから洗い出して**交換計画を立てましょう。[Claude Platformリリースノート](https://platform.claude.com/docs/en/release-notes/api) · [Releasebot](https://releasebot.io/updates/anthropic)

### 静かな2日間 — 7/9のOpus 4.8エラー上昇は解消 (7/10〜7/11)

ステータスページ基準で**7/10〜7/11は新規インシデントなし**で過ぎました。直近の1件は**7/9のClaude Opus 4.8のelevated errors**で、同日03:50 UTCに解消済みです。6月末〜7月初の短いインシデントが続いたパターン(6/30・7/2・7/3・7/6)と比べると静かな2日間です — 無人パイプラインの再試行・アラート構成(7/7ブリーフィング)はそのまま維持しつつ、不審な挙動があった場合はまず7/9の時間帯と重なるかを確認してください。[Claude Status](https://status.claude.com/history)

## コミュニティニュース

- **LWN、スクレイパー状況アップデート — 住宅用プロキシの前にIPブロックが無力化されつつある (7/11)**: LWNが**LLM学習データ収集のための大規模スクレイピングが独立サイトの運営を脅かす現状**を整理しました。攻撃側は**数百万の住宅用プロキシIP**(侵害された家庭用デバイス・感染したストリーミング機器・無料VPNの利用者)を回し、IPあたり数リクエストだけ送って交代するため従来のIPブロックが効かず、user-agent偽装で正常トラフィックを装います。防御側の**Anubis(Proof-of-Work)・CAPTCHA・ログイン壁・データ汚染**は正当なユーザーにもコストを課し、GoogleによるIPIDEA・NetNutプロキシ網の遮断も効果は一時的でした — *持続可能な解決策がなければ独立サイトは壁の内側へ後退し、オープンウェブの構造自体が揺らぐ*という警告です。エージェントツールを作り・使う開発者にとって両面の問題です — 自分のサイトの防御コストであると同時に、AI業界のデータ収集慣行が生んだ外部性でもあります。[LWN](https://lwn.net/SubscriberLink/1080822/990a8a5e2d379085/)
- **Ghost Font — 人間は読めるがAIには読みにくい「動くフォント」実験 (7/12)**: 文字を静的なグリフではなく**動く点の時間的パターン**としてエンコードし、人間は映像からメッセージを知覚できるのに、**AIはフレーム単位の解析では解読に失敗**するようにした実験です — 静止スクリーンショットにはノイズしか残らず、意味は動きからのみ立ち上がります。実験ではChatGPT 5.5 Proが19分かけてもメッセージを正確に抽出できませんでした。CAPTCHA応用・AI知覚ベンチマークの可能性を示しつつ、*人間にも読みにくく、ビデオネイティブなモデルがいずれ解けるかもしれない*という限界も自認しています。上のLWN記事と並べると — *AIが読むWeb*への抵抗手段がフォント・プロトコルの層まで降りてくる流れの小品です。[Ghost Font](https://www.mixfont.com/ghost-font)

## おすすめコラム＆読み物

- **『成功した企業はいかにして盲目になるか』**: **目を失ったメキシコ洞窟魚**(遺伝子は残るが発現が止まった)をメタファーに、成功した組織が**能力そのものではなく能力の「発現」を失う**過程を解剖したエッセイです。急成長期の採用は基準を下げ、他の環境を知らないエンジニアが1年で面接官になり*内部の混乱に慣れた人*を選ぶスパイラルが回り、参入障壁の高い市場は規律を強制する競合を消します — その環境で**丁寧なエンジニアリングは「洞窟が餌を与えない器官」**、つまり退化形質になります。希望も添えられています: 魚と違って人の抑圧された能力は**環境を変えれば再び発現**します。7/11の『人間が保守するつもりでコードを書け』(LLMは悪いパターンをセッションの標準として複製する)と重ねて読むと — *環境が形質の発現を決める*という同じテーゼが、コードベースと組織の2つの層で繰り返されます。[原文](https://ianreppel.org/how-successful-companies-go-blind/)
- **『Lispへの道: なぜLispか』**: Lispの本質は括弧ではなく**言語を問題に合わせて拡張すること**だと説く入門エッセイです(7/9)。コードとデータが同じ形をしている**同図像性(homoiconicity)**のおかげで、マクロは評価前のコードをデータとして変換し新しい制御構造を作り、**REPL駆動開発**は数週間生き続けるプロセスに関数・変数を再定義しながら無停止で開発させます — AutoLISPとEmacsがその拡張性の実例です。エージェント時代に読み直す価値があります — スキル・フック・スラッシュコマンドで**ツールを自分の問題に合わせて育てる**Claude Codeの慣行は、*言語(ツール)を固定された製品ではなく拡張可能な媒体として扱う*Lisp的感覚の現代版です。HNコメントの*ミスを防ぐ言語 vs 最大の力を与える言語*論争もあわせて読む価値があります。[原文](https://scotto.me/blog/2026-07-09-why-lisp/)

## 注目プロジェクト＆ツール

- **korea100 — 公務員がAIで韓国の制度100個を体系図に**: 現職の公務員が**韓国の主要な制度・行政手続き100個を、それぞれ1枚の体系図(フローチャート)**に整理したプロジェクトです。法令を人が読んで図式化すれば数年かかるところを、**AIで法令文書→体系図の変換を加速**して作り上げ、作者はこれを*一般的なAIスキル教育ではなく、AIリテラシーでドメイン(行政)の専門性を深める*事例として紹介しています — 行政手続きを宅配便のように追跡する将来構想も示しました。7/11のレトロゲーム韓国語化スキルが*バイナリのリバースエンジニアリング*だったとすれば、今回は*法令というドメインテキスト*でエージェント活用が専門性を増幅するもう1つの実例です。[korea100](https://hosungseo.github.io/korea100/)
- **MAYDAY — 燃料切れでゲームオーバーの代わりに救難信号を出すマルチプレイ飛行ゲーム**: *月1億3千万ウォン稼いだという飛行ゲーム*が気になって自作したというブラウザマルチプレイゲームです。燃料が尽きるとゲームオーバーではなく**全パイロットに見えるmayday救難信号**が上がり、他のプレイヤーが救助に向かいます — 進行度は救助回数(★)だけで積み上がります。技術構成が印象的です: **Three.js+Vite、外部アセットゼロ**(プリミティブと手続き的WebAudioのみ)、手続き生成の無限ワールド(砂漠・海・雪原・都市)、全ユーザー同期の24分昼夜サイクル、そして**マルチプレイはNode.jsのWebSocketファイル1つ**で処理します。*アセットなしの手続き生成でどこまで行けるか*を示す、小さく完結したリファレンスです。[MAYDAY](https://playmayday.com)
